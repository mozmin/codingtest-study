### 문제 설명 (요약)
- 1번부터 n번까지 번호가 붙은 배양체들이 트리 형태로 연결되어 있다. 
- 각 배양체를 연결하는 파이프는 A, B, C 세 종류 중 하나이며, 처음에는 모든 파이프가 닫혀 있다.
- 처음에는 하나의 배양체만 바이러스에 감염되어 있다. 한 번의 행동으로 A/B/C 중 한 종류의 파이프를 모두 열 수 있고, 열린 파이프를 통해 감염이 퍼진다.
- 이 행동을 최대 `k`번 반복했을 때, 감염시킬 수 있는 배양체 개수의 최대값을 구하는 문제이다.

### 입력
- int n : 배양체의 개수
- int infection : 처음 바이러스에 감염된 배양체 번호
- int[][] edges : 파이프 연결 정보, ex) edges[i] = [x, y, type] : x번 배양체와 y번 배양체가 tpye 종류의 파이프로 연결되어 있다라는 뜻
- int k : 파이프를 열었다 닫을 수 있는 최대 행동 횟수

### 출력
- 최대 `k`번 파이프를 열었다 닫은 후 감염시킬 수 있는 배양체 개수의 최대값을 반환한다.

<hr>

### 핵심 풀이 전략
1. DFS로 A/B/C 중 어떤 파이프를 열지 선택한다.
2. 선택한 타입의 파이프를 기준으로 BFS를 수행하여 감염을 확산시킨다.
3. 현재 감연된 배양체 개수를 세어 최대값을 갱신시킨다.
---

```java (오답, 해설 코드 참고)
import java.util.*;

class Solution {

    // 최대 감염 수 저장
    int answer = 0;

    // 최대 행동 횟수
    int maxK;

    public int solution(int n, int infection, int[][] edges, int k) {

        maxK = k;

        // 그래프 생성
        List<int[]>[] graph = new ArrayList[n + 1];

        for (int i = 1; i <= n; i++) {
            graph[i] = new ArrayList<>();
        }

        // 그래프 연결
        for (int[] edge : edges) {

            int from = edge[0];
            int to = edge[1];
            int type = edge[2];

            graph[from].add(new int[]{to, type});
            graph[to].add(new int[]{from, type});
        }

        // 감염 여부 배열
        boolean[] infected = new boolean[n + 1];

        // 시작 감염 노드
        infected[infection] = true;

        // DFS 시작
        dfs(graph, infected, 0);

        return answer;
    }

    // 행동 순서를 전부 탐색하는 DFS
    public void dfs(List<int[]>[] graph, boolean[] infected, int depth) {

        // 현재 감염 수 계산
        answer = Math.max(answer, countInfected(infected));

        // 행동 횟수 다 사용하면 종료
        if (depth == maxK) {
            return;
        }

        // A, B, C 타입 전부 시도
        for (int type = 1; type <= 3; type++) {

            // 현재 감염 상태 복사
            boolean[] nextInfected = infected.clone();

            // 해당 타입 파이프 열어서 감염 퍼뜨리기
            spread(graph, nextInfected, type);

            // 다음 행동으로 진행
            dfs(graph, nextInfected, depth + 1);
        }
    }

    // 특정 타입 파이프를 열었을 때 감염 확산
    public void spread(List<int[]>[] graph, boolean[] infected, int openType) {

        Queue<Integer> queue = new ArrayDeque<>();

        // 현재 감염된 노드들을 큐에 삽입
        for (int i = 1; i < infected.length; i++) {
            if (infected[i]) {
                queue.offer(i);
            }
        }

        // BFS 시작
        while (!queue.isEmpty()) {

            int now = queue.poll();

            for (int[] next : graph[now]) {

                int nextNode = next[0];
                int pipeType = next[1];

                // 같은 타입 파이프이고 아직 감염 안 됐으면
                if (pipeType == openType && !infected[nextNode]) {

                    infected[nextNode] = true;

                    queue.offer(nextNode);
                }
            }
        }
    }

    // 감염된 개수 세기
    public int countInfected(boolean[] infected) {

        int count = 0;

        for (int i = 1; i < infected.length; i++) {
            if (infected[i]) {
                count++;
            }
        }

        return count;
    }
}
```

### 삽질 기록 🧠
- 처음에는 DFS로 현재 노드를 기준으로 타입이 같으면 계속 이동하고, 타입이 다르면 파이프를 여는 방법으로 접근했다.
- 하지만 해당 문제 조건과 맞지 않았다. 그 이유로는 이 문제에서는 현재 노드의 이동이 아니라, 현재 감염된 전체 노드의 집합을 기준으로 풀어야 했기 때문이다.
---

### 배운 점 && 보완할 점
- boolean[] nextInfected = infected.clone(); 와 같이 조건 분기 때마다 배열의 초기화가 필요한 로직이라면,
- 매번 로직이 끝나고 방문 여부를 초기화 하는 것이 아닌 분기 내에서 원본 배열을 복사한 뒤 처리하는 방법도 배우게 되었다.
- 해설 코드의 핵심은 1. DFS(시작 위치에서 어떤 파이프를 열지), 2. 해당 시작 위치에서부터 BFS, 3.현재 위치에서 감염된 개수를 세기
- 이렇게 총 3단계로 나눠서 사고를 진행했다는 점이 중요한 것 같다.
