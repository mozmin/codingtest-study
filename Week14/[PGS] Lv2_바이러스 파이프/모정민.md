### 문제 설명 (요약)
`n`개의 노드와 3가지 종류의 파이프(1, 2, 3)로 연결된 네트워크가 주어질 때, 초기 감염 노드 `infection`에서 시작하여 `k`번 동안 특정 파이프를 열어 바이러스를 확산시켰을 때 가능한 최대 감염 노드 수를 구하는 문제.

- **조건**: 
  - 파이프의 종류는 3가지이며, 매 턴마다 하나의 파이프 종류만 선택하여 열 수 있다.
  - 선택한 파이프와 연결된 미감염 노드로만 바이러스가 전파된다.
  - 총 `k`번의 파이프 선택(분기)이 이루어진다.

---

### 핵심 풀이 전략
1. **모든 경우의 수 탐색 (DFS)**: 1, 2, 3번 파이프 중 하나를 선택하는 과정을 `k`번 반복해야 하므로, DFS를 활용해 $3^k$ 가지의 모든 파이프 선택 조합을 탐색한다.
2. **독립적인 상태 관리 (배열 복사)**: 각 파이프 선택 시나리오마다 감염 상태가 다르게 진행되므로, 이전 상태에 영향을 주지 않기 위해 매 분기마다 `visited`(감염 상태) 배열을 깊은 복사하여 넘겨준다. 백트래킹(상태 복구)이 까다로운 시뮬레이션 문제에서 매우 효과적인 방법이다.
3. **동시다발적 전파 (다중 출발점 BFS)**: 특정 파이프를 열었을 때, "현재 감염되어 있는 모든 노드"가 새로운 전파의 진원지가 될 수 있다. 따라서 BFS 큐를 초기화할 때 감염된 모든 노드를 큐에 넣고 시작한다.
4. `k`번의 탐색이 끝나면 `visited` 배열에서 `true`인 노드의 개수를 세어 `max` 값을 갱신한다.

---

### 내가 구현한 구현 코드
```java
import java.util.*;

class Solution {
    
    int max;
    public int solution(int n, int infection, int[][] edges, int k) {
        
        max = Integer.MIN_VALUE;
        
        // 노드 초기화
        ArrayList<int[]>[] nodes = new ArrayList[n+1];
        for(int i = 0; i <= n; i++){
            nodes[i] = new ArrayList<>();
        }
        
        for(int[] edge : edges){
            int u = edge[0];
            int v = edge[1];

            nodes[u].add(new int[]{v, edge[2]});
            nodes[v].add(new int[]{u, edge[2]});
        }
        
        // 감염 여부 및 방문 처리용
        boolean[] visited = new boolean[n+1];
        visited[infection] = true;
        
        // 분기 시작
        dfs(n, nodes, visited, 0, k);
        
        return max;
    }

    public void dfs(int n, ArrayList<int[]>[] nodes, boolean[] visited, int count, int k){
        
        // 만약 k만큼 돌았다면
        if(count >= k){
            int result = 0;
            for(boolean v : visited){
                if(v) result++;
            }
            
            max = Math.max(max, result);
            
            return;
        }
        
        // 경우의 수 (A, B, C) 분기점
        for(int i = 1; i <= 3; i++){
            
            // 새 분기점 전용 visited 배열
            boolean[] copy = new boolean[n+1];
            for(int j = 1; j <= n; j++){
                copy[j] = visited[j];
            }
            
            // pipe 확인, 얕은 복사 활용
            bfs(i, nodes, copy);
            
            // 다음 분기점, 얕은 복사 활용
            dfs(n, nodes, copy, count+1, k);
        }
        
        return;
        
    }
   
    public void bfs(int pipe, ArrayList<int[]>[] nodes, boolean[] visited){
        
        Deque<Integer> q = new ArrayDeque<>();
        
        // 감염된 노드들 큐에 삽입
        for(int i = 1; i < visited.length; i++){
            if(visited[i]) q.offer(i);
        }
        
        while(!q.isEmpty()){
            
            int curNode = q.poll();
            
            // curNode 인접 리스트 확인
            for(int[] n : nodes[curNode]){
                
                int nxtNode = n[0];
                int connectedPipe = n[1];
                
                // 만약 nxtNode 감염이 안된 상태(방문 상태)이고, connectedPipe가 열려있다면   
                if(!visited[nxtNode] && connectedPipe == pipe){
                    
                    // 큐에 추가 후 감염(방문 처리)
                    q.offer(nxtNode);
                    visited[nxtNode] = true;
                }
            }
        }
        
        return;
    }
}
```
### 칭찬 및 회고 🧠
- **탁월한 상태 관리 판단**: 이런 유형의 전파 시뮬레이션에서는 BFS를 돌리고 난 후 원래 상태로 복구(백트래킹)하는 것이 거의 불가능에 가깝다. 이때 `boolean[] copy`를 만들어 새로운 분기점으로 넘기는 판단이 정답의 핵심 키였고, 이를 완벽하게 구현했다.
- **다중 출발점 BFS**: 특정 파이프를 열었을 때 시작 노드 하나에서만 퍼지는 것이 아니라 "이미 감염된 모든 곳"에서 퍼져나가야 한다는 점을 캐치하여 큐에 `visited`가 `true`인 모든 노드를 초기 세팅한 논리 흐름이 아주 좋았다.

### 배운 점 && 보완할 점
- **배열의 얕은 복사/깊은 복사**: `for`문을 돌려 배열을 하나하나 복사하는 방식도 정확하게 작동하지만, 자바에서 제공하는 `.clone()`이나 `Arrays.copyOf()`를 사용하면 코드를 훨씬 깔끔하게 유지할 수 있다. (참고: 배열의 경우 `.clone()`은 깊은 복사(Deep Copy)로 동작한다.)
- **최댓값 초기화**: 노드의 개수는 음수가 될 수 없고 무조건 1개 이상 감염되므로 `max` 값을 `Integer.MIN_VALUE` 대신 `0`으로 초기화해도 무방하다.
