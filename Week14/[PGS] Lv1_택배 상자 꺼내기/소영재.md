### 문제 설명 (요약)
- 상자 n개를 가로 길이 w만큼 지그재그 형태로 쌓는다.
- 이때 특정 상자 번호 num을 꺼내기 위해 해당 상자 위에 쌓여 있는 상자까지 포함해 총 몇 개의 상자를 꺼내야 하는지 구하는 문제이다.

### 입력
- int n : 전체 상자의 개수
- int w : 한 층에 놓을 수 있는 상자의 개수
- int num : 꺼내야 하는 상자의 번호

### 출력
- num 상자를 꺼내기 위해 꺼내야 하는 상자의 개수 (num 포함)
<hr>

### 핵심 풀이 전략
1. 전체 상자 개수 n과 가로 길이 w를 이용해 필요한 높이를 구한다. (이때 나머지가 발생하면 높이를 하나 추가)
2. 구한 높이와 매개변수로 전달받은 가로 길이로 2차원 배열을 만든다.
3. 짝수 번째 줄은 왼쪽에서 오른쪽, 홀수 번째 줄은 오른쪽에서 왼쪽으로 상자를 한 줄씩 채운다.
4. 배열에서 num의 위치인 row, col을 찾는다.
5. num이 있는 위치부터 위쪽 방향으로 같은 열에 존재하는 상자의 개수를 센 뒤, 반환한다.

---

```java
class Solution {

    public int solution(int n, int w, int num) {
        int height = 0;
        int answer = 0;
        
        if(n%w == 0){
            height = n/w;
        } else {
            height = (n/w) + 1;
        }

        int[][] map = new int[height][w];

        int number = 1;

        for (int i = 0; i < height; i++) {

        
            if (i % 2 == 0) {
                for (int j = 0; j < w && number <= n; j++) {
                    map[i][j] = number;
                    number++;
                }
            } else {
                for (int j = w - 1; j >= 0 && number <= n; j--) {
                    map[i][j] = number;
                    number++;
                }
            }
        }

        int row = -1;
        int col = -1;

        for (int i = 0; i < height; i++) {
            for (int j = 0; j < w; j++) {

                if (map[i][j] == num) {
                    row = i;
                    col = j;
                }
            }
        }

        for (int i = row; i < height; i++) {
            if (map[i][col] != 0) {
                answer++;
            }
        }
        return answer;
    }
}
```

### 삽질 기록 🧠
- 짝수, 홀수 번째에 상자를 넣을 때 number <= n 조건을 넣지 않아서 값이 다르게 나왔었다.
- 배열과 실제 인덱스 번호 간 통일성에 대해 고민을 좀 해야겠다는 생각을 했다.
