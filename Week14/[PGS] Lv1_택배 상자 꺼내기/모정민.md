# [Level 1] 택배 상자 꺼내기

### 문제 설명 (요약)
`n`개의 택배 상자가 가로 길이 `w`인 창고에 지그재그(스네이크) 형태로 쌓여 있습니다. 이때 특정 번호의 상자 `num`을 꺼내기 위해, 그 위에 쌓여 있는 상자들까지 포함하여 총 몇 개의 상자를 꺼내야 하는지 구하는 문제.

- **조건**: 
  - 1층은 왼쪽에서 오른쪽으로, 2층은 오른쪽에서 왼쪽으로 방향이 번갈아가며 쌓인다.
  - 타겟 상자 `num`과 동일한 열(세로 줄)에 있고 층수가 더 높은 상자는 모두 꺼내야 한다.

---

### 핵심 풀이 전략
1. **2차원 배열 시뮬레이션**: 전체 상자가 들어갈 수 있는 층수(`v`)를 구하고 `v x w` 크기의 2차원 배열을 만든다.
2. **지그재그 배치**: `switchBox`라는 논리 변수를 활용해 짝수 층은 정방향(`0`부터 `w-1`까지), 홀수 층은 역방향(`w-1`부터 `0`까지)으로 상자 번호를 채워 넣는다.
3. **목표물 좌표 추적**: 상자 번호를 채우는 과정에서 타겟 번호 `num`이 등장하면 해당 좌표 `(y, x)`를 기록한다.
4. **결과 계산**: 기록해둔 좌표에서 시작하여 꼭대기 층까지 올라가며, 해당 위치에 상자가 존재(`!= 0`)하면 카운트를 1씩 증가시킨다.

---

### 내가 구현한 구현 코드
```java
import java.util.*;

class Solution {
    public int solution(int n, int w, int num) {
        int answer = 0;
        
        int v = (int) Math.ceil((double) n / (double) w);
        int[][] boxes = new int[v][w];
        for(int[] b : boxes) Arrays.fill(b, 0);
        
        // 꺼낼 박스 좌표
        int y = 0;
        int x = 0;
        
        // false -> 왼쪽에서 오른쪽으로 정방향
        // true - > 오른쪽에서 왼쪽으로 역방향
        boolean switchBox = false;
        
        // 박스 채우기
        int index = 1;
        for(int i = 0; i < v; i++){
            // 정방향
            if(index <= n && !switchBox){
                for(int j = 0; j < w; j++){
                    if(index <= n){
                        boxes[i][j] = index;
                        if(index == num){
                            y = i;
                            x = j;
                        }
                        index++;
                    }
                }
                switchBox = true;
            }
            // 역방향
            else if(switchBox){
                for(int j = w-1; j >= 0; j--){
                    if(index <= n){
                        boxes[i][j] = index;
                        if(index == num){
                            y = i;
                            x = j;
                        }
                        index++;
                    }
                }
                switchBox = false;
            }
        }
        
        // 상자꺼내기
        while(y < v){
            if(boxes[y][x] != 0) answer++;
            y++;
        }
        
        return answer;
    }
}
```

### 회고 🧠
- **직관적인 시뮬레이션 구현**: 문제에서 제시된 "지그재그 패턴"을 2차원 배열과 `switchBox` 변수를 사용해 현실의 모습 그대로 시뮬레이션. 복잡한 수학 공식을 떠올리려다 막히는 것보다, 이렇게 정확한 구현력으로 확실하게 풀어내는 것이 실전 코딩 테스트에서는 가장 강력한 무기가 된다.
- **`Math.ceil` 처리**: 총 필요한 층수 `v`를 구할 때 `int` 나눗셈으로 인한 소수점 절사를 막기 위해 `(double)`로 캐스팅하고 올림 처리한 꼼꼼함이 돋보인다.

### 배운 점 && 보완할 점
- **2차원 배열 초기화**: 자바에서 `new int[v][w]`로 배열을 생성하면 모든 요소가 자동으로 `0`으로 초기화되므로, 굳이 `Arrays.fill(b, 0)`을 순회하며 해줄 필요는 없다.
- **수학적 패턴 찾기 (O(1) 메모리)**: 지그재그나 달팽이 모양 배열 채우기 문제는 인덱스의 '몫'과 '나머지'를 활용하면 2차원 배열을 직접 만들지 않아도 좌표를 구할 수 있다. 
  - 정방향 층: `열 번호 = 나머지`
  - 역방향 층: `열 번호 = (가로 길이 - 1) - 나머지`
  이 공식을 숙지해두면 추후 비슷한 패턴의 문제에서 시간/공간 복잡도를 극적으로 줄일 수 있다.
