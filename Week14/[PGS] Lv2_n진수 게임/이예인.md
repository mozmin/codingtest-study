### 문제 설명
0부터 시작하는 숫자를 차례대로 `n진법`으로 변환한 뒤, 여러 사람이 한 글자씩 돌아가며 말하는 게임이다. <br>
이때 전체 참가자 수가 m명이고, 튜브의 순서가 p번째일 때, 튜브가 말해야 하는 문자 t개를 구하는 문제이다.<br>

### 입력
- n: 변환할 진법
- t: 튜브가 미리 구해야 하는 숫자의 개수
- m: 게임에 참가하는 인원
- p: 튜브의 순서

### 출력
- 튜브가 자신의 차례에 말해야 하는 숫자들을 이어 붙인 문자열

---

### 핵심 풀이 전략
- 0부터 시작해서 숫자를 하나씩 n진법으로 변환한다.
- 변환한 값을 하나의 긴 문자열에 계속 이어 붙인다.
- 튜브가 말할 숫자는 전체 문자열에서 일정한 간격으로 등장한다.
- 튜브의 첫 번째 위치는 p - 1이다.
  - 문제의 순서는 1부터 시작한다.
  - 문자열 인덱스는 0부터 시작하므로 p - 1로 맞춰야 한다.
- 이후에는 참가자 수 m만큼 건너뛰면서 문자를 선택한다.
- 총 t개의 문자를 모으면 정답이 된다.

---

### 제출 코드
```java
import java.util.*;
class Solution {
    public String solution(int n, int t, int m, int p) {
        
        StringBuilder sb = new StringBuilder();
        
        for(int i = 0 ; i < m * t; i++){
            sb.append(calculate(n, i));
        }
        
        StringBuilder answer = new StringBuilder();
        for(int i = 0 ; i < t; i++){
            answer.append(sb.charAt(i*m + p-1));    
        }
        
        return answer.toString();
    }
    
    // 진법으로 바꾸기 -> k를 n진법으로 바꾸기
    StringBuilder calculate (int n, int k){
        StringBuilder sb = new StringBuilder();
        if(k == 0) {
            sb.append(0);
            return sb;
        }
        while(k != 0){
            int a = k % n;
            k /= n;
            if(a >= 10) {
                sb.insert(0, (char)('A' + a - 10)) ;
            }
            else{
                sb.insert(0, a);
            }
            
        }
        return sb;
    }
}
```

### 삽길 기록 🧠
- 처음에는 p를 그대로 인덱스로 사용해서 한 칸 뒤의 문자를 가져와서 틀렸다.
- charAt()은 0부터 시작하는 인덱스라서 p가 아니라 p - 1 을 가져와야 했다.

### 배운 점 && 보완할 점 
- StringBuilder에서도 n번째 문자를 가져올때 sb.charAt(n)을 사용하면 된다.
- StringBuilder끼리도 append가 가능하다
