# [Level 2] n진수 게임

[문제 링크](https://school.programmers.co.kr/learn/courses/30/lessons/17687)

`#수학` `#문자열` `#진법변환` `#구현`

### 제출 일자
2026년 5월 11일

### 문제 설명 (요약)
숫자를 0부터 시작해서 1씩 키워가며 `n`진법으로 변환해 순서대로 나열할 때, 게임에 참가하는 인원 `m`명 중에서 튜브(순서 `p`)가 말해야 하는 숫자 `t`개를 구하는 문제. (단, 10~15는 각각 대문자 A~F로 출력한다.)

- **입력**: 진법 `n`, 미리 구할 숫자의 갯수 `t`, 게임에 참가하는 인원 `m`, 튜브의 순서 `p`
- **조건**: 
  - `n`진법 변환 시 알파벳은 대문자로 표기한다.
  - 튜브의 첫 차례는 `p-1`번째 인덱스부터 시작하며, 이후 `m`번째마다 돌아온다.

---

### 핵심 풀이 전략
1. **진법 변환 로직 활용**: 0부터 1씩 숫자를 증가시키며 `n`진법으로 변환한다. 자바의 `Integer.toString(index, n)`을 사용하면 진법 변환을 아주 쉽게 처리할 수 있다. (대문자 변환을 위해 `.toUpperCase()`를 함께 사용)
2. **필요한 전체 문자열 길이 계산**: 튜브가 `t`개의 숫자를 말하기 위해서는 튜브 차례가 `t`번 돌아와야 한다. 즉, 전체 문자열은 최소 `m * t` 길이만큼 필요하므로 이 길이 도달 시점까지 숫자를 계속 이어 붙인다 (`StringBuilder` 활용).
3. **튜브의 순서 추출**: 완성된 문자열에서 튜브의 차례에 해당하는 문자만 쏙쏙 뽑아낸다. 인덱스를 1씩 증가시키며 `i % m == p - 1` 조건으로 찾을 수도 있고, 애초에 시작점을 `p - 1`로 잡고 인덱스를 `m`씩 점프시켜 추출할 수도 있다.

---

### 내가 구현한 구현 코드
```java
import java.util.*;

class Solution {
    public String solution(int n, int t, int m, int p) {
        
        StringBuilder sb = new StringBuilder();

        // 0 ~ (m * 1000) 까지 (튜브가 말할 t개를 확보하기 위한 최소 길이 m*t)
        int index = 0;
        while(sb.length() <= (m * t)){
            String nxt = Integer.toString(index, n).toUpperCase();
            sb.append(nxt);
            
            index++;
        }
        
        String value = sb.toString();
        
        StringBuilder result = new StringBuilder();
        for(int i = 0; i < value.length(); i++){
            
            // 튜브의 차례이고, 아직 t개를 다 못 채웠을 때
            if(i % m == p-1 && result.length() < t){
                char c = value.charAt(i);
                result.append(c);
            }
        }
        
        return result.toString();
    }
}
```
### 회고 🧠
- **진법 변환 내장 메서드 활용**: 직접 나머지를 구하고 뒤집는 로직을 짜지 않고 `Integer.toString(index, n)`을 떠올린 것이 신의 한 수! 덕분에 코드가 아주 간결해졌다.
- **종료 조건 설정**: `while`문의 종료 조건을 `sb.length() <= (m * t)`로 설정하여 필요한 만큼의 문자열만 딱 생성해 낸 판단이 아주 훌륭하다. 무한 루프나 불필요한 메모리 낭비를 완벽히 차단했다.

### 배운 점 && 보완할 점
- **반복문 스텝(Step) 조절**: 조건문(`if(i % m == p-1)`)을 통해 필터링하는 방식도 좋지만, 애초에 `for`문의 증감식을 `i += m`으로 설정하면 반복 횟수를 `1/m`로 줄일 수 있고 가독성도 더 직관적으로 좋아진다는 것을 배웠다.
- 문자열 추출 시 굳이 전체를 검사하지 않고 **필요한 인덱스만 점프하며 뽑아내는 스킬**을 기억해 두자.
