### 문제 설명 (요약)
두 문자열 `str1`과 `str2`를 2글자씩 끊어서 다중집합을 만든 뒤, 두 집합의 자카드 유사도를 구하는 문제.

- **입력**: 영문자, 숫자, 특수문자가 섞인 문자열 `str1`, `str2` (길이 2 이상 1,000 이하)
- **조건**: 
  - 영문자로만 이루어진 2글자 쌍만 집합의 원소로 인정하며, 그 외의 문자가 포함된 쌍은 버린다.
  - 대소문자는 구분하지 않는다.
  - 자카드 유사도 J(A, B)는 두 집합의 교집합 크기를 합집합 크기로 나눈 값이다.
  - 집합 A와 집합 B가 모두 공집합일 경우에는 나눗셈이 정의되지 않으므로 J(A, B) = 1로 정의한다.
  - 다중집합의 경우, 교집합은 원소 빈도수의 최솟값, 합집합은 원소 빈도수의 최댓값을 취한다.
- **출력**: 계산된 자카드 유사도에 65536을 곱한 후 소수점 아래를 버린 정수.

---

### 핵심 풀이 전략
1. 대소문자 구분을 없애기 위해 입력된 두 문자열을 `toUpperCase()`를 이용해 모두 대문자로 변환한다.
2. `substring`을 활용해 문자열을 2글자씩 순회하며, 두 글자 모두 영문자인 경우에만 `HashMap`에 담아 다중집합(문자열 쌍 : 빈도수)을 구성한다.
3. 두 `HashMap`을 비교하여 교집합의 크기를 구한다. 이때 다중집합의 교집합은 공통된 키(Key)에 대해 `Math.min(map1의 빈도, map2의 빈도)`를 누적하여 더한다.
4. 합집합의 크기는 |A ∪ B| = |A| + |B| - |A ∩ B| 공식을 활용하여 `(map1의 모든 원소 수 합 + map2의 모든 원소 수 합 - 교집합 크기)`로 간단히 구한다.
5. 합집합의 크기가 0인 경우(두 문자열 모두 유효한 원소가 없는 공집합) 예외 처리하여 65536을 반환하고, 그렇지 않으면 `(교집합 / 합집합) * 65536` 값을 반환한다.

---

### 내가 구현한 구현 코드
```java
import java.util.*;
class Solution {
    
    public int solution(String str1, String str2) {
        
        Map<String, Integer> mapStr1 = new HashMap<>();
        Map<String, Integer> mapStr2 = new HashMap<>();
        
        // 모두 대문자로 변환
        String upperStr1 = str1.toUpperCase();
        String upperStr2 = str2.toUpperCase();

        // 2글자씩 변환
        for(int i = 0; i < upperStr1.length()-1; i++){
            String value = upperStr1.substring(i, i+2);
            if(check(value)) {
                mapStr1.put(value, mapStr1.getOrDefault(value, 0) + 1);
            }
        }
        
        for(int i = 0; i < upperStr2.length()-1; i++){
            String value = upperStr2.substring(i, i+2);
            if(check(value)) {
                mapStr2.put(value, mapStr2.getOrDefault(value, 0) + 1);
            }
        }
        
        // map을 통해 교집합 계산
        int same = 0;
        for(String key : mapStr1.keySet()){
            same += Math.min(mapStr1.getOrDefault(key, 0), mapStr2.getOrDefault(key, 0));
        }
        
        // 합집합 계산
        
        int str1Count = 0;
        for(int i : mapStr1.values()) str1Count += i;
        
        int str2Count = 0;
        for(int i : mapStr2.values()) str2Count += i;
        
        int union = str1Count + str2Count - same;
        
        if(union == 0) return 65536;
        double jaccard = (double) same / union;
        return (int) (jaccard * 65536);
    }
    
    public boolean check(String value){
        
        char c1 = value.charAt(0);
        char c2 = value.charAt(1);
        
        if('A' <= c1 && c1 <= 'Z' && 'A' <= c2 && c2 <= 'Z') return true;
        
        return false;
    }
}
```

### 삽질 기록 🧠
- 처음엔 `substring(i, i+1)`로 작성해서 2글자가 아닌 1글자씩만 잘라오는 실수를 했다.
- `ArrayList`를 써서 빈도를 세려다가 오히려 인덱스와 삭제 관리가 꼬일 뻔했다.
- 다중집합의 전체 원소 개수를 구해야 하는데 `for(int i : mapStr1.values()) str1Count++;`로 작성해서 '요소의 빈도수'가 아닌 '키의 개수'를 세버렸다.
- 가장 치명적으로, 문제 조건에 있던 "둘 다 공집합일 경우 유사도는 1"이라는 조건을 빼먹어 합집합(`union`)이 0일 때 `NaN`이 발생하게 만들었다.

### 배운 점 && 보완할 점
- **다중집합의 교집합과 합집합 계산**: 다중집합 문제를 풀 때 배열이나 리스트를 직접 다루는 것보다 `HashMap`을 통해 `원소: 빈도수` 구조로 매핑하면 훨씬 구현이 편해진다. 
- **다중집합의 합집합 공식**: 합집합을 구하기 위해 `Math.max`를 돌며 모든 키를 취합하는 것보다, 일반적인 집합의 합집합 공식인 $|A \cup B| = |A| + |B| - |A \cap B|$를 빈도수 총합에 적용하면 연산이 훨씬 간결해진다는 것을 배웠다.
