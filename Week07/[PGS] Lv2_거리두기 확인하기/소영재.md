`#BFS`

### 문제 설명 (요약)
- 코로나 바이러스 감염 예방을 위해 응시자들이 거리를 둬서 대기해야 한다.
- 대기실은 5개이며, 각 대기실의 크기는 5x5 크기이다.
- 거리두기를 위하여 응시자들 끼리는 맨해튼 거리가 2이하로 앉아야 한다.
- 단, 응시자가 앉아있는 자리 사이가 파티션으로 막혀 있을 경우에는 허용된다.
- 대기실 요소 중 'P'는 응시자, 'O'는 빈 테이블, 'X'는 파티션을 의미한다.

### 입력
- String[][] places : 응시자들의 정보와 대기실 구조를 대기실별로 담은 2차원 문자열 배열

### 출력
- 대기실별로 거리두기를 지키고 있다면 1, 한 명이라도 지키지 않고 있으면 0을 int 배열에 담아 출력

<hr>

### 핵심 풀이 전략
1. 반복문을 돌며 대기실을 하나 고르고 그 안에서 사람 'P'를 찾는다.
2. isValid 변수는 현재 대기실의 거리두기를 판별하는 boolean 변수이다.
3. 'P'를 발견하면, 해당 위치를 기준으로 맨해든 커리 2 이하 범위 내에 거리두기 위반이 있는지 확인하기 위해 BFS를 수행한다.
4. bfs 메서드는 현재 사람을 기준으로 거리 2이내에 다른 사람 'P'가 있는지 찾는 역할을 한다.
5. BFS 탐색 과정에서 파티션 'X'는 통과하지 않고, 방문한 좌표는 다시 탐색하지 않으며, 거리가 2를 초과하지 않도록 제한한다.
6. 하나의 대기실에서 거리두기가 한 건이라도 발생한다면 해당 대기실은 0, 모든 'P'에 대해 검사를 통과하면 1을 결과 배열에 저장한다.
---

```java
import java.util.*;
class Solution {
    public int[] solution(String[][] places) {
        int[] answer = new int[places.length];
        
        for(int i = 0; i < places.length; i++){
           //현재 대기실의 거리두기가 정상이라고 가정
           boolean isValid = true;
           
           for(int j = 0; j < places[i].length; j++){
               for(int k = 0; k < places[i][j].length(); k++){
                   //'P'를 찾게된다면 bfs 메서드를 호출, 거리두기가 제대로 되지 않았다면 isValid 값을 false로 변경
                   if(places[i][j].charAt(k) == 'P'){
                      if(!bfs(places[i], j, k)){
                          isValid = false;
                          break;
                      }
                   }
               }
               //해당 행의 거리두기가 이뤄지지 않았다면 해당 대기실은 0이므로 더 찾을 필요 없이 반복문을 종료
               if (!isValid) {
                   break;
               }
           }
           answer[i] = isValid ? 1 : 0;
       }
        return answer;
    }
    
    //'P'가 발견된 멘헤튼 거리 2이하 범위 안에서 거리두기를 위반하는 다른 사람이 있는지 확인
    public boolean bfs(String[] place, int row, int column){
        //Queue 자료구조를 사용하는 이유 : 가까운 칸부터 차례대로 탐색해야 하기 때문 (ex: 거리 0, 거리1, 거리2)
        Queue<int[]> queue = new ArrayDeque<>();
        
        //방문 체크 배열, 같은 칸을 여러 번 탐색하면 안 되기 때문에 사용
        boolean [][] visited = new boolean[place.length][place.length];
        
        //상하좌우 이동을 위한 배열, dr은 행 변화량, dc는 열 변화량을 의미
        //ex : (-1,0) 위, (1,0) 아래, (0,-1) 왼쪽, (0,1) 오른쪽
        int[] dr = {-1, 1, 0, 0};
        int[] dc = {0, 0, -1, 1};
        
        //현재 위치(row, column)와 시작점으로부터 거리(0)을 queue에 등록
        queue.offer(new int[]{row, column, 0});
        
        //시작점은 본 칸이니 방문처리
        visited[row][column] = true;
        
        //queue가 빌 때까지 계속 탐색 = 아직 검사할 좌표가 남아 있으면 반복
        while(!queue.isEmpty()){
            //queue의 맨 앞 요소를 꺼냄 = 검사할 좌표 하나를 가져옴
            int[] current = queue.poll();
            
            //현재 위치와 거리를 각각 꺼내어 대입
            int currentRow = current[0];
            int currentCol = current[1];
            int dist = current[2];
            
            //현재 위치가 거리 2라면 더 이상 주변 칸으로 확장 탐색하지 않음
            if(dist == 2) continue;
            
            //상하좌우 4방향을 하나씩 보는 반복문
            for(int i = 0; i < 4; i++){
                int nr = currentRow + dr[i];
                int nc = currentCol + dc[i];
                
                //배열 범위를 벗어나는 좌표인지 검사, 행이나 열이 0보다 작거나, 배열 길이와 같거나 크면 잘못된 좌표
                if(nr < 0 || nc < 0 || nr >= place.length || nc >= place[0].length()){
                    continue;
                }
                
                //이미 방문한 칸이면 건너뛰기
                if(visited[nr][nc]){
                    continue;
                }
                
                //'X'면 지나갈 수 없으므로 건너뛰기 = 그 방향을 더 이상 탐색하지 않아도 됨
                if(place[nr].charAt(nc) == 'X'){
                    continue;
                }
                
                //다른 사람('P')를 만나게 되면 거리두기 실패, false 리턴
                if(place[nr].charAt(nc) == 'P'){
                    return false;
                }
                
                //방문처리
                visited[nr][nc] = true;
                
                //거리 +1을 증가시키고 다시 탐색
                queue.offer(new int[]{nr, nc, dist +1});
            }
        }
        //queue가 다 빌 때까지 탐색했는데도 다른 P를 찾지 못했다는 뜻, true 리턴
        return true;
    }
}
```

### 삽질 기록 🧠
- 처음 문제를 본 순간 응시자를 기준으로 2칸 이내의 값을 검사해야 한다고 생각했다.
- 응시자 기준으로 가장 가까운 곳을 탐색해야 하므로 DFS가 아닌 BFS를 떠올리게 되었다.
- 하지만 BFS에 대한 개념만 알고 있고 실제로 문제를 풀어본 적이 없었기 때문에 코드를 구현하기가 너무 어려워 예시 코드도 참고하면서 풀었다.
- 시간을 아무리 써도 정말 모르는 부분들은 GPT에게 질문하면서 문제를 풀어나갔다.

### 배운 점 && 보완할 점
- BFS에서는 Queue와 방문처리가 사실상 짝꿍처럼 사용된다는 점을 배우게 되었다.
- Queue 자료구조를 사용하는 이유는 BFS는 거리가 가장 가까운 곳부터 탐색해야 하기 때문이다.
- 방문처리가 필요한 이유는 상하좌우 탐색을 하면서 현재 위치가 계속 변하게 되는데, 이 때 이전에 방문한 곳을 또 방문하여 BFS 탐색을 시작할 수 있기 때문이다.
- (dist == 2)일 때 continue 하는 이유는 거리가 0일 때 상하좌우로 1칸 검사했기 때문이다. (거리가 2일 때는 거리가 3인 상하좌우를 탐색하게 되므로)
