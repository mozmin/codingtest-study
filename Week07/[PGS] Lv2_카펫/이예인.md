### 문제 설명
갈색 격자 수 `brown`과 노란색 격자 수 `yellow`가 주어진다. <br>
카펫의 테두리는 갈색으로 이루어져 있고, 내부는 노란색으로 채워져 있다. <br>
이때 카펫의 가로, 세로 크기를 구하는 문제이다. <br>

카펫의 전체 크기를 `가로 × 세로`라고 할 때,  
내부 노란색 영역의 크기는 `(가로 - 2) × (세로 - 2)`가 된다. <br>

### 입력
- `brown` : 갈색 격자의 개수
- `yellow` : 노란색 격자의 개수

### 출력
- 카펫의 가로 길이와 세로 길이를 담은 배열 `[가로, 세로]`

### 핵심 풀이 전략
노란색 격자 수 `yellow`를 만들 수 있는 모든 약수 쌍을 구한 뒤,  
각 약수 쌍에 테두리 2칸씩을 더한 크기가 전체 카펫 크기와 일치하는지 확인했다. <br>

1. `yellow`의 약수 쌍 `(a, b)`를 찾기
2. 내부 노란색 영역이 `a × b`라고 가정
3. 실제 카펫 크기는 `(a + 2) × (b + 2)`
4. 이 값이 `brown + yellow`와 같으면 정답
5. 가로가 세로보다 크거나 같도록 정렬해서 반환

### 제출 코드
```java
class Solution {
    public int[] solution(int brown, int yellow) {
        int[] answer = new int[2];
        
        if (yellow == 1) return new int[] {3, 3};
        
        for (int i = 1; i <= yellow / 2; i++) {
            if (yellow % i != 0) continue;
            
            int a = yellow / i;
            int b = yellow / a;
            
            if ((a + 2) * (b + 2) == brown + yellow) {
                if (a + 2 >= b + 2) answer = new int[] {a + 2, b + 2};
                else answer = new int[] {b + 2, a + 2};
            }
        }
        
        return answer;
    }
}
```

### 배운 점 && 보완할 점
- 현재는 `yellow / 2`까지 탐색하고 있는데, 사실 `sqrt(yellow)`까지만 탐색해도 충분해서 더 효율적으로 개선할 수 있다.
- `yellow == 1` 예외 처리 없이도 일반 로직으로 해결되도록 작성하면 코드가 조금 더 깔끔해질 것 같다.
