# [Level 2] 서버 증설 횟수

[문제 링크](https://school.programmers.co.kr/learn/courses/30/lessons/389479) 

`#구현` `#시뮬레이션` `#큐`

### 제출 일자
2026년 3월 18일 0.15ms <br>
시간복잡도: **$O(N)$** ( $$N = 24$$ 로 데이터 크기가 고정되어 있으므로 사실상 **$O(1)$** 상수 시간 )

### 문제 설명 (요약)
시간대별(0시~23시) 게임 접속자 수에 맞춰 서버를 증설할 때, 하루 동안 증설된 총 서버의 대수를 구하는 문제. 
초기 서버는 0대이며, 서버 1대당 `m`명의 인원을 수용할 수 있음. 특정 시간대의 접속자 수를 `m`으로 나눈 몫이 해당 시간에 필요한 최소 서버 대수임. 
현재 운영 중인 서버 대수가 필요한 서버 대수보다 적다면, 부족한 개수만큼 서버를 새롭게 증설해야 함. 한 번 증설된 서버는 `k` 시간 동안만 유지된 후 자동으로 만료됨. 매 시간마다 만료되는 서버를 제외하여 현재 유지 중인 서버 수를 갱신하고, 새롭게 증설한 횟수를 계속 누적하여 반환해야 함.

### 입력
- `players`: 0시부터 23시까지 각 시간대별 게임 접속자 수를 나타내는 1차원 정수 배열 (길이는 24로 고정)
- `m`: 서버 1대가 수용할 수 있는 최대 인원 수를 나타내는 정수
- `k`: 한 번 증설된 서버가 운영 및 유지되는 시간을 나타내는 정수

### 출력
하루(24시간) 동안 증설한 서버의 총 개수를 반환

---

### 핵심 풀이 전략
1. Map<반납시간, 증설된 서버 수> 를 저장할 Map 구현
2. 0시부터 23시까지 반복문으로 탐색
3. 현재 i시에서 플레이어 수가 증설된 서버 수로 감당이 되는지 확인 (단, 중요한 점은 플레이어 수가 m 보다 작은 경우는 서버 증설을 할 필요가 없음. 말 그대로 '증설' 이기에 기본적으로 서버 1개는 가지고 있는 상황)
4. 플레이어가 수가 m보다 많은 경우 서버 증설이 필요하므로, `(현재 플에이어 수 - (증설된 서버 수 x m)) / m 으로 증설할 서버 수를 계산. 나눈 후 나머지는 기본 서버로 충족이 되므로 나머지에 대한 계산은 필요 없음.
5. 서버 증설이 될 경우 answer += (증설된 서버) 계산
6. 각 반복문마다 현재 Map<>에서 반납해야할 서버를 삭제.
7. answer 반환

---

### Java 구현 코드

```java
import java.util.*;

class Solution {
    public int solution(int[] players, int m, int k) {
        int answer = 0;
        
        // 증설한 서버 <반납 시간, 갯수>
        Map<Integer, Integer> server = new HashMap<>();
        
        for(int i = 0; i < players.length; i++){
            
            // i시간의 플레이어 수
            int playersCount = players[i];
            
            // i시간 기준 증설된 서버 수
            int currentServer = 0;
            for(Integer count : server.values()){
                currentServer += count;
            }
            
            // 현재 증설한 서버 수가 감당 가능한지 체크
            // 증설이 필요한 경우 Map<>의 합 n * m < playersCount
            if(currentServer * m < playersCount && playersCount >= m){
                
                int needsServer = (playersCount - (currentServer * m)) / m;
                
                server.put(i+k, needsServer);
                answer += needsServer;

            }
            
            // 반납해야 할 서버 확인
            if(server.containsKey(i+1)){
                server.remove(i+1);
            }
            
        }
        
        return answer;
    }
}
```

### 삽질 기록 🧠
1. 잘못된 Map<> 반복문 사용
- 처음에 Map<>에 대한 반복문 사용을 몰라 해맸는데, map은 keySet()이나 values()를 통해 돌 수 있었다.
- 처음에 for문으로 반복을 돌면서 i+1 key 값이 Map<>에 있으면 지우는 if문을 만들었다. 하지만 Map<>은 for문을 돌 때 값이 변경 되면 에러가 발생한다.
- if문 밖에다가 break문을 두어서 매번 for문이 첫번째로 확인한 key 값만 보고 바로 반복문이 종료되어버렸다. -> **`Map<>은 순서를 보장하지 않는다.`**
- for문 사용이 비효율적인 것 같아 containsKey()를 활용하여 리팩터링을 진행했다.

2. 나머지 값의 활용
- 증설할 서버 수를 구할 때, 몫과 나머지를 따로 구해서 나머지가 0보다 크면 `증설할 서버`를 `+1` 하는 로직을 만들었다.
- 하지만, 실제 문제에서는 `플레이어 수` 가 `m` 보다 작은 경우는 서버를 굳이 증설하지 않는다. -> 기본적으로 서버를 하나 가지고 있기 때문이다.
- 따라서 나머지를 활용하던 로직을 삭제하였다.
