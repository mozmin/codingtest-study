### 문제 설명 (요약)
- 시간대 별로 이용자가 늘어날 때마다 서버의 대수가 몇 대 필요한지 구하는 문제
- 서버는 초기에 0대로 시작하며, 서버 한 대당 (m)명을 수용할 수 있다.
- 서버의 운영 시간은 (k)시간동안 운영되며, (k)시간이 지나면 서버는 만료된다.
### 입력
- int[] players : 0시에서 23시까지의 시간대별 게임 이용자의 수를 나타내는 1차원 정수 배열
- int m : 서버 한 대당 감당할 수 있는 최대 이용자의 수
- int k : 서버 한 대가 운영 가능한 시간을 나타내는 정수
### 출력
- 모든 게임 이용자를 감당하기 위한 최소 서버 증설 횟수
<hr>

### 핵심 풀이 전략
- 1. 종료될 서버의 수를 먼저 선언할 때 (k)시간만큼의 배열을 추가로 선언한다.
- 2. 반복문을 돌면서 현재 활성화된 서버에 종료될 서버를 반영한다.
- 3. m명당 필요한 서버의 개수를 계산하고 활성화된 서버보다 필요한 서버의 수가 부족하다면 추가해야 할 서버와 종료될 서버를 계산한 뒤 정답에 더한다.
---

```java
import java.util.*;
class Solution {
    public int solution(int[] players, int m, int k) {
        int time = 24;
        int answer = 0;
        //종료될 서버의 수
        int[] server = new int[time+k];
        //현재 시점에 활성화된 서버의 수
        int activeServer = 0;
        
        for(int i = 0; i < time; i++){
            //현재 시점에 활성화된 서버에 종료될 서버를 반영함
           activeServer -= server[i];
           //m명당 필요한 서버의 개수를 계산
            int needServer = players[i] / m;
            
            //만약 활성화된 서버보다 필요한 서버의 수가 부족하다면 추가될 서버의 개수와 k시간 뒤 종료될 서버를 계산
            if(needServer > activeServer){
                int addServer = needServer - activeServer;
                activeServer += addServer;
                server[i+k] += addServer;
                answer += addServer;
            }
        }
        return answer;
    }
}
```

### 삽질 기록 🧠
- 현재 시점의 서버, 종료될 서버, 추가될 서버에 대해 어떻게 선언하고 계산을 할지 좀 헤매었다.
