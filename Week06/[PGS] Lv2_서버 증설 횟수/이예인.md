### 문제 설명
각 시간대별 이용자 수가 주어질 때, 서버 1대는 최대 m명의 이용자를 감당할 수 있고, 한 번 증설한 서버는 k시간 동안 유지된다.<br>
매 시간 필요한 서버 수를 확인하면서, 현재 동작 중인 서버 수가 부족하면 그만큼 서버를 추가한다. <br>
이때 전체 서버를 몇 번 증설했는지 구하는 문제이다.<br>

### 입력
- players: 시간대별 이용자 수
- m: 서버 1대가 감당할 수 있는 이용자 수
- k: 서버가 유지되는 시간

### 출력
- 전체 증설한 서버 수

### 핵심 풀이 전략
현재 시간에 동작 중인 서버 수를 server[i]로 관리했다. <br>
players[i] / m을 통해 현재 시간에 필요한 서버 수를 계산 <br>
현재 동작 중인 서버 수 server[i]보다 필요 서버 수가 크면 부족한 만큼 서버를 증설 <br>
새로 증설한 서버는 k시간 동안 유지되므로, 이후 k개의 시간대에 모두 반영 <br>
매 시간마다 필요한 서버 수 - 현재 서버 수를 구해 부족한 만큼만 추가하고, 그 추가분을 이후 k시간 동안 누적시키는 방식으로 해결했다. <br>

### 제출 코드
```java
class Solution {
    public int solution(int[] players, int m, int k) {
        int answer = 0;
        int time = 0;
        int[] server = new int [players.length];
        for(int i = 0 ; i < players.length; i++){
            if(players[i] / m > server[i]){
                int add = players[i] / m - server[i];  // 증가한 server 수
                answer += add;
                for(int j = 0 ; j < k ; j++){
                    if(i+j < players.length) server[i+j] += add;

                }
            }
        }
        
        
        
        return answer;
    }
}
```

### 삽길 기록 🧠
처음에는 아래처럼 반복문 안에서 바로 증가량을 계산했다.
```
server[i + j] += (players[i] / m - server[i]);
```
하지만 이 경우 j=0일 때 server[i] 값이 먼저 바뀌어 버려서, 그 다음 반복부터는 증가량이 0이 되는 문제가 있었다.<br>

그래서 증가해야 할 서버 수를 먼저 add 변수에 저장한 뒤, 그 값을 k시간 동안 동일하게 더해주는 방식으로 수정했다. <br>

또한 처음에는 players[i] / 3처럼 상수를 사용했는데, 문제 조건에 맞게 players[i] / m으로 고쳤다. <br>
