### 문제 설명 (요약)
직사각형 형태의 공원에서 빈 공간을 찾아 가지고 있는 돗자리 중 가장 큰 것을 펼치려 합니다. 공원에는 이미 사람들이 앉아 있는 위치가 `'A'~'Z'`와 같은 알파벳으로 표시되어 있으며, 빈 공간은 `'-1'`로 표시되어 있습니다.

돗자리를 펼치기 위해서는 해당 크기의 정사각형 영역 전체가 `'-1'`로 채워져 있어야 합니다. 가지고 있는 돗자리들의 변의 길이가 주어졌을 때, 공원의 빈 공간에 펼칠 수 있는 **가장 긴 변의 길이**를 구하는 문제입니다. 만약 어떤 돗자리도 펼칠 수 없다면 `-1`을 반환해야 합니다.

### 입력
- `mats`: 지니고 있는 돗자리들의 한 변의 길이들이 담긴 정수 배열
- `park`: 공원의 자리 배치 상태를 나타내는 2차원 문자열 배열

### 출력
공원에 펼칠 수 있는 돗자리 중 가장 큰 한 변의 길이를 반환 (불가능할 경우 `-1`)

---

### 핵심 풀이 전략
1. park의 최대 길이는 50x50 이므로 이중 for문으로 모든 공원 칸을 전부 돌아도 된다. (50*50d은 끽해야 2500이므로)
2. 돗자리 int[] mat를 내림차순으로 정렬하여 큰 돗자리부터 확인할 수 있도록 한다.
3. park 칸마다 이중 for문을 돌며 -1 값만 돗자리를 필 수 있는지 확인한다.
4. 돗자리 크기 m x m 모두 -1이면 answer에 값을 넣고 true를 반환한다.
5. 최종적으로 answer를 반환한다.

---

### Java 구현 코드

```java
import java.util.*;

class Solution {
    
    int answer;
    
    public int solution(int[] mats, String[][] park) {
        
        answer = -1;
        
        int[] newMat = Arrays.stream(mats).boxed()
            .sorted(Comparator.reverseOrder())
            .mapToInt(i -> i).toArray();
        
        for(int m : newMat){
        
            for(int i = 0; i<park.length; i++){
            for(int j = 0; j < park[0].length; j++){
                
                if(park[i][j].equals("-1") && findMat(i, j, m, park)) return answer;
            }
        }
            
        }
        
        return answer;
    }
    
    public boolean findMat(int i, int j, int m, String[][] park){
        
            
        int count = 0;
        outer: for(int x = i; x < i+m; x++){
            for(int y = j; y < j+m; y++){
                    
                if(x < park.length && y < park[0].length && park[x][y].equals("-1")) count++;
                else break outer;
            }
        }
            
        if(count == m*m){
            answer = m;
            return true;
        }
        
        return false;
    }
    
}
```

### 삽질 기록 🧠
1. 배열 실수
  - stream().sorted() 결과를 원본 배열에 반영하지 않음. Arrays.stream(mats)...toArray()는 새 배열을 반환하는데, 반환값을 저장하지 않아서 mats가 실제로 정렬되지 않았음.
  - int[]는 Comparator를 직접 사용할 수 없다는 점을 헷갈림. int[]는 primitive 배열이라 Arrays.sort(arr, Comparator) 불가. → 내림차순이 필요하면 정렬 후 뒤집기 또는 스트림으로 boxing 후 정렬해야 함.

2. Math.pow(m, 2)를 비교식에 사용함
  - Math.pow()는 double을 반환해서 정수 비교에 불필요하고 비효율적임. → m * m으로 비교하는 것이 더 안전하고 명확함.

3. 탐색 순서를 잘못 설계함
  - 가장 큰 돗자리부터 모든 park를 다 돌아야 하는데, 각 park 칸 마다 돗자리 for문을 돌려서 작은 돗자리가 answer가 되는 케이스가 존재했음.
    
---

### 배운 점 && 보완할 점
1. 정렬에 대해서
  - `Comparator.reverOrder()` 와 `Collection.reverseOrder()` 기능은 동일하다. 단, 그냥 통일성을 위해 `Comparator.reverseOrder()`를 쓰자.
  - int[] 는 내림차순 정렬을 그냥 할 수 없다. 객체인(String, Integer) 경우는 Arrays.sort(arr, Comparator.reverseOrder())로 가능하다.
  - int[] 내림차순은 **Arrays.stream(arr).boxed().sorted(Comparator.reversedOrder()).mapToInt(i -> i).toArray()**를 쓰자. 생각이 나지 않는다면 **직접 for문으로 temp**를 써서 만들자.

2. 문제 조건
  - 탐색 순서에 따라 정답이 상이해줄 수 있다는 것을 배웠다.
