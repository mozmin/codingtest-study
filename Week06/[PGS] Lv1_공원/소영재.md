### 문제 설명 (요약)
- 이미 깔린 돗자리를 제외한 빈 공간에 지민이가 깔 수 있는 돗자리의 가장 긴 한 변의 길이를 구하는 문제
- 만약 돗자리를 깔 수 없다면 -1을 반환
### 입력
- int[] mats : 지민이가 가진 돗자리들의 한 변의 길이가 담긴 정수 배열
- String[][] park : 현재 공원의 자리 배치도를 의미하는 2차원 문자열 배열

### 출력
- 지민이가 깔 수 있는 돗자리의 가장 긴 한 변의 길이, 만약 깔 수 있는 돗자리가 없다면 -1
<hr>

### 핵심 풀이 전략
- 1. 돗자리를 깔 수 있는 경우의 수에서 가장 한 변의 길이가 가장 긴 돗자리부터 확인한다.
- 2. 반복문을 돌면서 돗자리를 깔 수 있는 ("-1")지점을 찾고, 해당 위치를 시작점으로 잡고 돗자리를 깔 수 있는지 확인한다.
---

```java
import java.util.*;
class Solution {
    public int solution(int[] mats, String[][] park) {
        //가장 큰 돗자리를 찾는 문제이기 때문에 오름차순으로 정렬
        Arrays.sort(mats);
        //돗자리의 길이가 긴 것부터 검사
        for(int m = mats.length -1; m >=0; m--){
            int mat = mats[m];
            //시작점을 구하는 반복문, 시작점은 돗자리 크기보다 값이 작아야 함
            for(int i = 0; i <= park.length - mat; i++){
                for(int j = 0; j <= park[i].length - mat; j++){
                    //시작점이 -1이라면 돗자리를 깔 수 있는지 확인
                    if(park[i][j].equals("-1")){
                        //돗자리를 깔 수 있다면 해당 mat를 반환
                        if(calPlace(park, mat, i, j)) {
                            return mat;
                        }
                    }
                }
            }
        }
        //모든 돗자리를 깔 수 없다면 -1 반환
        return -1;
    }
    //시작점을 기준으로 돗자리를 깔 수 있는지 확인
    public boolean calPlace(String[][] park, int mat, int x, int y){
        for(int i = x; i < x + mat; i++){
            for(int j = y; j < y + mat; j++){
                if(!park[i][j].equals("-1")){
                    return false;
                }
            }
        }
        return true;
    }
}
```

- 고정된 시각(모든 공원의 상태를 확인하고, 해당 위치에 돗자리를 깔 수 있는지를 확인)으로 접근하느라 오래 걸렸다.

### 배운 점 && 보완할 점
- 모든 경우의 수를 확인하는 것이 아니라 시작점을 기준으로 생각을 한다면 접근이 쉬워진다는 것을 깨닫게 되었다.
- 의식의 흐름대로 코드를 작성하다보니 내가 생각한 것과 반대로 조건을 적는다거나, 변수 명을 다르게 쓰는 등 실수가 잦았던 문제이다.
- 조금 더 머리 속에서 정리하고 푸는 습관을 만들어야겠다.
- 배열을 오름차순으로 정리하고 반복문에서 큰 값부터 변수에 담는 방식을 선택했는데 찾아보니 오름차순부터 정렬하는 것이 아니라 내림차순부터 정렬할 수 있는 메서드도 존재했다.
- Arrays.sort(mats, Collections.reverseOrder());
- 하지만 위 코드의 단점은 우선 Collections.reverseOrder()는 객체(Integer)타입만 가능해서 형변환을 해줘야 한다는 단점이 있었다.
- 만약 문제에 적용하게 된다면
- Integer[] temp = Arrays.stream(mats).boxed().toArray(Integer[]::new);
- Arrays.sort(temp, Collections.reverseOrder());
- mats = Arrays.stream(temp).mapToInt(Integer::intValue).toArray();
- 위와 같은 복잡한 방식으로 구현하게 되어 번거로워지는 단점이 있다.
