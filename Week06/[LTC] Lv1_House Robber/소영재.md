# [난이도] House Robber

[문제 링크](https://leetcode.com/problems/house-robber/)

`#DFS`

### 제출 일자

2026년 3월 17일

### 문제 설명 (요약)
- 인접한 두 집을 제외한 거리의 집들을 털면서 경찰에 발각되지 않고 오늘 밤 털 수 있는 최대 금액을 반환하는 문제

### 입력
- int nums[] : 집에 보관된 돈이 담겨있는 int 배열

### 출력
- 집에 있는 돈을 최대로 털 수 있는 금액
<hr>

### 핵심 풀이 전략
1. 시간초과 문제로 동일한 재귀함수 ex : dfs(1), dfs(2), dfs(3)..의 호출 결과는 같으므로 이 값을 저장하는 Integer 배열 memo를 선언했다.
2. 여기서 int[] 배열이 아닌 Integer[] 배열을 선언한 이유는 null을 판단하기 위해서이다. (int는 초기값 0)
3. dfs를 호출하면서 만약 이전에 계산된 dfs(1), dfs(2).. 값이 나올 경우 재귀함수를 호출하지 않고 저장된 값을 반환한다.
4. memo[i]에는 dfs(i)의 결과 즉 i번째 집부터 마지막 집까지 훔칠 수 있는 최대 금액이 저장된다.
---

```java
class Solution {
    //dfs 호출 결과를 담는 변수
    Integer[] memo;

    public int rob(int[] nums) {
        memo = new Integer[nums.length];
        return dfs(nums, 0);
    }

    public int dfs(int[] nums, int depth) {
        //현재 위치가 nums.length를 넘어서면 종료
        if (depth >= nums.length) {
            return 0;
        }
        
        //memo[depth]안에 null이 아닌 값이 존재한다면 이를 활용
        if (memo[depth] != null) {
            return memo[depth];
        }
    
        //인접한 두 집을 털 수 없기 때문에 해당 depth를 +2 해줌
        int nowCurrent = nums[depth] + dfs(nums, depth + 2);
        //현재 집을 털지 않고, 다음 집을 털기
        int skipCurrent = dfs(nums, depth + 1);
        
        memo[depth] = Math.max(robCurrent, skipCurrent);
        return memo[depth];
    }
}


```

### 삽질 기록 🧠
- 시간 초과가 나서 다른 방식을 찾게 되었다.
```java
 class Solution {
    int answer = 0;
    public int rob(int[] nums) {
        dfs(nums, 0, 0);
        return answer;
    }
    public void dfs(int[] nums, int depth, int sum) {
        if (depth >= nums.length) {
            answer = Math.max(answer, sum);
            return;
        }
        dfs(nums, depth + 2, sum + nums[depth]);
        dfs(nums, depth + 1, sum);
    }
}
```
- 이미 계산된 dfs메서드의 호출 값을 다시 계산하고 있어서 시간 초과가 난 것 이었다..

### 배운 점 && 보완할 점
- 재귀함수를 호출하면서 동일한 값이 반복적으로 나타나는 형태가 보인다면 이를 기억하는 변수를 선언하면 시간 초과 문제를 어느정도 해겷할 수 있다는 점을 배웠다.
- 만약, 같은 호출이라도 경우의 수가 달라지는 문제라면 이를 활용할 수 없기 때문에 사용할 때 조심해야 할 것 같다.
