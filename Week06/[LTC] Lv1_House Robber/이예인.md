시간복잡도: O(N)

### 문제 설명
일직선으로 이어진 여러 집이 있고, 각 집에는 돈이 들어 있다.<br>
인접한 두 집을 동시에 털면 경보가 울리기 때문에, 서로 붙어 있는 집은 함께 털 수 없다. <br>
각 집에 있는 돈의 양이 배열 `nums`로 주어질 때, 경보를 울리지 않고 훔칠 수 있는 돈의 최댓값을 구하는 문제이다. <br>

### 입력
- 정수 배열 `nums`

### 출력
- 훔칠 수 있는 최대 금액 

### 핵심 풀이 전략
이 문제는 현재 집을 터는 경우와 털지 않는 경우 중 더 큰 값을 선택해 나가는 DP 문제이다. <br>

`dp[i]`를 0번째 집부터 i번째 집까지 고려했을 때 훔칠 수 있는 최대 금액이라고 정의했다. <br>

그러면 점화식은 다음과 같다. <br>

- 현재 집을 털지 않는 경우: `dp[i - 1]`

- 현재 집을 터는 경우: `nums[i] + dp[i - 2]`

```
dp[i] = Math.max(dp[i - 1], nums[i] + dp[i - 2]);
```
로 계산할 수 있다. <br>

초기값은 다음과 같이 설정했다. <br>

- `dp[0] = nums[0]`

- `dp[1] = Math.max(nums[0], nums[1])`

또한 집이 1개뿐인 경우를 대비해 예외 처리도 해주었다. <br>

### 제출 코드
```java
import java.util.*;

class Solution {
    public int rob(int[] nums) {
        // 집이 1개인 경우
        if(nums.length == 1) return nums[0];

        int[] dp = new int[nums.length];
        dp[0] = nums[0];
        dp[1] = Math.max(nums[0], nums[1]); // 2번째 집의 경우 

        for(int i = 2; i < nums.length; i++){
            dp[i] = Math.max(nums[i] + dp[i - 2], dp[i - 1]);
        }

        return dp[nums.length - 1];
    }
}
```

### 삽길 기록 🧠
처음에는 i == 0 || i == 1일 때 모두 dp[i] = nums[i]로 처리했다. <br>

하지만 이렇게 하면 dp[1]이 1번째 집까지 고려했을 때의 최대값이 아니라 그냥 1번째 집의 값만 저장하게 된다.<br>

예를 들어 dp[1]은 nums[1]이 아니라 Math.max(nums[0], nums[1])가 되어야 한다. <br>

즉, DP 배열의 의미를 정확히 정의하지 않으면 초기값에서부터 점화식이 어긋날 수 있다는 점을 배웠다. <br>

또한 처음 구현할 때 집이 1개만 있는 경우와 2개만 있는 경우 같은 엣지 케이스를 충분히 고려하지 못했다. <br>

집이 1개인 경우 → 그 집의 돈이 바로 정답 <br>

집이 2개인 경우 → 두 집 중 더 큰 값을 선택 <br>
