### 문제 설명 (요약)
거리를 따라 늘어선 집들의 돈을 훔치려 할 때, 인접한 두 집을 연속으로 털면 보안 시스템이 작동해 경찰에 경보가 울린다는 제약 조건 하에서 훔칠 수 있는 최대 금액을 구하는 문제. 
동적 계획법(DP)을 활용하여, 특정 위치의 집까지 도달했을 때 해당 집을 털었을 경우와 털지 않았을 경우의 누적 최댓값을 계산하며 최적해를 도출해야 함.

### 입력
- nums: 각 집에 보관된 금액이 거리의 순서대로 담긴 0 이상의 정수 배열

### 출력
경보를 울리지 않고 하룻밤 동안 훔칠 수 있는 최대 금액 반환

<hr>

### 핵심 풀이 전략
1. nums[] 와 동일한 크기의 int[] dp 리스트 선언
2. for문을 돌면서 dp의 값을 채움
3. i-1은 방문하지 못하므로 i-2, i-3 값을 비교
4. 최종적으로 dp 리스트에서 가장 큰 값 반환
---

### 핵심 풀이 전략

```java
class Solution {

    public int rob(int[] nums) {

        int[] list = new int[nums.length];

        for(int i = 0; i < nums.length; i++){

            if(i == 0 || i == 1) list[i] = nums[i];
            else{
                int pop = 0;
                list[i] = nums[i];
                for(int j = 0; j < i-1; j++){
                    if(pop < list[j]){
                        System.out.println(pop + "");
                        list[i] = list[j] + nums[i];
                        pop = list[j];
                    }
                }
            }
        }

        int max = -1;
        for(int i = 0; i < nums.length; i++){
            max = Math.max(max, list[i]);
        }
        return max;
    }
}
```

### 배운 점 && 보완할 점
1. 내가 푼 방식은 이중 for문을 사용해서 시간 복잡도가 **꽝**이다... 내가 푼 방식은 i-1 이전까지 이중for문으로 가장 큰 dp 값을 사용하는 방식이다.
2. 하지만 가장 정석적인 방식은 i-2와 i-3만 비교하여 nums[i]와 더하는 방식이다.
3. 마지막 max 값을 구하는 경우도 정렬을 사용한 후 dp 리스트 맨 마지막 값만 반환 해주면 된다.
4. 그 외 leet code에서 본 다른 풀이 방식들은 `Array.stream()` 을 사용하였다.

### 개선된 코드
```java
class Solution {
    public int rob(int[] nums) {
        if(nums == null || nums.length < 1) return 0;
        // Edge case: only one house
        if (nums.length == 1) return nums[0];

        int n = nums.length;
        int[] dp = new int[n];
        dp[0] = nums[0];
        dp[1] = Math.max(nums[0], nums[1]);

        for(int i = 2; i < n; i++) {
            dp[i] = Math.max(dp[i-1], nums[i] + dp[i-2]);
        }

        return dp[n - 1];
    }
}
```
