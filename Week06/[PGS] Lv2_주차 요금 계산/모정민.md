# [Level 2] 주차 요금 계산

[문제 링크](https://school.programmers.co.kr/learn/courses/30/lessons/92341) 

`#구현` `#시뮬레이션` `#해시`

### 제출 일자
2026년 3월 19일 13.99ms <br>

### 문제 설명 (요약)
주차장의 요금표와 차량의 입차, 출차 기록이 주어졌을 때, 차량별로 누적 주차 시간을 계산하여 요금을 정산하는 문제.
- 입차 후 출차 기록이 없다면 23:59에 출차된 것으로 간주함.
- 누적 주차 시간이 기본 시간 이하라면 기본 요금만 부과.
- 초과 시 초과 시간 / 단위 시간을 올림하여 초과 요금을 가산.
- 최종 결과는 차량 번호가 낮은 순서대로 요금만 배열에 담아 반환.

### 입력
- fees: [기본 시간, 기본 요금, 단위 시간, 단위 요금]
- records: ["시각 차량번호 내역", ...] (예: "05:34 5961 IN")

### 출력
차량 번호 오름차순으로 정산된 요금을 담은 1차원 정수 배열

---

### 핵심 풀이 전략
1. 시간 변환: "HH:MM" 형태의 문자열을 계산하기 편하도록 HH * 60 + MM (분 단위 정수)로 변환함.
2. 입차 기록 관리: Map<String, Integer> parking을 사용하여 차량별 입차 시각을 저장하고, 출차 시 삭제하며 시간을 계산함.
3. 누적 시간 합산: Map<String, Integer> totalTime을 사용하여 차량별로 하루 동안 머문 총 시간을 합산함. (이때 차량 번호 순 정렬을 위해 TreeMap을 쓰면 유리함)
4. 미출차 차량 처리: 반복문 종료 후 parking 맵에 남아있는 차량들을 1439분(23:59) 기준으로 일괄 정산함.
5. 요금 계산: 각 차량의 누적 시간이 기본 시간을 초과했는지 판단하고, 초과 시 Math.ceil()과 (double) 형변환을 사용하여 올림 계산을 정확히 수행함.

---

### Java 구현 코드

```java
import java.util.*;

class Solution {
    public int[] solution(int[] fees, String[] records) {
        
        // 기록용 Map<>
        Map<String, Integer> map = new HashMap<>();
        
        // 총 누적 시간 기록 Map<>
        Map<String, Integer> result = new HashMap<>();
            
        int defaultTime = fees[0];
        int defaultFee = fees[1];
        int partTime = fees[2];
        int partFee = fees[3];
        
        // Map<>에 값 넣기
        for(int i = 0; i < records.length; i++){
            
            int HH = Integer.parseInt(records[i].substring(0, 2));
            int MM = Integer.parseInt(records[i].substring(3, 5));
            int curTime = HH * 60 + MM;
            
            String carNumber = records[i].substring(6, 10);
            String value = records[i].substring(11);
            
            if(value.equals("IN")){
                map.put(carNumber, curTime);
            }else if(value.equals("OUT")){
                int inTime = map.get(carNumber);
                result.put(carNumber, result.getOrDefault(carNumber, 0) + (curTime - inTime));
                map.remove(carNumber);
            }
        }
        
        // 남아있는 Map<> 값 처리
        int lastTime = 23 * 60 + 59;
        for(Map.Entry<String, Integer> entry : map.entrySet()){
            result.put(entry.getKey(), result.getOrDefault(entry.getKey(), 0) + (lastTime - entry.getValue()));
        }
        
        // 정답 형태로 변환 + 정산
        int index = 0;
        int[][] resultFees = new int[result.size()][2];
        for(Map.Entry<String, Integer> entry : result.entrySet()){
            
            resultFees[index][0] = Integer.parseInt(entry.getKey());
            
            if(entry.getValue() <= defaultTime){
                resultFees[index][1] = defaultFee;
            }else{
                resultFees[index][1] = defaultFee + (int) Math.ceil((double) (entry.getValue() - defaultTime) / partTime) * partFee;
            }
            index++;
        }
        
        Arrays.sort(resultFees, Comparator.comparingInt((int[] a) -> a[0]));
        
        int[] answer = new int[result.size()];
        for(int i = 0; i < answer.length; i++){
            answer[i] = resultFees[i][1];
        }
        return answer;
    }
}
```

### 삽질 기록 🧠
1. 문자열 파싱 범위 오류 (substring 미스)
- substring(11, 13)을 사용하여 "OUT"을 읽으려 했으나, 끝 인덱스 미포함 원칙에 의해 **"OU"**만 추출됨.
- 이로 인해 if(value.equals("OUT")) 조건이 항상 거짓이 되어 출차 처리가 전혀 되지 않음.
- 해결: substring(11)로 끝까지 가져오거나 split(" ")을 사용하여 공백 기준으로 자름.

2. 시간 계산 순서 오류
- (inTime - curTime)으로 계산하여 시간이 음수로 누적됨.
- 해결: 나중 시각(curTime)에서 이전 시각(inTime)을 빼야 함.

3. 요금 계산 시 기본 시간 차감 누락
- 초과 요금 계산 시 전체 시간 / 단위 시간으로 계산함.
- 해결: 반드시 (전체 시간 - 기본 시간) / 단위 시간으로 계산해야 함.

4. 정수 나눗셈의 올림 문제
- (int / int)는 소수점을 버리므로 Math.ceil()이 무의미해짐.
- 해결: (double) 캐스팅을 추가하여 소수점 연산이 수행되도록 수정함.

5. 반환 타입 및 인덱스 에러
- resultFees[1]과 같이 2차원 배열의 특정 행을 반환하려다 ArrayIndexOutOfBoundsException 발생.
- 해결: 문제 요구사항에 맞춰 요금만 담긴 1차원 int[] 배열을 새로 생성하여 반환함.

### 배운 점 && 보완할 점
1. Math.ceil() 사용 시 피연산자 중 하나는 반드시 실수가 되어야 의도한 '올림' 결과가 나옴을 재확인함.
2. 복잡한 substring보다는 split이 에러를 줄이는 데 유리할 수 있음.
3. fees[]를 분리할 때, `StringTokenizer` 를 썼어도 됐을 것 같다.
4. 차량번호를 미리 ArrayList<>에 담아 정렬한 후 해당 순서에 맞게 int[]을 만들었어도 됐을 것 같다.
