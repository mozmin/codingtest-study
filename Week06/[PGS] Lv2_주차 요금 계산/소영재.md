### 문제 설명 (요약)
- 주차장의 요금표와 차량이 들어오고(입차) 나간(출차)기록이 주어졌을 때, 차량별로 주차 요금을 계산하는 문제
- 누적 주차 시간이 기본 시간 이하라면, 기본 요금을 청구
- 초과한 시간이 단위 시간으로 나눠지지 않을 경우, 올림 처리를 하여 계산
- 최종적으로 차량 번호가 작은 순서대로 자동차 별 계산된 주차 요금을 반환

### 입력
- int[] fees : 기본시간, 기본요금, 단위 시간, 단위 요금이 들어있는 1차원 정수 배열
- String[] records : 시각, 차량번호, 내역(입차, 출차)가 담겨있는 1차원 문자열 배열

### 출력
- 차량 번호가 작은 순서대로 자동차 별 계산된 주차요금
<hr>

### 핵심 풀이 전략
1. 입차 시간과 총 주차한 시간을 담는 HashMap 자료구조를 선언
2. 차량이 여러 번 입차와 출차를 할 수 있기 때문에 set 자료구조를 선언
3. 계산의 편의성을 위해 시각을 분으로 치환하여 저장
4. records[] 배열 안에서 입차인 경우와 출차인 경우를 분리하여 startParking(입차한 시간)과 sumParking(총 주차시간)에 저장
5. 출차를 계산하는 경우에는 기존에 차가 다시 들어온 경우도 존재하므로 반영하여 계산, 출차했으므로 startParking에서 입차한 기록을 제거
6. 출차를 하지 않은 차량의 경우에는 .keySet()메서드를 활용하여 입차기록이 남아있는 차량 번호를 처리 1439 -> 23:59분을 분으로 바꾼 값
7. 차량 번호가 작은 순서대로 정렬을 해야 하기 때문에 set -> List로 변환한 뒤 정렬한다.
8. 총 주차 요금을 계산하는 메서드를 호출한 뒤, 정답 배열에 넣고 반환한다.
---

```java
import java.util.*;
class Solution {
    //입차 시간을 담는 HashMap
    Map<String, Integer> startParking = new HashMap<>();
    //총 주차한 시간을 담는 HashMap
    Map<String, Integer> sumParking = new HashMap<>();
    //차량 번호를 담는 Set (중복 제거)
    Set<String> carNumbers = new HashSet<>();
    
    public int[] solution(int[] fees, String[] records) {
        for(int i = 0; i < records.length; i++){
            String[] str1 = records[i].split(" ");
            String[] str2 = str1[0].split(":");
            //계산의 편의성을 위해 시각을 분으로 치환하여 저장
            int recordTime = Integer.parseInt(str2[0]) * 60 + Integer.parseInt(str2[1]);
            
            carNumbers.add(str1[1]);
            //입차인 경우, 입차 시간을 저장
            if(str1[2].equals("IN")){
                startParking.put(str1[1], recordTime);
            //출차인 경우, 차가 다시 들어온 경우도 존재하므로 반영하여 계산
            } else{
                sumParking.put(str1[1],sumParking.getOrDefault(str1[1], 0) + (recordTime - startParking.get(str1[1])));
                //출차했으므로 입차 기록을 제거
                startParking.remove(str1[1]);
            }
        }
        //출차가 찍히지 않은 차량 처리
           for (String carNumber : startParking.keySet()) {
            int start = startParking.get(carNumber);
            sumParking.put(carNumber,sumParking.getOrDefault(carNumber, 0)+ (1439 - start)
            );
        }
        //차량 번호 정렬
        List<String> sortedCars = new ArrayList<>(carNumbers);
        Collections.sort(sortedCars);
        
        //차량의 숫자가 정해졌으므로 정답을 반환할 배열 설정
        int answer[] = new int[sortedCars.size()];
        
        //반복문을 돌면서 요금 계산
        for (int i = 0; i < sortedCars.size(); i++) {
            answer[i] = calResult(sortedCars.get(i), fees);
        }
        return answer;
    }
    public int calResult(String carNumber, int []fees){
         int sumTime = sumParking.getOrDefault(carNumber, 0);
        // 기본 시간 이하일 경우 계산
        if (sumTime <= fees[0]) {
            return fees[1];
        }
        // 초과 시간 계산
        int overTime = sumTime - fees[0];

        // 분단위 올림 처리
        int unitTime = (int) Math.ceil((double) overTime / fees[2]);

        return fees[1] + unitTime * fees[3];
    }
}
```

### 배운 점 && 보완할 점
- 이전 코테 문제에서 습득한 getOrDefault()를 적용했다. 뿌듯하다.
- 차량이 여러 번 입차와 출차를 할 수 있다는 점에서 set 자료구조를 사용했는데, 문제 마지막에 차량번호가 작은 자동차부터 주차 요금을 정렬해서 출력해야 하는 부분을 놓쳤었다.
- List<String> sortedCars = new ArrayList<>(carNumbers); Collections.sort(sortedCars); 과 같은 형식으로 set자료구조를 List로 바꿔서 중복된 데이터를 저장하지 않으면서도 정렬을 할 수 있다는 방법을 배우게 되었다.
- .keySet()이라는 메서드도 이번에 배우게 되었다. .keySet()은 Map에 들어있는 모든 key값을 Set으로 반환한다.
