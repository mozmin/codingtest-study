### 문제 설명
현재 피로도 k와 여러 개의 던전 정보가 주어진다.<br>
각 던전은 최소 필요 피로도, 소모 피로도 두 가지 값을 가진다. <br>
현재 피로도가 최소 필요 피로도 이상일 때만 던전에 입장할 수 있으며, <br>
던전에 들어가면 피로도가 소모된다.<br>
가능한 던전 방문 순서를 고려하여 최대 몇 개의 던전을 탐험할 수 있는지 구하는 문제이다.<br>

### 입력
- k : 현재 피로도
- dungeons : [최소 필요 피로도, 소모 피로도] 형태의 2차원 배열

### 출력
- 탐험할 수 있는 최대 던전 개수

### 핵심 풀이 전략
이 문제는 던전 방문 순서에 따라 결과가 달라지는 문제이므로 <br>
모든 방문 순서를 탐색하는 DFS + 백트래킹 방식으로 해결할 수 있다. <br>

1. DFS로 던전 방문 순서 탐색
현재 피로도로 갈 수 있는 던전을 선택하고 다음 단계로 탐색한다. <br>
현재 피로도 → 갈 수 있는 던전 선택 → 피로도 감소 → 다음 던전 탐색 <br>

2. 방문 여부 관리
한 번 방문한 던전은 다시 방문할 수 없으므로 visited 배열을 사용한다.<br>
```
visited[i] = true   // 던전 방문
dfs(...)
visited[i] = false  // 백트래킹
```

3. 최대 탐험 개수 갱신
DFS를 진행하면서 현재까지 탐험한 던전 개수를 계속 기록하고 가장 큰 값을 answer로 갱신한다. <br>

### 제출 코드
```java
import java.util.*;

class Solution {
    
    int answer = 0;
    boolean[] visited;
    
    public int solution(int k, int[][] dungeons) {
        visited = new boolean[dungeons.length];
        dfs(k, dungeons, 0);
        return answer;
    }
    
    public void dfs(int k, int[][] dungeons, int count){
        answer = Math.max(answer, count);
        
        for(int i = 0; i < dungeons.length; i++){
            if(!visited[i] && k >= dungeons[i][0]){
                visited[i] = true;
                dfs(k-dungeons[i][1], dungeons, count+1);
                visited[i] = false;
            }
        }
    }
}
```
