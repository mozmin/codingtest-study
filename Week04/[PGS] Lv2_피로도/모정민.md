### 문제 설명 (요약)
유저가 가진 초기 피로도를 바탕으로 최대한 많은 던전을 탐험하는 문제.
각 던전에는 입장을 위해 충족해야 하는 '최소 필요 피로도'와, 탐험 후 차감되는 '소모 피로도'가 존재함.
어떤 순서로 던전을 방문하느냐에 따라 탐험할 수 있는 총 던전 수가 달라지므로, 가능한 모든 던전 방문 순서를 확인하여 최댓값 반환

### 입력
- k: 유저의 현재 남은 피로도 (1 이상 5,000 이하의 자연수)

- dungeons: 각 던전의 정보가 담긴 2차원 정수 배열 (세로 길이 1 이상 8 이하)

  - 배열의 각 행은 [최소 필요 피로도, 소모 피로도] 

  - '최소 필요 피로도'는 항상 '소모 피로도'보다 크거나 같다.

### 출력
- 유저가 탐험할 수 있는 최대 던전 수 (정수 반환)

<hr>

### 핵심 풀이 전략
- DFS(깊이 우선 탐색)를 활용한 완전탐색(백트래킹)
- 던전 크기가 8개로 적으므로, 완전탐색으로 풀이 가능
- dfs() 메서드 구현. visited 배열을 활용하여 이미 방문한 지점은 중복으로 방문하지 않도록 체크
- dfs() 메서드 안에 for문으로 현재 피로도 k가 다음 방문할 던전의 최소 필요도 이상인지 검사
  - 필요도 이상이면 방문 체크와 피로도 차감 후 dfs()로 재귀 호출, 호출 종료 이후 다른 경로의 탐색을 위해 방문 체크 해제(백트래킹)
---

```java
class Solution {
    
    static int max;
    public int solution(int k, int[][] dungeons) {
        
        max = 0;
        boolean[] visited = new boolean[dungeons.length];
        dfs(k, 0, visited, dungeons);
        
        return max;
    }
    
    public static void dfs(int k, int count, boolean[] visited, int[][] dungeons){
        
        max = Math.max(max, count);
        
        for(int i = 0; i < dungeons.length; i++){
            if(visited[i]) continue;
            
            if(k >= dungeons[i][0] && k >= dungeons[i][1]){
                visited[i] = true;
                dfs(k - dungeons[i][1], count+1, visited, dungeons);
                visited[i] = false;
            }
        }
    }
}
```

### 삽질 기록 🧠
1. max 갱신 시점의 누락: 처음에는 dfs()에서 max 부분을 검사할 때 if문을 써서 조건 처리를 하도록 했는데, 그러다보니 피로도가 남았음에도 던전을 돌 수 없는 경우에는 max를 검사할 수 없었다.
2. static 변수의 상태 유지 문제: static int를 선언과 동시에 초기화하여 이전 테스트 케이스의 정답이 다음 테스트 케이스에 영향을 주어 오답이 발생하게 됨.

### 배운 점 && 보완할 점
1. 개선 방향 (전역 변수 의존성 줄이기): static 변수 자체를 사용하지 않고, dfs 메서드가 직접 최댓값을 반환하도록 구조를 변경하면 부작용 걱정 없는 훨씬 견고하고 깔끔한 Java 코드를 설계할 수 있을 것 같다.
```java
public int solution(int k, int[][] dungeons) {
        int n = dungeons.length;
        boolean[] visited = new boolean[n];
        
        // dfs 자체가 최댓값을 반환하도록 호출
        return dfs(k, 0, visited, dungeons);
    }
```
