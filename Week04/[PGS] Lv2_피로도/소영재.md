### 문제 설명 (요약)
- 일정 피로도를 사용해서 던전을 탐험한다.
- 이때, 각 던전마다 탐험을 시작하기 위해 필요한 최소 필요 피로도와 던전 탐험을 마쳤을 때 소모되는 소모 피로도가 있다.
- 유저가 피로도를 소모하면서 던전들을 최대한 많이 탐험할 수 있는 횟수를 구하는 문제.

### 입력
- int k : 현재 피로도
- int[][] dungeons : 던전의 최소 필요 피로도와 소모 피로도가 담긴 2차원 정수 배열

### 출력
- 유저가 탐험할 수 있는 최대 던전 수를 리턴
<hr>

### 핵심 풀이 전략
- 유저가 탐험할 수 있는 최대 던전 수를 알아야 하기 때문에 완전탐색 기법 중 DFS를 활용하여 접근한다.
- 방문했던 곳을 다시 방문하게 될 수 있으므로 방문 여부를 표시하는 boolean[] 배열을 선언한다. (visited)
- dfs를 재귀적으로 호출하면서 가능한 모든 경우를 탐색한다.
- dfs 메서드가 최종적으로 종료되면 최대값을 반환한다.
---

```java
import java.util.*;
class Solution {
    int answer = 0;
    public int solution(int k, int[][] dungeons) {
        boolean[] visited = new boolean[dungeons.length];
        dfs(k, dungeons, visited, 0);
        return answer;
    }
    
    public void dfs(int k, int[][] dungeons, boolean[] visited, int sum){
        
        answer = Math.max(answer, sum);
        
        for(int i = 0; i < dungeons.length; i++){
            //방문했다면 건너뛰기
           if(visited[i] == true){
               continue;
           }
            //방문하지 않았고, 현재 피로도가 들어갈 수 던전의 피로도보다 크면서 소모될 피로도보다 크다면 입장
           if(k >= dungeons[i][1] && k >= dungeons[i][0]){
               visited[i] = true;
               dfs(k - dungeons[i][1], dungeons, visited, sum +1);
               visited[i] = false;
           } 
        }
    }
}
```
