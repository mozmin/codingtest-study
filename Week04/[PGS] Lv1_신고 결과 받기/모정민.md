### 문제 설명 (요약)
불량 이용자를 신고하고 처리 결과를 메일로 통보받는 시스템을 구현하는 문제.

각 유저는 한 번에 한 명의 유저를 신고할 수 있다. (단, 한 유저가 같은 유저를 여러 번 신고하더라도 1회로 처리된다.)

k번 이상 신고를 받은 유저는 게시판 이용이 정지된다.

정지된 유저를 신고한 모든 사람에게는 정지 사실을 알리는 결과 메일이 발송된다.
목표는 각 유저가 최종적으로 받게 될 결과 메일의 개수를 구하는 것이다.

### 입력
- id_list: 전체 유저 목록이 담긴 문자열 배열 (길이 2 ~ 1,000)

- report: "신고자 피신고자" 형태로 띄어쓰기로 구분된 신고 기록이 담긴 문자열 배열 (길이 1 ~ 200,000)

- k: 정지 기준이 되는 누적 신고 횟수 (1 이상 200 이하의 자연수)

### 출력
- id_list에 담긴 유저 순서대로, 각 유저가 받은 결과 메일 수를 담은 정수 배열(int[])

<hr>

### 핵심 풀이 전략
1. 중복 신고 방지를 위해 Set<>으로 매 반복마다 검사
2. Map<>으로 신고당한 사람의 신고 횟수 카운팅
3. ArrayList<>로 신고 조건에 부합하는 사람을 add()
4. '신고한 사람' + " " + "신고 당한 사람" 으로 Set<>에 값이 있는지 검사 -> Set<>에 있다면 해당 사람에게 메일을 발송해야 함
5. 메일 발송 횟수만큼 int[]에 기록

---

```java
import java.util.*;

class Solution {
    public int[] solution(String[] id_list, String[] report, int k) {
        int[] answer = new int[id_list.length];
        
        // 중복 신고 방지를 위한 Set
        Set<String> set = new HashSet<>();
        
        // 신고를 기록하기 위한 Map
        Map<String, Integer> map = new HashMap<>();
        
        // 신고당한 사람들 리스트
        ArrayList<String> banned = new ArrayList<>();
        
        // 신고 기록
        for(String s : report){
            if(set.add(s)){
                String[] split = s.split(" ");
                String to = split[1];
                map.put(to, map.getOrDefault(to, 0) + 1);
            }
        }
        
        for(String i : id_list){
            if (map.containsKey(i) && map.get(i) >= k){
                banned.add(i);
            }
        }
        
        for(String b : banned){
            
            for(int i = 0; i < id_list.length; i++){
                if(set.contains(id_list[i] + " " + b)){
                    answer[i]++;
                }
            }
        }
        
        return answer;
    }
}
```

### 삽질 기록 🧠
이번에도 자바 문법을 잘 몰라서 시간을 오래 씀
1. containsKey() 메서드가 기억이 안나서 에러 발생
2. map.put(to, map.getOrDefault(to, 0) + 1); 에서 +1 를 빼먹어버림
