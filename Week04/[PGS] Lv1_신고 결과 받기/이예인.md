### 문제 설명
유저들이 서로를 신고한 기록이 주어질 때,<br>
한 유저가 동일한 유저를 여러 번 신고해도 1회만 인정되고<br>
특정 유저가 k번 이상 신고 당하면 정지되며<br>
정지된 유저를 신고했던 사람에게 메일(알림)이 발송된다.<br>
각 유저가 받는 메일 수를 id_list 순서대로 반환한다.<br>

### 입력
- id_list: 사용자 ID 목록
- report: "신고자 피신고자" 형태 문자열 배열
- k: 정지 기준 신고 횟수

### 출력
- id_list 순서대로 각 사용자가 받는 메일 수 배열

### 핵심 풀이 전략
1. 중복 신고 제거
- 같은 신고자→피신고자 쌍은 1번만 인정되므로 Set<String>으로 report 중복 제거

2. 피신고자별 신고 횟수 집계
- 중복 제거된 reportSet을 순회하며 to(피신고자)를 key로 카운트(Map)

3. 정지 대상 추출
- 신고 횟수가 k 이상인 유저들을 Set<String> banned에 저장

4. 메일 수 계산
- 다시 reportSet을 순회하며 to가 정지 대상이면 from(신고자)의 메일 수 +1
- 마지막에 id_list 순서에 맞게 answer를 구성

### 제출 코드
```java
import java.util.*;

class Solution {
    public int[] solution(String[] id_list, String[] report, int k) {

        // report의 중복 제거
        Set<String> reportSet = new HashSet<>();
        for(int i = 0 ; i < report.length; i++){
            reportSet.add(report[i]);
        }
        
        HashMap<String, Integer> map = new HashMap<>();
        for(String id : reportSet){
            String[] idReport = id.split(" ");
            // 신고 받은 사람 : + 1
            map.put(idReport[1], map.getOrDefault(idReport[1], 0) + 1);
        }
        
        // k번 이상 신고 받은 사람 
        Set<String> user = new HashSet<>();
        for(String key : map.keySet()){
            if(map.get(key) >= k){
                user.add(key);
            }
        }
        
        int[] answer = new int[id_list.length];
        
        for(String rep : reportSet){
            String[] idReport = rep.split(" ");
            String from = idReport[0];
            String to = idReport[1];
            
            // user(정지되는 회원)에게 신고한적이 있으면 ++
            if(user.contains(to)){
                for(int j = 0; j < id_list.length; j++){
                    if(!from.equals(id_list[j])){
                        continue;
                    }
                    answer[j]++;
                }
            }
        }
        
        return answer;
    }
}
```

### 삽길 기록 🧠
- Set에 put을 쓰려 함 → Set은 add(), 크기는 size()

- 중복 제거를 reportSet으로 해놓고, 메일 계산 단계에서 실수로 report(원본 배열)를 다시 순회해서 중복 신고가 다시 반영되어 오답
해결: 메일 카운트 단계도 반드시 reportSet으로 순회
