# [Lv1] 모의고사

[문제 링크](https://school.programmers.co.kr/learn/courses/30/lessons/42840) 

`#완전탐색`

### 제출 일자

2026년 2월 28일 2.74ms

### 문제 설명 (요약)
수포자 3명이 각자의 고정된 패턴으로 모의고사 문제를 찍는다. 
정답 배열이 주어질 때, 완전탐색을 통해 가장 많은 문제를 맞힌 사람을 구하는 문제

### 입력
1번 문제부터 마지막 문제까지의 정답이 순서대로 담긴 정수 배열 answers

### 출력
가장 많은 문제를 맞힌 수포자의 번호가 담긴 정수 배열 (최고점자가 여럿일 경우 오름차순 정렬)

<hr>

### 핵심 풀이 전략
1. 수포자 3명의 반복되는 찍기 패턴을 각각 배열(first, second, third)로 정의한다.
2. answers 배열을 순회하면서 수포자들의 패턴과 정답을 비교하여 점수를 누적한다.
3. 이때, 문제의 개수가 찍기 패턴의 길이보다 길 수 있으므로 **나머지 연산자(%)**를 활용해 각 패턴의 인덱스를 추출한다.
4. Math.max를 이용해 최고 점수를 구하고, 최고 점수와 일치하는 수포자를 리스트에 담은 뒤 배열로 변환하여 반환한다.


---

```java
import java.io.*;
import java.util.*;

class Solution {
    public int[] solution(int[] answers) {
        ArrayList<Integer> list = new ArrayList<>();
        
        int[] first = new int[]{1, 2, 3, 4, 5};
        int[] second = new int[]{2, 1, 2, 3, 2, 4, 2, 5};
        int[] third = new int[]{3, 3, 1, 1, 2, 2, 4, 4, 5, 5};
        
        int f = 0;
        int s = 0;
        int t = 0;
        
        int f_count = 0;
        int s_count = 0;
        int t_count = 0;
        for(int i = 0; i < answers.length; i++){
            
            f = i % first.length;
            s = i % second.length;
            t = i % third.length;
            
            if(answers[i] == first[f]) f_count++;
            if(answers[i] == second[s]) s_count++;
            if(answers[i] == third[t]) t_count++;
                
        }
        
        int max = Math.max(f_count, s_count);
        max = Math.max(max, t_count);
        
        if(max == f_count) list.add(1);
        if(max == s_count) list.add(2);
        if(max == t_count) list.add(3);
            
        return list.stream().mapToInt(i -> i).toArray();
    }
}
```

### 삽질 기록 🧠
- 변수 덮어쓰기 및 잘못된 배열 참조: 인덱스를 계산할 때 s와 t 변수를 써야 할 자리에 f 변수에 값을 계속 덮어쓰는 실수를 함.
  - 정답을 비교할 때도 second, third 배열과 비교해야 하는데 first 배열과만 비교함 (first[s], first[t]).

- 불필요하고 잘못된 조건문 사용: 인덱스가 배열 범위를 벗어날 때만 순환시키려고 if(i > 배열.length) 조건문을 씌움.
  - 이렇게 하면 i가 배열 길이보다 작거나 같을 때는 조건이 false가 되어 인덱스가 전혀 갱신되지 않고 초기값 0에 멈춰버리는 치명적인 논리 오류가 발생함. (경계값 계산 오류)

---

### 배운 점 && 보완할 점
- 배열 순환에는 나머지 연산자(%)가 무조건 정답이다: 반복되는 배열의 인덱스를 구할 때 굳이 if문으로 길이를 넘었는지 체크할 필요가 전혀 없다. i % 배열.length를 사용하면 인덱스가 배열 길이를 초과하든 안 하든 알아서 0부터 다시 순환하므로 훨씬 간결하고 정확하게 동작한다.

- 불필요한 변수 줄이기: f, s, t 같은 인덱스 저장용 변수를 따로 선언할 필요 없이, if(answers[i] == first[i % first.length]) 처럼 비교문 안에서 바로 계산하면 코드를 한결 더 깔끔하게 작성할 수 있다.

- Stream API 활용: ArrayList<Integer>에 담긴 결과를 기본형 int[] 배열로 변환할 때 list.stream().mapToInt(i -> i).toArray()를 사용하는 패턴은 확실히 외워두자.
