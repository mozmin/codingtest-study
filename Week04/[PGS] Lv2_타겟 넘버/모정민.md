`#DFS`

### 제출 일자

### 문제 설명 (요약)
n개의 음이 아닌 정수가 주어질 때, 
이 정수들의 순서를 바꾸지 않고 각각을 더하거나 빼서 목표하는 숫자(target)를 만드는 방법의 수를 구하는 문제. 

### 입력
- numbers: 사용할 수 있는 숫자가 담긴 1차원 정수 배열 (길이는 2 이상 20 이하, 각 숫자는 1 이상 50 이하의 자연수)

- target: 만들어야 하는 타겟 넘버 (1 이상 1,000 이하의 자연수)

### 출력
- 주어진 숫자들을 적절히 더하거나 빼서 target과 같아지는 경우의 수

<hr>

### 핵심 풀이 전략
1. 기본적인 DFS로 풀이
2. DFS static 메서드를 만들고, if문으로 target 조건과 dfs 종료조건을 검사
3. 양수와 음수를 파라미터인 sum에 더해서 dfs() 재귀호출
---

```java
import java.io.*;
import java.util.*;

class Solution {
    
    static int count;
    public int solution(int[] numbers, int target) {
        
        count = 0;
        
        dfs(1, numbers[0], numbers, target);
        dfs(1, -numbers[0], numbers, target);
        
        return count;
    }
    
    public static void dfs(int index, int sum, int[] numbers, int target){
        
        if(index > numbers.length-1 && sum == target) count++;
        
        if(index > numbers.length-1) return;

        dfs(index+1, sum+numbers[index], numbers, target);
        dfs(index+1, sum-numbers[index], numbers, target);
        
    } 
}
```

### 삽질 기록 🧠
1. if(index > numbers.length-1) return; 종료 조건을 처음에 설정하지 않아서 count 값이 큰 숫자가 나왔음.
2. 처음에 무의식적으로 for문을 사용했는데, 이 문제에서는 분기점이 각 인덱스마다 2번 밖에 없기 때문에 for문을 쓸 필요가 없음.
