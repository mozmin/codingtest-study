### 문제 설명
- 주어진 numbers 배열의 각 원소에 + 또는 -를 붙여 더했을 때, 최종 합이 target이 되는 경우의 수를 구한다.

### 입력
- numbers: 사용할 숫자 배열
- target: 만들어야 하는 목표 합

### 출력
- target을 만들 수 있는 경우의 수

### 핵심 풀이 전략
- DFS(재귀)로 “각 숫자에 +를 붙이는 경우 / -를 붙이는 경우”를 모두 탐색한다.
- 재귀 함수 상태는 다음 두 가지가 핵심이다.
- depth: 현재 몇 번째 숫자까지 처리했는지
- sum: 현재까지 누적 합
- 종료 조건: depth == numbers.length일 때 누적 합이 target이면 answer++

### 제출 코드
```java
class Solution {
    int answer = 0;
    public int solution(int[] numbers, int target) {
        
        dfs(numbers, 0, 0, target);
        return answer;
    }
    
    public void dfs(int[] numbers, int depth, int sum, int target){
        if(depth == numbers.length){
            if(sum == target) answer++;
            return;
        }
        
        dfs(numbers, depth+1, sum+numbers[depth], target);
        dfs(numbers, depth+1, sum-numbers[depth], target);
        
    }
}
```

### 배운 점 && 보완할 점
dfs 문제를 풀 때 항상 파라미터를 정하는게 어렵다. 
공식처럼 정리를 하자면,
1. 고정 정보 (안 바뀌는 것)
 - 이 문제에서는 numbers, target  
2. 상태 정보 (계속 바뀌는 것)
 - 이 문제에서는 depth, sum
<br>
dfs 문제를 풀 때 스스로에게 이렇게 물어보면 편한거 같다
1. 지금 단계에서 무엇이 바뀌는가?
2. 다음 단계로 가려면 어떤 정보가 필요한가?

- 현재 상태 찾기
- 종료 조건 생각하기
- 템플릿 기억하기
- 선택지를 찾기
이 템플릿을 기억해두면 좋을거 같다
```
void dfs(상태){

    if(종료조건){
        정답처리
        return;
    }

    for(가능한 선택){
        dfs(다음상태)
    }
}
```
