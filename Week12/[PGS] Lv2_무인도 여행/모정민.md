### 문제 설명 (요약)
격자 형태의 지도에서 바다('X')와 무인도의 식량(1~9의 숫자) 정보가 주어질 때, 각 무인도에서 며칠 동안 머물 수 있는지 구하는 문제.

- **maps**: 지도 정보를 담은 문자열 배열 (가로, 세로 길이 최대 100)
- **조건**: 
  - 상, 하, 좌, 우로 연결된 땅(숫자)은 하나의 무인도로 간주.
  - 각 무인도에 적힌 숫자의 합이 해당 무인도에서 머물 수 있는 최대 일수(식량의 합).
  - 지도를 탐색해 각 무인도의 식량 합을 구한 뒤, **오름차순으로 정렬**하여 배열로 반환.
  - 지도를 모두 탐색했는데 지낼 수 있는 무인도가 없다면 `[-1]`을 반환.

---

### 핵심 풀이 전략
1. **지도 전처리**: 다루기 까다로운 `String[]` 형태의 지도를 `int[][] world` 2차원 배열로 변환한다. 이때 바다('X')는 탐색하지 않도록 `0`으로, 땅은 아스키코드를 연산(`c - '0'`)하여 실제 정수로 매핑한다.
2. **완전 탐색과 BFS**: 2차원 배열을 이중 `for`문으로 순회하며, 값이 `0`이 아닌 곳(아직 방문하지 않은 무인도)을 발견할 때마다 BFS 탐색을 시작한다.
3. **방문 처리 최적화**: 별도의 `boolean[][] visited` 방문 배열을 만들지 않고, 큐에 좌표를 넣음과 동시에 `world[nextY][nextX] = 0`으로 값을 덮어씌워 **재방문을 방지**하고 공간 복잡도를 절약한다.
4. **결과 집계**: BFS 탐색이 끝날 때마다 반환된 식량의 합을 `ArrayList`에 저장한다.
5. **정렬 및 반환**: 모든 탐색이 끝난 후 리스트가 비어있으면 `[-1]`을 반환하고, 값이 있다면 오름차순 정렬 후 `int[]` 원시 타입 배열로 변환하여 반환한다.

### 내가 구현한 구현 코드
```java
import java.util.*;

class Solution {
    
    int[] dy = {0, 0, 1, -1};
    int[] dx = {1, -1, 0, 0};
    public int[] solution(String[] maps) {
        
        int m = maps.length;
        int n = maps[0].length();
        
        // 배열 초기화
        int[][] world = new int[m][n];
        
        for(int i = 0; i < m; i++){
            for(int j = 0; j < n; j++){
                char c = maps[i].charAt(j);
                
                if(c == 'X') world[i][j] = 0;
                else world[i][j] = c - '0';
            }
        }
        
        // 정답 배열
        ArrayList<Integer> result = new ArrayList<>();
        
        // 탐색하면서 섬 확인
        for(int i = 0; i < m; i++){
            for(int j = 0; j < n; j++){
                
                // 0이 아니라면 (방문한 곳은 0처리)
                if(world[i][j] != 0){
                    int value = bfs(i, j, world[i][j], world, m, n);
                    result.add(value);
                }
            }
        }
        
        if(result.isEmpty()) return new int[]{-1};
        else{
            int[] answer = result.stream().sorted().toArray(int[]::new);

            return answer;
        }
    }
    
    public int bfs(int y, int x, int value, int[][] world, int m, int n){
        
        Deque<int[]> que = new ArrayDeque<>();
        que.offer(new int[]{y, x, value});
        world[y][x] = 0;
        
        int totalValue = 0;
        
        while(!que.isEmpty()){
            int[] poll = que.poll();
            int curY = poll[0];
            int curX = poll[1];
            int curValue = poll[2];
            
            totalValue += curValue;
            
            for(int i = 0; i < 4; i++){
                int nextY = curY + dy[i];
                int nextX = curX + dx[i];
                
                if(nextY >= 0 && nextY < m && nextX >= 0 && nextX < n && world[nextY][nextX] != 0){
                    que.offer(new int[]{nextY, nextX, world[nextY][nextX]});
                    world[nextY][nextX] = 0;
                }
            }
        }
        
        return totalValue;
    }
}
```

### 정답 코드 (타입 안전성 및 불필요한 매개변수 제거)
자바에서는 객체 스트림(`Stream<Integer>`)을 `int[]`로 바꿀 때 직접 `.toArray(int[]::new)`를 호출할 수 없으므로 `mapToInt()`를 거쳐야 런타임 에러나 컴파일 에러를 방지할 수 있다. 또한 큐에 `value`까지 굳이 담지 않아도, `world` 배열의 값을 통해 바로 `totalValue`에 누적시킬 수 있다.
```java
import java.util.*;

class Solution {
    int[] dy = {0, 0, 1, -1};
    int[] dx = {1, -1, 0, 0};
    
    public int[] solution(String[] maps) {
        int m = maps.length;
        int n = maps[0].length();
        int[][] world = new int[m][n];
        
        for(int i = 0; i < m; i++){
            for(int j = 0; j < n; j++){
                char c = maps[i].charAt(j);
                world[i][j] = (c == 'X') ? 0 : c - '0';
            }
        }
        
        List<Integer> result = new ArrayList<>();
        
        for(int i = 0; i < m; i++){
            for(int j = 0; j < n; j++){
                if(world[i][j] != 0){
                    result.add(bfs(i, j, world, m, n));
                }
            }
        }
        
        if(result.isEmpty()) return new int[]{-1};
        
        // List<Integer> -> int[] 변환 시 mapToInt 활용
        return result.stream()
                     .mapToInt(Integer::intValue)
                     .sorted()
                     .toArray();
    }
    
    public int bfs(int y, int x, int[][] world, int m, int n){
        Deque<int[]> que = new ArrayDeque<>();
        que.offer(new int[]{y, x});
        
        int totalValue = world[y][x];
        world[y][x] = 0; // 시작점 방문 처리
        
        while(!que.isEmpty()){
            int[] poll = que.poll();
            int curY = poll[0];
            int curX = poll[1];
            
            for(int i = 0; i < 4; i++){
                int nextY = curY + dy[i];
                int nextX = curX + dx[i];
                
                if(nextY >= 0 && nextY < m && nextX >= 0 && nextX < n && world[nextY][nextX] != 0){
                    que.offer(new int[]{nextY, nextX});
                    totalValue += world[nextY][nextX]; // 큐에 넣을 때 값을 미리 더함
                    world[nextY][nextX] = 0; // 큐에 넣음과 동시에 방문 처리
                }
            }
        }
        
        return totalValue;
    }
}
```

### 배운 점 && 보완할 점
- `ArrayList<Integer>`를 배열로 바꾸는 과정에서 `result.stream().sorted().toArray(int[]::new)`를 사용했다. 자바 문법상 객체 타입인 `Stream<Integer>`은 바로 원시 타입인 `int[]`로 치환될 수 없어 에러가 발생하기 쉽다. 기본형 스트림으로 변환해주는 중간 연산자 `mapToInt()`를 거쳐야 안전하다는 것을 잊지 말자.
- **보완할 점 (객체 지향적 최적화)**: 큐 안에 `new int[]{y, x, value}` 처럼 3개의 요소를 넣으면 탐색 범위가 커질수록 배열 객체 생성 비용이 늘어난다. 리팩토링한 정답 코드처럼 큐에는 최소한의 좌표 `{y, x}`만 넣고 값은 `totalValue`에 직접 합산해주는 방식으로 최적화하는 습관을 기르면 대규모 데이터에서도 강건한 코드를 작성할 수 있다.
