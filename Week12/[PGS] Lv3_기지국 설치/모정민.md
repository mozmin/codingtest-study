### 문제 설명 (요약)
N개의 아파트에 전파를 전달하기 위해 기지국을 설치하려고 한다. 이미 설치된 기지국의 위치가 주어질 때, 모든 아파트에 전파를 전달하기 위해 증설해야 할 기지국 개수의 최솟값을 구하는 문제.

- **n**: 아파트의 총 개수 (1 이상 200,000,000 이하의 자연수)
- **stations**: 현재 기지국이 설치된 아파트의 번호가 담긴 오름차순 배열 (크기 10,000 이하)
- **w**: 기지국의 전파 도달 거리 (1 이상 10,000 이하)
- **조건**: 전파는 기지국이 설치된 아파트를 기준으로 좌우 `w`만큼 전달된다. 즉, 커버 범위는 `2 * w + 1`이다.

---

### 핵심 풀이 전략
1. **O(N) 탐색 지양**: 아파트 개수 `n`이 최대 2억 개이므로, 아파트 전체를 일일이 순회하면 시간 초과(TLE)가 발생한다. 기지국 배열 `stations`(최대 1만 개)을 순회하며 빈 공간을 찾는 방식으로 접근해야 한다.
2. **전파가 닿지 않는 구간(Gap) 계산**: 현재 설치된 기지국들의 사이, 그리고 양 끝(1번 아파트 앞, n번 아파트 뒤)에 전파가 닿지 않는 사각지대의 길이를 구한다.
3. **최소 기지국 설치**: 구한 사각지대의 길이를 하나의 기지국이 커버할 수 있는 최대 범위(`2 * w + 1`)로 나눈다.
4. **나머지 처리**: 나누어 떨어지지 않고 나머지가 존재한다면, 해당 자투리 공간을 커버하기 위해 기지국이 1개 더 필요하므로 몫에 +1을 해준다.
---

### 내가 구현한 구현 코드
```java
import java.util.*;

class Solution {
    public int solution(int n, int[] stations, int w) {
        int answer = 0;
        
        // 기지국 하나가 커버하는 범위: 2 * w + 1
        int range = 0;
        int left = 0;
        int right = 0;
        for(int s : stations){
            
            right = s - w;
            
            range = right - left - 1;
            if(range >= 0) answer += cal(range, w);
            
            left = s + w;
        }
        
        if(left < n){
            range = n - left;
            answer += cal(range, w);
        }

        return answer;
    }
    
    public int cal(int range, int w){
        
        int value = range / (2*w+1);
        int rest = range % (2*w+1);
        
        if(rest > 0) value += 1;
        
        return value;
    }
}
```

### 정답 코드 (수학적 연산 및 가독성 최적화)
기존 코드도 O(M)의 시간 복잡도(M은 stations의 길이)를 가지는 매우 훌륭한 최적화 코드이다. 다만 별도의 `cal` 메서드 없이 수학의 **올림(Ceil)** 개념이나 정수 나눗셈 기법을 활용하면 코드를 훨씬 간결하게 만들 수 있다.
```java
class Solution {
    public int solution(int n, int[] stations, int w) {
        int answer = 0;
        
        int coverage = 2 * w + 1; // 기지국 하나가 커버하는 최대 범위
        int left = 1;             // 탐색을 시작할 아파트 번호 (1번부터 시작)
        
        for (int s : stations) {
            int right = s - w;    // 현재 기지국의 전파가 닿는 제일 왼쪽 아파트
            
            // 전파가 닿지 않는 구간(Gap)이 존재할 경우
            if (left < right) {
                int range = right - left;
                // 정수 나눗셈 올림 처리 기법: (range + coverage - 1) / coverage
                answer += (range + coverage - 1) / coverage;
            }
            
            // 다음 탐색 시작점 갱신 (현재 기지국 전파의 제일 오른쪽 다음 아파트)
            left = s + w + 1;
        }
        
        // 마지막 기지국 이후부터 n번 아파트까지 사각지대가 남은 경우
        if (left <= n) {
            int range = n - left + 1;
            answer += (range + coverage - 1) / coverage;
        }

        return answer;
    }
}
```

### 삽질 기록 🧠
- 처음에는 전파가 안닿는 모든 구간의 수를 구한 수에서 (2*w+1)를 나누려고 했다.

### 배운 점 && 보완할 점
- 올림을 위한 전용 라이브러리가 있었다. `Math.ceil()` 잊지말자. 그 외에도 반올림을 위한 `Math.round()`도 있다.
- **보완할 점 (정수 나눗셈 올림 기법)**: `a`를 `b`로 나눌 때 나머지가 있으면 몫을 올림 처리하는 로직은 `(a + b - 1) / b` 공식으로 해결하면 코드가 매우 깔끔해진다. `Math.ceil((double) a / b)`를 써도 되지만, 부동소수점 형변환 비용을 피할 수 있어 실무나 코딩 테스트에서 자주 쓰이는 정수 나눗셈 식이니 꼭 외워두자.
