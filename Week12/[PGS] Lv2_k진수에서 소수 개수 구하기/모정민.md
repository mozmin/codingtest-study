### 문제 설명 (요약)
양의 정수 `n`을 `k`진수로 변환했을 때, 변환된 문자열 안에서 조건에 맞는 소수(Prime number)의 개수를 구하는 문제.

- **n**: 진수 변환을 할 양의 정수 (1 이상 1,000,000 이하)
- **k**: 변환할 진수 (3 이상 10 이하)
- **조건**: 
  - `0P0`, `P0`, `0P`, `P` 형태로 조건에 맞는 소수 `P`를 찾아야 함.
  - 즉, 변환된 진수에서 `0`을 파티션(구분자)으로 삼아 나누어진 숫자들이 소수인지 판별하는 문제.
  - `P`는 0을 포함하지 않는 소수여야 하며, `1`은 소수가 아님.

---

### 핵심 풀이 전략
1. **k진수 변환 (수동 구현)**: 주어진 정수 `n`을 `k`로 계속 나누며 나머지를 스택(`Deque`)에 `push` 한다. LIFO 구조를 활용해 나중에 들어온 값부터 `pop` 하여 `StringBuilder`로 이어 붙이면 올바른 순서의 k진수 문자열이 완성된다.
2. 완성된 k진수 문자열을 `split("0")` 메서드를 사용해 `0`을 기준으로 분리한다.
3. 분리된 문자열 배열을 순회할 때, 연속된 `0`으로 인해 생성된 빈 문자열(`""`)은 예외 처리하여 건너뛴다(`continue`).
4. **자료형 고려**: 분리된 숫자는 `int`형의 범위를 거뜬히 초과할 수 있으므로, 반드시 `Long.parseLong()`을 사용하여 `Long` 타입으로 파싱한다.
5. **소수 판별 (시간 복잡도 최적화)**: $O(N)$으로 순회하면 시간 초과가 발생하므로, `Math.sqrt(value)`를 활용해 제급근까지만 약수를 검사하는 $O(\sqrt{N})$ 방식으로 소수 여부를 판별한다.
---

### 내가 구현한 구현 코드
```java
import java.util.*;

class Solution {
    public int solution(int n, int k) {
        
        int answer = 0;
        
        // k진수로 변환하기
        Deque<String> que = new ArrayDeque<>();
        StringBuilder sb = new StringBuilder();
        
        while(n > 0){
            que.push(String.valueOf(n % k));
            n = n / k;
        }
        
        while(!que.isEmpty()) sb.append(que.pop());
        String kValue = sb.toString();
        String[] kList = kValue.split("0");
        
        for(String kListValue : kList){
            
            if(kListValue.equals("")) continue;
            
            Long kValueLong = Long.parseLong(kListValue);
            
            if(isPrime(kValueLong)) answer++;
        }
            
        return answer;
    }
    
    // 2부터 ~ sqrt(value) 까지 0으로 나누어지는지 검사
    // 나누어진다면 return false
    // 안나누어지면 return true
    public boolean isPrime(Long value){
        
        if(value == 1 || value == 0) return false;
        
        for(int i = 2; i <= Math.sqrt(value); i++){
            if(value % i == 0) return false;
        }
        
        return true;
    }
}
```

### 정답 코드 (내장 함수 활용 최적화)
내장 함수인 `Integer.toString(n, k)`를 사용하면 진수 변환을 위한 스택 구조 및 반복문을 한 줄로 단축시킬 수 있다.
```java
import java.util.*;

class Solution {
    public int solution(int n, int k) {
        int answer = 0;
        
        // 자바 내장 함수를 활용한 k진수 변환 로직 단축
        String kValue = Integer.toString(n, k);
        String[] kList = kValue.split("0");
        
        for(String kListValue : kList){
            if(kListValue.isEmpty()) continue;
            
            long kValueLong = Long.parseLong(kListValue);
            
            if(isPrime(kValueLong)) answer++;
        }
            
        return answer;
    }
    
    public boolean isPrime(long value){
        if(value <= 1) return false;
        
        for(long i = 2; i <= Math.sqrt(value); i++){
            if(value % i == 0) return false;
        }
        
        return true;
    }
}
```

### 삽질 기록 🧠
- '0'으로 나누면 굳이 0에 대한 조건 검사를 할 필요가 없다. 그 생각을 못하고 1칸~n/2칸 마다 탐색하려고 삼중 for문을 썼다.
- 진수 변환 후 `0`을 기준으로 나눈 값들이 생각보다 엄청 커질 수 있다는 것을 놓치기 쉽다. 습관적으로 `Integer.parseInt()`를 사용했다면 런타임 에러(`NumberFormatException`)를 마주했을 것이다. 이번에는 `Long.parseLong()`을 적절히 활용하여 에러를 피할 수 있었다.

### 배운 점 && 보완할 점
- **보완할 점 (내장 함수 숙지)**: 자바의 `Integer.toString(int i, int radix)` 메서드를 알면 복잡한 진수 변환 로직을 단 한 줄로 줄일 수 있다. 코딩 테스트에서 시간 단축을 위해 이와 같은 언어의 내장 함수를 적극적으로 활용하는 습관을 들이자.
- 소수를 판별할 때는 Math.sqrt() 까지만 탐색하면 된다. a x b 에서 a = b를 기준으로 쌍으로 나타나기 때문이다.
- 소수 판별 로직에서 루프 인덱스 `i`의 타입을 `int`로 선언했는데, 판별해야 할 수 `value`가 Long의 최대치에 가까운 극한의 케이스라면 오버플로우가 날 수도 있다. 파라미터가 `Long`이라면 `for(long i = 2; ...)` 형태로 루프 변수도 함께 맞춰주는 것이 더 안전한 코딩 습관이다.
