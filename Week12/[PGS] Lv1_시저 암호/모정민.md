### 문제 설명 (요약)
어떤 문장의 각 알파벳을 일정한 거리만큼 밀어서 다른 알파벳으로 바꾸는 '시저 암호'를 구현하는 문제.

- **s**: 암호화할 문자열 (길이 8000 이하의 알파벳 소문자, 대문자, 공백으로 구성)
- **n**: 밀기 거리 (1 이상 25 이하의 자연수)
- **조건**: 
  - 공백은 아무리 밀어도 공백이다.
  - 대소문자를 구분하며, 'z'나 'Z'를 넘어가면 다시 'a'나 'A'로 돌아온다. (예: 'z'를 1만큼 밀면 'a'가 됨)

---

### 핵심 풀이 전략
1. `for`문을 이용해 문자열의 길이만큼 순회하며 `charAt(i)`로 문자를 하나씩 추출한다.
2. 추출한 문자가 대문자(`'A'` ~ `'Z'`)인지, 소문자(`'a'` ~ `'z'`)인지 조건문을 통해 판별한다.
3. 알파벳인 경우, 해당 문자에 밀기 거리 `n`을 직접 더해준다. (`c += n`)
4. `n`을 더한 결과값이 알파벳의 끝 문자(`'Z'` 또는 `'z'`)의 아스키코드 값을 넘어간다면, 초과한 만큼 다시 앞에서부터 시작하도록 알파벳의 총 개수인 `26`을 빼준다.
5. 대소문자 범위에 속하지 않는 문자(공백 등)는 아무 처리 없이 그대로 결과 문자열에 이어 붙인다.
6. 빈번한 문자열 추가 연산에 따른 메모리 낭비를 방지하기 위해 `StringBuilder`를 사용한다.
---

### 내가 구현한 구현 코드
```java
import java.util.*;

class Solution {
    public String solution(String s, int n) {
        
        StringBuilder sb = new StringBuilder();       
        
        for(int i = 0; i < s.length(); i++){
            char c = s.charAt(i);
            
            // 대문자
            if(c >= 'A' && c <= 'Z'){
                c += n;
                
                if(c > 'Z'){    
                    sb.append((char) (c-26));
                }else{
                    sb.append(c);
                }
            }
            else if(c >= 'a' && c <= 'z'){
                c += n;
                
                if(c > 'z'){
                    sb.append((char) (c-26));
                }else{
                    sb.append(c);
                }
            }else{
                sb.append(c);
            }
        }
        
        return sb.toString();
    }
}
```

### 정답 코드
1. 만약 n이 26 이상일 경우 `%26`를 활용하여 순환 처리 해야한다.
2. `c-'a'+n`을 하는 이유는 기준을 `26`에 맞추기 위해서이다.
```java
class Solution {
    public String solution(String s, int n) {
        StringBuilder sb = new StringBuilder();
        
        for (char ch : s.toCharArray()) {
            if (ch == ' ') {
                sb.append(ch);
            } else if (Character.isLowerCase(ch)) {
                // 소문자 순환 처리
                sb.append((char) ((ch - 'a' + n) % 26 + 'a'));
            } else if (Character.isUpperCase(ch)) {
                // 대문자 순환 처리
                sb.append((char) ((ch - 'A' + n) % 26 + 'A'));
            }
        }
        
        return sb.toString();
    }
}
```

### 삽질 기록 🧠
- sb.append() 안에 그대로 c- 26을 넣었다가 에러가 났다. sb.append() 안에 캐스팅 없이 그대로 계산을 하면 이를 (int)로 인식하여 결과 값으로 숫자나 나온다. 따라서 (char) (c-26)으로 캐스팅을 해야한다.

### 배운 점 && 보완할 점
- **보완할 점**: 현재 문제 조건에서는 `n`이 25 이하이기 때문에 `c + n`을 한 뒤 `- 26`을 한 번만 해주는 것으로 순환 처리가 가능하다. 만약 문제 조건이 바뀌어 `n`이 **무한히 큰 수까지 주어질 수 있다면**, 현재 문자를 상대적인 인덱스(0~25)로 치환한 뒤 나머지 연산(`% 26`)을 적용하는 방식을 고려해 볼 수 있겠다.
- 알파벳이나 특정 주기가 반복되는 순환 구조 문제를 만날 때는 반드시 **나머지 연산자(`%`)**를 가장 먼저 떠올리자.
