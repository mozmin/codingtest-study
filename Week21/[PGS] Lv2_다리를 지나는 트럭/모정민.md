# [Level 2] 다리를 지나는 트럭

### 문제 설명 (요약)
트럭들이 정해진 순서대로 1차선 다리를 건너려 합니다. 다리의 길이(`bridge_length`)와 다리가 견딜 수 있는 최대 무게(`weight`)가 주어질 때, 모든 트럭이 다리를 완전히 건너는 데 걸리는 최소 시간(초)을 구하는 문제입니다. 

---

### 핵심 풀이 전략
1. **다리를 큐(Queue)로 모델링**: `Deque`를 사용하여 다리를 건너고 있는 트럭들의 정보(`[트럭의 무게, 빠져나갈 절대 시간]`)를 관리합니다.
2. **이벤트 기반 시간 점프 (Time Skip)**: 무의미하게 1초씩(무게 0을 넣으며) 시간을 흘려보내는 대신, **다음 트럭이 올라갈 수 없을 때 맨 앞 트럭이 나가는 시간으로 단번에 점프**하여 불필요한 연산을 줄입니다.
3. **대기 트럭 기준의 시뮬레이션**: 특정 시각이 아니라 '대기 중인 트럭'을 기준으로 반복문을 돌며, 해당 트럭이 다리에 무사히 진입할 때까지 다리 위의 트럭을 빼고 시간을 갱신하는 `while(true)` 구조를 사용합니다.

---

### 내가 구현한 구현 코드 (정답 및 리팩토링 완료)
```java
import java.util.*;

class Solution {
    public int solution(int bridge_length, int weight, int[] truck_weights) {
        int answer = 0;
        
        Deque<int[]> q = new ArrayDeque<>();
        int time = 0;
        int curWeight = 0;
        
        for(int truck : truck_weights){
            
            while(true){
                
                if (q.size() == bridge_length || curWeight + truck > weight) {
                    int[] firstTruck = q.poll();
                    curWeight -= firstTruck[0];
                    
                    time = Math.max(time, firstTruck[1] - 1); 
                }
                else {
                    time++; 
                    curWeight += truck;
                    q.offer(new int[]{truck, time + bridge_length});
                    break;
                }
            }
            
        }
        
        return time + bridge_length;
    }
}
```

## 회고 및 삽질기록 🧠
- **다리 초기화(clear) 대참사**: 초반에는 다리가 무게를 견디지 못한다고 큐를 통째로 `clear()` 시켜버려 다리 위 모든 트럭을 바다에 빠뜨렸다. 무게 초과 시에는 맨 앞 트럭 **하나만** `poll()`로 빼고, 그 트럭의 무게만큼만 `curWeight`에서 덜어내야 한다는 점을 깨달았다.
- **`while(true)` 루프의 부재 (트럭 증발 현상)**: `for`문 안에서 단순 `if ~ else`로만 분기를 치면, 무게 초과로 앞 트럭을 뺐을 때 **현재 대기 중인 트럭은 다리에 오르지도 못한 채 다음 트럭으로 순서가 넘어가는** 문제가 발생했다. 대기 트럭이 완전히 다리에 올라갈 때(`break`)까지 시점을 묶어두는 루프가 필수적이다.
- **분기 통합과 시간 점프 셋업(-1)**:
  1. 다리가 꽉 찬 경우와 무게를 초과하는 경우는 결국 둘 다 **"당장 트럭이 못 올라가니 맨 앞 트럭이 나갈 때까지 시간을 기다린다"**는 동일한 본질을 가지므로 `||` 연산자를 통해 하나의 분기로 묶을 수 있었다.
  2. 시간을 건너뛸 때 `firstTruck[1]` 시각으로 바로 맞추지 않고 `-1`을 하는 이유는, 루프를 다시 돌아 트럭 진입 분기(2번 조건)를 탈 때 무조건 `time++;` 이 선행되기 때문이다. 이 타이밍을 딱 맞춰야 지각 없이 앞 트럭이 나감과 동시에 새 트럭을 진입시킬 수 있다.
