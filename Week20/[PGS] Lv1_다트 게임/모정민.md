### 문제 설명 (요약)
다트 게임은 총 3번의 기회로 구성되며, 각 기회마다 0~10점의 점수를 얻을 수 있습니다. 점수와 함께 주어지는 보너스(S, D, T)에 따라 점수를 1, 2, 3제곱합니다. 옵션으로 스타상(\*)은 이전 점수와 현재 점수를 각각 2배로 만들고, 아차상(#)은 현재 점수를 마이너스로 만듭니다. 주어진 문자열을 분석하여 총점수를 계산하는 문제입니다.

---

### 핵심 풀이 전략
1. **가변 길이 문자열 파싱 (10점 처리)**: 점수는 1자리(0~9)일 수도 있고 2자리(10)일 수도 있습니다. 문자열을 한 글자씩 읽거나 길이를 기반으로 자를 때 10점에 대한 예외 처리를 가장 우선적으로 고려해야 합니다.
2. **이전 턴의 데이터 참조 (스택 활용)**: 스타상(\*) 옵션은 '바로 직전에 얻은 점수'를 수정해야 하므로, 후입선출(LIFO) 특성을 가진 `Stack` 또는 `Deque`를 사용하여 각 턴의 최종 점수를 순서대로 누적해야 합니다.
3. **옵션 및 보너스 분리 기준**: 점수, 보너스(1글자 고정), 옵션(없거나 1글자)의 패턴을 가지므로, 보너스 문자(S/D/T)의 위치를 기준으로 그 뒤에 남은 문자가 있다면 옵션으로 취급하는 것이 안전합니다.

### 내가 구현한 구현 코드 (오류 포함)
```java
import java.util.*;

class Solution {
    public int solution(String dartResult) {
        int answer = 0;
        
        Deque<Integer> q = new ArrayDeque<>();
        
        String split = "";

        for(int i = 0; i < dartResult.length(); i++){
            
            char c = dartResult.charAt(i);
            
            // 숫자가 나오면 컷
            if(c - '0' >= 0 && c - '0' <= 9){
                
                // 만약 1이면 다음 숫자가 0인지 검사
                if(c - '0' == 1){
                    char nxt = dartResult.charAt(i+1);
                    
                    if(nxt == '0'){
                        if(!split.isEmpty()) cal(split, q);
                    
                        split = String.valueOf(c);
                        split += String.valueOf(nxt);
                        
                        ++i;
                    }else{
                        if(!split.isEmpty()) cal(split, q);
                    
                        split = String.valueOf(c);
                    }
                    

                }else{
                    // 계산
                    if(!split.isEmpty()) cal(split, q);
                
                    // split 초기화
                    split = String.valueOf(c);   
                }
            }
            
            // 숫자가 아니라면 그냥 +
            else{
                split += String.valueOf(c);
            }
        }
        
        // 마지막 남은 split 처리
        cal(split, q);
        
        // 점수 계산
        while(!q.isEmpty()){
            
            System.out.println(q.peek());
            answer += q.pop();
        }
        
        return answer;
    }
    
    private void cal(String split, Deque<Integer> q){
        
        int total = 0;
        int point = 0;
        char bonus =  ' ';
        char option = ' ';
        
        if(split.charAt(0) - '0' == 1 && split.charAt(1) - '0' == 0){
            point = 10;
            bonus = split.charAt(2);
            option = split.length() != 4 ? ' ' : split.charAt(3);
        }else{
            point = (int) (split.charAt(0) - '0');
            bonus = split.charAt(1);
            option = split.length() != 3 ? ' ' : split.charAt(2);
        }
            
        // 보너스 확인
        if(bonus == 'S') total = (int) Math.pow(point, 1);
        else if(bonus == 'D') total = (int) Math.pow(point, 2);
        else if(bonus == 'T') total = (int) Math.pow(point, 3);

        
        // 옵션 확인
        if(option != ' '){
            
            // 본인과 이전 값 x2
            if(option == '*'){
                
                if(!q.isEmpty()){
                    int prev = q.poll();
                    q.push(2 * prev);
                }
                
                total = 2 * total;
            }
            // 본인 점수 x-1
            else if(option == '#'){
                total = -1 * total;
            }
        }
        
        q.push(total);
    }
}
```

## 정답 코드 (리팩토링 - 직관적인 파싱 및 로직 분리)
복잡하게 문자열을 잘라 넘기는 대신, 문자열을 순회하며 숫자를 만들고 보너스를 만날 때마다 점수를 계산하여 스택에 넣는 방식이 훨씬 깔끔합니다. 옵션은 스택의 가장 위(최근 점수)를 꺼내서 조작하면 됩니다.

```java
import java.util.*;

class Solution {
    public int solution(String dartResult) {
        int answer = 0;
        Deque<Integer> stack = new ArrayDeque<>();
        
        int currentPoint = 0;
        
        for (int i = 0; i < dartResult.length(); i++) {
            char c = dartResult.charAt(i);
            
            // 1. 점수 파싱 (연속된 숫자인 경우 10점 처리)
            if (Character.isDigit(c)) {
                // 이전 문자가 숫자였다면 10점 (0~9 범위이므로 숫자가 연속되는 경우는 10뿐임)
                if (i > 0 && Character.isDigit(dartResult.charAt(i - 1))) {
                    currentPoint = 10;
                } else {
                    currentPoint = c - '0';
                }
            } 
            // 2. 보너스 파싱 및 스택 저장
            else if (c == 'S' || c == 'D' || c == 'T') {
                if (c == 'S') stack.push((int) Math.pow(currentPoint, 1));
                else if (c == 'D') stack.push((int) Math.pow(currentPoint, 2));
                else if (c == 'T') stack.push((int) Math.pow(currentPoint, 3));
            } 
            // 3. 옵션 파싱 (스택 조작)
            else if (c == '*' || c == '#') {
                int latest = stack.pop();
                
                if (c == '*') {
                    // 이전 점수가 있다면 꺼내서 2배 후 다시 넣기
                    if (!stack.isEmpty()) {
                        int prev = stack.pop();
                        stack.push(prev * 2);
                    }
                    // 현재 점수 2배
                    stack.push(latest * 2);
                } else if (c == '#') {
                    // 현재 점수 마이너스
                    stack.push(latest * -1);
                }
            }
        }
        
        // 스택에 남은 3번의 기회 점수를 모두 합산
        while (!stack.isEmpty()) {
            answer += stack.pop();
        }
        
        return answer;
    }
}

## 회고 및 삽질기록 🧠
- **문자열 길이 의존의 함정**: `split.length() != 4 ? ' ' : split.charAt(3)` 처럼 길이를 하드코딩해서 옵션 유무를 판단하려다 보니, 10점인지 1자리 점수인지에 따라 문자열 길이가 계속 달라져서 `StringIndexOutOfBoundsException`을 맞았다. 조건식은 최대한 직관적으로 `startsWith("10")`을 쓰거나, 아예 보너스 문자 인덱스를 기준으로 그 뒤에 남은 글자가 있는지를(`length > index`) 체크하는 방식으로 가독성을 높여야겠다.
- **수학적 오해 (`* 2` vs `pow(2)`)**: 스타상(\*)은 분명 '2배'라고 명시되어 있는데, 무의식적으로 보너스의 제곱 로직과 혼동하여 `Math.pow(prev, 2)`로 2제곱을 해버렸다. 요구사항의 '배수'와 '제곱'을 정확히 분리해서 읽고 코딩하는 습관을 들이자.
- **상태 저장 누락**: 가장 치명적인 실수. 열심히 현재 턴의 `total` 점수를 계산해 놓고 정작 `q.push(total)`을 빼먹어서 큐가 텅 비거나 이전 값만 남는 대참사가 일어났다. 메서드가 끝나는 시점에 **계산된 최종 데이터가 어디로 향해야 하는지(return 인지, 자료구조 삽입인지)** 반드시 한 번 더 점검하는 루틴을 만들어야겠다.
