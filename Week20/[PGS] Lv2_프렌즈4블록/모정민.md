### 문제 설명 (요약)
2x2 형태로 4개가 붙어있는 같은 블록을 찾아 지우고, 빈 공간을 채우기 위해 위에 있는 블록들이 중력에 의해 아래로 떨어지는 과정을 더 이상 지워질 블록이 없을 때까지 반복합니다. 연쇄적으로 지워진 블록의 총 개수를 반환하는 시뮬레이션 문제입니다.

---

### 핵심 풀이 전략
1. **상태 분리 및 일괄 삭제 (Batch Update)**: 2x2 블록을 발견하자마자 즉시 빈 칸(`""`)으로 비워버리면 겹쳐서 터져야 할 다른 2x2 블록들이 조건을 만족하지 못하게 됩니다. 발견된 좌표들을 `List`나 `Set`에 모아두고 맵 탐색이 완전히 끝난 뒤에 한 번에 지워야 합니다.
2. **턴마다 컬렉션 초기화**: 매 연쇄 폭발(턴)이 시작될 때마다 지울 좌표를 담는 컬렉션을 초기화하여 이전 턴의 쓰레기 데이터가 섞이지 않도록 격리해야 합니다.
3. **중력(Drop) 로직은 Bottom-Up**: 블록을 아래로 떨어뜨려 빈 칸을 채울 때는, 위에서 아래로(Top-Down) 탐색하면 블록이 공중에 붕 뜨는 버그가 생깁니다. 반드시 바닥(`arr.length - 1`)에서부터 꼭대기(`0`) 방향으로 훑으면서 진짜 블록을 당겨와야 연속된 빈 칸도 깔끔하게 채울 수 있습니다.

### 내가 구현한 구현 코드 (오류 포함)
```java
import java.util.*;

class Solution {
    
    int answer;
    public int solution(int m, int n, String[] board) {
        answer = 0;
        
        // 배열 초기화
        String[][] arr = new String[m][n];
        for(int i = 0; i < m; i++){
            
            String split = board[i];
            for(int j = 0; j < n; j++){
                arr[i][j] = String.valueOf(split.charAt(j));
            }
        }
        
        ArrayList<int[]> removed = new ArrayList<>();
        
        // 지울 블록이 없을 때까지 반복
        while(true){
            
            removed.clear();
            boolean isBreak = false;
            
            // m-1 x n-1 까지만 반복
            for(int i = 0; i < m-1; i++){
                for(int j = 0; j < n-1; j++){
                    if(check(i, j, arr)){
                        removed.add(new int[]{i, j});
                        removed.add(new int[]{i, j+1});
                        removed.add(new int[]{i+1, j});
                        removed.add(new int[]{i+1, j+1});
                        isBreak = true;
                    }
                }
            }
            
            
            if(!isBreak) break;
            
            // 4개인 그룹 카운트 및 제거
            for(int[] r : removed){
                int y = r[0];
                int x = r[1];
                
                if(!arr[y][x].equals("")){
                    arr[y][x] = "";
                    answer++;
                }
            }
            
            
            // 지울 블록들 일괄 삭제
            drop(arr);
        }
        
        return answer;
    }
    
    private boolean check(int y, int x, String[][] arr){
        
        String base = arr[y][x];
        
        if(base == "") return false;
        
        if(arr[y+1][x].equals(base) && arr[y][x+1].equals(base) && arr[y+1][x+1].equals(base)) return true;
        
        return false;
        
    }
    
    private void drop(String[][] arr){
    
        // 블록 재정렬
        for(int i = arr.length-1; i >= 0; i--){
            for(int j = arr[0].length-1; j >= 0; j--){
                if(arr[i][j] == ""){
                    
                    int index = i-1;
                    
                    while(index >= 0){
                        if(index >= 0 && arr[index][j] != ""){
                            arr[i][j] = arr[index][j];
                            arr[index][j] = "";
                            break;
                        }
                        index--;
                    }
                }
                
            }
        }
        
    }
}

```

## 회고 및 삽질기록 🧠
- **컬렉션 초기화 스코프(Scope)의 중요성**: 매 턴마다 지울 블록 목록을 갱신해야 하는데 리스트 선언을 `while`문 바깥에 둬버렸다. 이 때문에 이전 턴에 지웠던 좌표까지 계속 리스트에 누적되어 새로 떨어진 멀쩡한 블록들이 억울하게 지워지는 대참사가 났다. 반복적인 작업을 수행할 때는 **상태를 초기화하는 변수 선언 위치**를 최우선으로 점검하자.
- **중력(Gravity) 구현 시 탐색 방향**: 블록을 아래로 떨어뜨리는 로직을 생각할 때, 직관적으로 '위에서 아래로' 내리려고 하다 보니 연속된 빈 칸이 생겼을 때 공중에 블록이 붕 뜨는 이빨 빠짐 현상이 생겼다. 물리적인 낙하를 코드로 구현할 때는 반대로 **'바닥에서부터 위로(Bottom-Up)'** 훑으며 채워 넣어야 틈 없이 쌓인다.
- **Java 문자열 비교 `==` 함정**: 코딩 테스트 도중 무의식적으로 빈 문자열 체크를 `arr[i][j] == ""` 로 작성해 버렸다. Java에서 문자열 값 자체를 비교할 때는 무조건 `.equals()`를 사용해야 버그를 원천 차단할 수 있다.
- **Set vs List**: 좌표를 담을 때 중복을 방지하려고 `Set<int[]>`를 시도했으나, 배열은 내용물이 같아도 주소값이 다르면 `Set`이 중복을 걸러주지 못한다는 사실을 체감했다. 이번엔 `List`에 담은 뒤 `.equals("")` 체크로 중복 카운팅을 막는 방법으로 우회했지만, 다음에는 좌표를 문자열(`"y,x"`)로 변환하여 `Set<String>`에 넣거나 `Set<List<Integer>>`를 활용하는 방법을 적극 도입해야겠다.
