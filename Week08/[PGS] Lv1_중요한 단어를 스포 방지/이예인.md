# [Lv1] 중요한 단어를 스포 방지

메모리:  KB, 시간:  ms <br>
시간복잡도: O(단어 개수 × 스포일러 구간 개수)

### 제출 일자
2026년 3월 29일

### 문제 설명
메시지 `message`와 스포일러 구간 `spoiler_ranges`가 주어진다. <br>
각 스포일러 구간은 해당 범위의 문자를 가리고, 사용자가 왼쪽부터 하나씩 해제한다고 가정한다. <br>

단어를 구성하는 문자들 중 하나 이상이 스포일러 구간에 포함되면, 해당 단어 전체를 **스포 방지 단어**로 간주한다. <br>
또한 한 단어가 여러 개의 스포 방지 구간에 걸칠 수 있고, 하나의 스포 방지 구간이 여러 단어에 포함될 수도 있다. <br>

스포일러를 순서대로 해제하면서 조건을 만족하는 **중요한 단어의 개수**를 구하는 문제이다.

### 입력
- `message` : 전체 메시지 문자열
- `spoiler_ranges` : 스포일러 처리된 구간 정보

### 출력
- 중요한 단어의 개수

### 핵심 풀이 전략
처음에는 문자열을 직접 지우고 복원하는 방식으로 접근했지만,  
이 문제는 **현재 문자열 상태**보다 **단어별 상태 관리**가 더 중요했다. <br>

1. `message`를 공백 기준으로 나누어 각 단어의 문자열, 시작 인덱스, 끝 인덱스를 저장
2. 각 스포일러 구간이 어떤 단어들과 겹치는지 미리 기록
3. 각 단어가 총 몇 개의 스포일러 구간에 걸쳐 있는지 `remain` 배열에 저장
4. 스포일러에 한 번도 걸치지 않은 단어들을 `normalWords`에 저장
5. 스포일러를 왼쪽부터 하나씩 해제하면서, 영향을 받는 단어들의 `remain` 값을 감소
6. 어떤 단어의 `remain` 값이 `0`이 되는 순간, 그 단어가 완전히 공개된 시점으로 판단
7. 이때
   - 일반 구간에서 등장한 적이 없고
   - 이전에 공개된 스포 단어와 중복되지 않으면  
   중요한 단어로 카운트

### 제출 코드
```java
import java.util.*;

class Solution {
    public int solution(String message, int[][] spoiler_ranges) {
        int answer = 0;
        
        // 1. message에서 단어 정보 저장 (단어, 시작인덱스, 끝나는 인덱스)
        List<String> wordTexts = new ArrayList<>();
        List<Integer> wordStarts = new ArrayList<>();
        List<Integer> wordEnds = new ArrayList<>();
        
        String[] words = message.split(" ");
        int pos = 0;
        
        for(int i = 0 ; i < words.length; i++){
            wordTexts.add(words[i]);
            wordStarts.add(pos);
            wordEnds.add(pos + words[i].length() - 1);
            
            pos += words[i].length() + 1; // 단어길이 + 공백 1칸
        }
            
        int wordCount = wordTexts.size();
        
        // 2. 각 range가 어떤 단어들이랑 겹치는지 저장
        // rangeToWords.get(r) : r번째 스포일러 구간과 겹치는 단어 번호들 
        List<List<Integer>> rangeToWords = new ArrayList<>();
        for(int i = 0 ; i < spoiler_ranges.length; i++){
            rangeToWords.add(new ArrayList<>());
        }
        
        // 각 단어가 몇 개의 스포 구간이랑 겹치는지
        int[] remain = new int[wordCount];
        for(int r = 0 ; r < spoiler_ranges.length; r++){
            int rs = spoiler_ranges[r][0];
            int re = spoiler_ranges[r][1];
            
            for(int w = 0 ; w < wordCount; w++){
                int ws = wordStarts.get(w);
                int we = wordEnds.get(w);
                
                // 단어 [ws, we] 와 스포일러 [rs, re] 가 겹치면
                if(!(ws > re || we < rs)){
                    rangeToWords.get(r).add(w);
                    remain[w]++;
                }            
            }
        }
        
        // 3. 스포일러에 안 걸친 단어 저장
        Set<String> normalWords = new HashSet<>();
        for(int w = 0 ; w < wordCount; w++){
            if(remain[w] == 0){
                normalWords.add(wordTexts.get(w));
            }
        }
        
        // 4. 이전에 공개된 스포 방지 단어 저장
        Set<String> openedSpoilerWords = new HashSet<>();
        
        // 5. 스포 방지 하나씩 해제하기
        for(int r = 0 ; r < spoiler_ranges.length; r++){
            
            // 스포 해지로 영향 받는 단어
            List<Integer> affectedWords = rangeToWords.get(r);
            
            for(int wordIdx : affectedWords){
                remain[wordIdx]--;
            }
            
            for(int wordIdx : affectedWords){
                if(remain[wordIdx] != 0) continue; // 아직 다른 스포가 남아 있는 단어
                
                String word = wordTexts.get(wordIdx);
                
                // 일반 구간에 포함된 단어면 중요 단어 아님
                if(normalWords.contains(word)) continue;

                // 이전에 공개된 스포 단어와 중복이면 중요 단어 아님
                if(openedSpoilerWords.contains(word)) continue;
                
                answer++;
                openedSpoilerWords.add(word);
            }
        }
         
        return answer;
    }  
}

```

### 삽길 기록 🧠
- 처음에는 StringBuilder로 스포일러 문자를 직접 지웠다가 다시 넣는 방식으로 접근했다.
- 하지만 한 단어가 여러 개의 스포일러 구간에 걸칠 수 있다는 점을 놓쳐서,
- 어떤 단어가 이번 클릭으로 완전히 공개된 것인지 판단하는 데 어려움을 겪었다.
- 또한 현재 문자열에서 단어 등장 횟수를 세는 방식으로 풀려고 했는데,
- 문제의 핵심은 단순 등장 횟수가 아니라 일반 구간에 등장했는지
- 이전에 공개된 스포 단어와 중복되는지를 따져야 한다는 점을 뒤늦게 파악했다.
- 결국 문자열 자체를 복원하는 방식보다, 단어별 시작/끝 인덱스와 남은 스포일러 개수를 관리하는 방식으로 바꾸면서 해결했다.
