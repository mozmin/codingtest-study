### 문제 설명
비밀 코드는 `1`부터 `n`까지의 수 중 서로 다른 5개 정수로 이루어져 있고, 오름차순으로 정렬되어 있다. <br>  
각 시도마다 5개의 수를 입력하면, 시스템은 그중 비밀 코드에 포함된 숫자의 개수를 알려준다.  <br>
주어진 여러 번의 시도 결과를 바탕으로, 가능한 비밀 코드 조합의 개수를 구하는 문제다.<br>

### 입력
- `n` : 사용할 수 있는 정수의 최대값
- `q` : 각 시도에서 입력한 5개의 정수 목록
- `ans` : 각 시도에서 비밀 코드와 일치한 숫자 개수

### 출력
- 모든 조건을 만족하는 비밀 코드 조합의 개수

### 핵심 풀이 전략
처음엔 조건을 보고 숫자를 줄여가며 추론해야 할 것 같았지만,  <br> 
이 문제는 제한이 작아서 5개 조합을 전부 만들어 확인하는 완전탐색이 가장 깔끔했다. <br>

1. `1 ~ n` 중에서 5개를 뽑는 모든 조합을 만든다.
2. 만들어진 조합이 각 시도 `q[i]`와 몇 개의 숫자를 공유하는지 센다.
3. 그 개수가 `ans[i]`와 모두 같으면 가능한 비밀 코드다.
4. 가능한 조합의 개수를 누적해 반환한다.

문제의 핵심은 위치를 비교하는 게 아니라 포함된 숫자 개수만 비교하는 점이다. <br>

### 제출 코드
```java
class Solution {
    public int solution(int n, int[][] q, int[] ans) {
        int[] selected = new int[5];
        return comb(1, 0, n, q, ans, selected);
    }
    
    int comb(int start, int depth, int n, int[][] q, int[] ans, int[] selected) {
        if (depth == 5) {
            if (isValid(q, selected, ans)) return 1;
            return 0;
        }
        
        int count = 0;
        
        for (int i = start; i <= n; i++) {
            selected[depth] = i;
            count += comb(i + 1, depth + 1, n, q, ans, selected);
        }
        
        return count;
    }
    
    boolean isValid(int[][] q, int[] selected, int[] ans) {
        for (int i = 0; i < q.length; i++) {
            int count = 0;
            
            for (int j = 0; j < 5; j++) {
                for (int k = 0; k < 5; k++) {
                    if (q[i][j] == selected[k]) {
                        count++;
                    }
                }
            }
            
            if (count != ans[i]) return false;
        }
        
        return true;
    }
}
```
---
### 삽질 기록 🧠
- 처음엔 q[i][j] == selected[j]처럼 같은 인덱스끼리 비교하려고 했다.
- 그런데 이 문제는 위치가 중요한 게 아니라, 같은 숫자가 몇 개 포함되었는지만 중요했다.
- 예를 들어 [1,2,3,4,5]와 [5,4,3,2,1]은 위치는 다르지만 같은 숫자 5개를 공유한다.
- 그래서 단순 인덱스 비교가 아니라, 각 숫자가 상대 배열 안에 존재하는지를 전부 확인해야 했다.
- 그다음부터는 조건 분석보다 그냥 모든 5개 조합을 만들고 검증하는 방식으로 가는 게 가장 안전하다고 판단했다.
