### 문제 설명
컴퓨터의 개수 `n`과 연결 정보가 담긴 2차원 배열 `computers`가 주어진다.  
각 컴퓨터가 서로 연결되어 있는지 확인하여 총 네트워크의 개수를 구하는 문제이다.

여기서 네트워크란 직접 또는 간접적으로 연결된 컴퓨터들의 집합을 의미한다.

### 입력
- `n` : 컴퓨터의 개수
- `computers` : 컴퓨터 간 연결 여부를 나타내는 2차원 배열

### 출력
- 네트워크의 개수

### 핵심 풀이 전략
이 문제는 그래프에서 연결 요소의 개수를 구하는 문제로 볼 수 있다.

1. `visited` 배열로 각 컴퓨터의 방문 여부를 관리한다.
2. 0번부터 `n-1`번까지 순회하면서 아직 방문하지 않은 컴퓨터를 찾는다.
3. 방문하지 않은 컴퓨터를 발견하면 새로운 네트워크라고 판단하여 `count`를 증가시킨다.
4. 해당 컴퓨터를 시작점으로 BFS를 수행하여 연결된 모든 컴퓨터를 방문 처리한다.
5. BFS가 한 번 끝날 때마다 하나의 네트워크 탐색이 완료된다.
6. 전체 순회가 끝난 뒤 `count`를 반환한다.

방문하지 않은 컴퓨터에서 BFS를 시작한 횟수 = 네트워크 개수가 된다.

### 제출 코드
```java
import java.util.*;

class Solution {
    public int solution(int n, int[][] computers) {
        int count = 0;
        boolean[] visited = new boolean[n];
        Queue<Integer> queue = new LinkedList<>();
        
        for (int i = 0; i < n; i++) {
            if (visited[i] == true) continue;
            
            count++;
            queue.add(i);
            
            while (!queue.isEmpty()) {
                int cur = queue.poll();
                visited[cur] = true;
                
                for (int j = 0; j < n; j++) {
                    if (cur == j) continue;
                    if (visited[j] != true && computers[cur][j] == 1) {
                        queue.add(j);
                    }
                }
            }
        }
        
        return count;
    }
}
```

### 삽길 기록 🧠
- 처음에는 BFS 구조는 맞게 잡았지만, 탐색 기준을 i로 두고 있어서 현재 노드 기준으로 연결을 확인하지 못했다.
- 이후 cur를 기준으로 연결된 컴퓨터를 탐색하도록 수정하면서 정상적으로 네트워크를 찾을 수 있었다.
