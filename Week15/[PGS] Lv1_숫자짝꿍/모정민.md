### 문제 설명 (요약)
두 정수 `X`, `Y`의 짝꿍은 `X`, `Y`에서 공통으로 나타나는 정수(0~9)들을 이용하여 만들 수 있는 가장 큰 정수를 의미합니다. 

- **조건**: 
  - 공통으로 나타나는 정수가 없다면 짝꿍은 `-1`입니다.
  - 짝꿍이 0으로만 구성되어 있다면 짝꿍은 `0`입니다.
  - `X`와 `Y`의 길이는 최대 3,000,000입니다.

---

### 핵심 풀이 전략
1. **문자 빈도수 카운팅**: `X`와 `Y`에 등장하는 각 숫자(0~9)의 개수를 각각 센다.
2. **교집합(공통 숫자) 추출**: 각 숫자에 대해 두 문자열에서 등장한 횟수 중 최솟값(`Math.min`)만큼 해당 숫자를 추출한다.
3. **가장 큰 수 만들기**: 추출된 숫자들을 내림차순으로 정렬하여 이어 붙인다. (가장 큰 자릿수에 가장 큰 숫자가 와야 하므로)
4. **예외 처리**: 공통 숫자가 없으면 `-1`, 가장 큰 숫자가 `0`이라면 (즉, 0만 여러 개 있다면) `0`을 반환한다.

---

### 내가 구현한 구현 코드
```java
import java.util.*;

class Solution {
    public String solution(String X, String Y) {
        
        Map<Character, Integer> mapX = new HashMap<>();
        Map<Character, Integer> mapY = new HashMap<>();
        
        // 카운팅
        for(int i = 0; i < X.length(); i++){
            mapX.put(X.charAt(i), mapX.getOrDefault(X.charAt(i), 0) + 1);
        }
        
        for(int i = 0; i < Y.length(); i++){
            mapY.put(Y.charAt(i), mapY.getOrDefault(Y.charAt(i), 0) + 1);
        }
        
        // 교집합 구하기
        ArrayList<Integer> result = new ArrayList<>();
        for(Character key : mapX.keySet()){
            
            if(mapY.containsKey(key)){
                int count = Math.min(mapX.get(key), mapY.get(key));
                for(int i = 0; i < count; i++){
                    result.add(key - '0');
                }
            }
        
        }
        
        // 정렬
        result.sort(Comparator.reverseOrder());
        
        if(result.isEmpty()) return "-1";
        if(result.get(0) == 0) return "0";
        
        StringBuilder sb = new StringBuilder();
        for(int r : result){
            sb.append(r);
        }

        return sb.toString();
    }
}
```

### 정답 코드 (리팩토링 - 배열 인덱스를 활용한 $O(N)$ 최적화)
작성하신 코드는 로직상 완벽하지만, `X`와 `Y`의 길이가 최대 300만이기 때문에 `ArrayList`에 넣고 `sort`를 수행하면 시간 초과(TLE)가 발생할 위험이 있습니다. 
숫자가 0부터 9까지 10개뿐이라는 점을 이용해 **길이가 10인 배열**로 카운팅하고, **9부터 0까지 역순으로 순회**하면 별도의 정렬 과정 없이 $O(N)$의 시간 복잡도로 빠르게 해결할 수 있습니다.

```java
class Solution {
    public String solution(String X, String Y) {
        // 0부터 9까지 숫자의 개수를 저장할 배열
        int[] countX = new int[10];
        int[] countY = new int[10];
        
        // 숫자 카운팅 (문자 '0'을 빼서 인덱스로 사용)
        for (char c : X.toCharArray()) countX[c - '0']++;
        for (char c : Y.toCharArray()) countY[c - '0']++;
        
        StringBuilder sb = new StringBuilder();
        
        // 9부터 0까지 역순으로 순회하여 자연스럽게 내림차순 정렬 효과를 냄
        for (int i = 9; i >= 0; i--) {
            int commonCount = Math.min(countX[i], countY[i]);
            for (int j = 0; j < commonCount; j++) {
                sb.append(i);
            }
        }
        
        // 예외 처리
        String result = sb.toString();
        if (result.isEmpty()) return "-1";
        if (result.charAt(0) == '0') return "0";
        
        return result;
    }
}
```

### 회고 및 삽질기록 🧠
- 저번 '뉴스 클러스터링' 문제에서 익히셨던 다중집합의 교집합 로직(`Math.min`을 활용한 빈도수 추출)을 활용
- '0'을 처리하는 과정에서 실수가 있었다. result가 '000000'이면 '0'으로 바꾸는 로직이 누락되었다.

### 배운 점 && 보완할 점
- **문자열 순회 최적화**: `X.charAt(i)`로 인덱스 접근을 하는 것도 좋지만, 자바에서는 `X.toCharArray()`를 활용하여 향상된 `for`문(`for(char c : X.toCharArray())`)을 사용하면 코드가 한결 간결해진다.
