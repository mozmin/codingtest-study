### 문제 설명 (요약)
카카오톡 이모티콘 플러스 서비스 가입자를 최대한 늘리고, 이모티콘 판매액을 그다음으로 극대화하는 최적의 이모티콘 할인율 조합을 찾는 문제입니다.

- **조건**:
  - 할인율은 `10%`, `20%`, `30%`, `40%` 중 하나로 설정됩니다.
  - 이모티콘의 개수(`emoticons`의 길이)는 1 이상 7 이하입니다.
  - 사용자의 수(`users`의 길이)는 1 이상 100 이하입니다.
  - 각 사용자는 자신의 기준 할인율 이상인 이모티콘을 모두 구매하며, 총 구매 금액이 자신의 기준 가격을 넘어가면 구매를 모두 취소하고 '이모티콘 플러스' 서비스에 가입합니다.

---

### 핵심 풀이 전략
1. **완전 탐색(Brute Force) 가능성 파악**: 이모티콘의 최대 개수가 7개이고 각 이모티콘마다 적용할 수 있는 할인율이 4가지이므로, 가능한 모든 할인율의 조합은 $4^7 = 16,384$가지입니다. 여기에 사용자 100명을 매번 계산해도 약 160만 번의 연산으로 충분히 시간 내에 해결이 가능하므로 **백트래킹(DFS)**을 이용해 모든 조합을 탐색합니다.
2. **우선순위 기반 갱신**: 문제의 최우선 목적은 **1) 서비스 가입자 수 극대화**, 그 다음이 **2) 판매 매출 극대화**입니다. 모든 할인율 조합을 탐색할 때마다 이 두 조건을 비교하여 최적의 결과값을 갱신합니다.
3. **사용자별 구매 시뮬레이션**: 특정 할인율 조합이 결정되면, 각 사용자별로 이모티콘을 순회하며 설정된 할인율이 사용자의 기준 할인율 이상인지 확인하고 구매 금액을 누적합니다. 이후 누적 금액이 사용자의 한도 금액 이상이면 가입자 수를 늘리고, 아니라면 매출액에 더해줍니다.

---

### 정답 코드 (백트래킹을 활용한 $O(4^M \times N)$ 완전 탐색 풀이)

```java
import java.util.*;

class Solution {
    
    int[] discount = {10, 20, 30, 40};
    
    int maxJoin = 0;
    int maxSales = 0;
    
    public int[] solution(int[][] users, int[] emoticons) {
        int[] curChoiceSalesPercent = new int[emoticons.length];
        
        dfs(0, curChoiceSalesPercent, users, emoticons);
        
        return new int[]{maxJoin, maxSales};
    }
    
    public void dfs(int count, int[] curChoice, int[][] users, int[] emoticons){
        
        // 만약 count가 이모티콘 갯수와 동일하다면 -> user 계산
        if(count == emoticons.length){
            calculate(curChoice, users, emoticons);
            return;
        }
        
        // 모든 할인율 조합을 시도하며 dfs 호출
        for(int i = 0; i < discount.length; i++) {
            curChoice[count] = discount[i];
            dfs(count + 1, curChoice, users, emoticons);
        }
    }
    
    // 유저별 구매 금액과 플러스 가입 여부를 계산하는 헬퍼 메서드
    private void calculate(int[] curChoice, int[][] users, int[] emoticons) {
        int currentJoin = 0;
        int currentSales = 0;
        
        for(int[] user : users) {
            int userMinDiscount = user[0]; 
            int userMaxBudget = user[1];   
            
            int totalSpent = 0;
            
            // 현재 할인율 조합으로 이모티콘 구매액 계산
            for(int i = 0; i < emoticons.length; i++) {
                if(curChoice[i] >= userMinDiscount) {
                    // 할인된 가격 계산
                    totalSpent += emoticons[i] * (100 - curChoice[i]) / 100;
                }
            }
            
            // 예산 이상을 썼다면 구매를 모두 취소하고 플러스 가입
            if(totalSpent >= userMaxBudget) {
                currentJoin++;
            } else {
                // 예산 미만이면 판매액에 누적
                currentSales += totalSpent;
            }
        }
        
        
        if(currentJoin > maxJoin) {
            maxJoin = currentJoin;
            maxSales = currentSales;
        } else if(currentJoin == maxJoin) {
            if(currentSales > maxSales) {
                maxSales = currentSales;
            }
        }
    }
}
```
### 삽질 기록
- DFS를 구현하면서 각 이모티콘의 할인율 조합을 담을 curChoiceSalesPercent 배열을 매 재귀 호출마다 복사(clone)해서 넘겨야 한다고 생각함
-> 어차피 dfs()는 백트레킹 방식이기에 다음 경우의 수를 살펴볼 때 덮어씌워진다.

### 배운점
- DFS/백트래킹의 완벽한 흐름 이해: DFS는 한 번에 하나의 조합을 향해 일직선으로 끝까지 탐색(count == emoticons.length)한 뒤, 연산(calculate)을 수행하고 다시 한 단계씩 되돌아온다는(백트래킹) 점을 명확히 이해함.
