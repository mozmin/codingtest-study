# [Level 2] 이모티콘 할인행사

[문제 링크](https://school.programmers.co.kr/learn/courses/30/lessons/150368?language=java)

`#구현` `#브루트포스` `#백트래킹` `#DFS`

### 제출 일자
2026년 5월 21일

### 문제 설명 (요약)
카카오톡 이모티콘 플러스 서비스 가입자를 최대한 늘리고, 이모티콘 판매액을 그다음으로 극대화하는 최적의 이모티콘 할인율 조합을 찾는 문제입니다.

- **조건**:
  - 할인율은 `10%`, `20%`, `30%`, `40%` 중 하나로 설정됩니다.
  - 이모티콘의 개수(`emoticons`의 길이)는 1 이상 7 이하입니다.
  - 사용자의 수(`users`의 길이)는 1 이상 100 이하입니다.
  - 각 사용자는 자신의 기준 할인율 이상인 이모티콘을 모두 구매하며, 총 구매 금액이 자신의 기준 가격을 넘어가면 구매를 모두 취소하고 '이모티콘 플러스' 서비스에 가입합니다.

---

### 핵심 풀이 전략
1. **완전 탐색(Brute Force) 가능성 파악**: 이모티콘의 최대 개수가 7개이고 각 이모티콘마다 적용할 수 있는 할인율이 4가지이므로, 가능한 모든 할인율의 조합은 $4^7 = 16,384$가지입니다. 여기에 사용자 100명을 매번 계산해도 약 160만 번의 연산으로 충분히 시간 내에 해결이 가능하므로 **백트래킹(DFS)**을 이용해 모든 조합을 탐색합니다.
2. **우선순위 기반 갱신**: 문제의 최우선 목적은 **1) 서비스 가입자 수 극대화**, 그 다음이 **2) 판매 매출 극대화**입니다. 모든 할인율 조합을 탐색할 때마다 이 두 조건을 비교하여 최적의 결과값을 갱신합니다.
3. **사용자별 구매 시뮬레이션**: 특정 할인율 조합이 결정되면, 각 사용자별로 이모티콘을 순회하며 설정된 할인율이 사용자의 기준 할인율 이상인지 확인하고 구매 금액을 누적합니다. 이후 누적 금액이 사용자의 한도 금액 이상이면 가입자 수를 늘리고, 아니라면 매출액에 더해줍니다.

---

### 정답 코드 (백트래킹을 활용한 $O(4^M \times N)$ 완전 탐색 풀이)

```java
class Solution {
    // 할인율 종류
    private static final int[] DISCOUNTS = {10, 20, 30, 40};
    
    // 최적의 결과를 저장할 변수 (0: 가입자 수, 1: 매출액)
    private int maxSignUps = 0;
    private int maxSales = 0;

    public int[] solution(int[][] users, int[] emoticons) {
        // 각 이모티콘의 할인율을 담을 배열
        int[] selectedDiscounts = new int[emoticons.length];
        
        // 백트래킹 시작 (0번째 이모티콘부터 시작)
        dfs(0, selectedDiscounts, users, emoticons);
        
        return new int[]{maxSignUps, maxSales};
    }

    private void dfs(int depth, int[] selectedDiscounts, int[][] users, int[] emoticons) {
        // 모든 이모티콘의 할인율 조합이 완성된 경우
        if (depth == emoticons.length) {
            calculateResult(selectedDiscounts, users, emoticons);
            return;
        }

        // 10%, 20%, 30%, 40% 할인율을 각각 적용해보며 탐색
        for (int i = 0; i < 4; i++) {
            selectedDiscounts[depth] = DISCOUNTS[i];
            dfs(depth + 1, selectedDiscounts, users, emoticons);
        }
    }

    private void calculateResult(int[] selectedDiscounts, int[][] users, int[] emoticons) {
        int currentSignUps = 0;
        int currentSales = 0;

        // 모든 사용자에 대해 시뮬레이션 진행
        for (int[] user : users) {
            int userDiscountThreshold = user[0];
            int userPriceThreshold = user[1];
            int userSpent = 0;

            for (int i = 0; i < emoticons.length; i++) {
                // 사용자의 기준 할인율 이상인 경우에만 구매
                if (selectedDiscounts[i] >= userDiscountThreshold) {
                    // 할인 가격 계산: 원가 * (100 - 할인율) / 100
                    userSpent += emoticons[i] * (100 - selectedDiscounts[i]) / 100;
                }
            }

            // 구매 총액이 사용자의 한도 가격 이상이면 플러스 가입
            if (userSpent >= userPriceThreshold) {
                currentSignUps++;
            } else {
                // 한도 미만이면 정기 매출로 합산
                currentSales += userSpent;
            }
        }

        // 최적의 결과 조건 비교 및 갱신
        if (currentSignUps > maxSignUps) {
            maxSignUps = currentSignUps;
            maxSales = currentSales;
        } else if (currentSignUps == maxSignUps) {
            maxSales = Math.max(maxSales, currentSales);
        }
    }
}
```
