### 문제 설명 (요약)
$m \times n$ 크기의 격자 모양 지도가 주어집니다. 가장 왼쪽 위인 집 $(1, 1)$에서 가장 오른쪽 아래인 학교 $(m, n)$까지 가려고 합니다. 일부 지역은 물에 잠겨 지나갈 수 없습니다.

- **조건**:
  - **오른쪽과 아래쪽으로만** 움직여서 이동해야 합니다 (최단거리 이동).
  - 격자의 크기 $m, n$은 1 이상 100 이하의 자연수입니다.
  - 물에 잠긴 지역의 좌표를 담은 2차원 배열 `puddles`가 주어집니다.
  - 최단경로의 개수를 `1,000,000,007`로 나눈 나머지를 반환해야 합니다.

---

### 핵심 풀이 전략
1. **동적 계획법(DP) 적용**: 특정 칸 $(i, j)$까지 오는 최단 경로의 개수는 왼쪽 칸 $(i-1, j)$에서 오는 경로의 개수와 위쪽 칸 $(i, j-1)$에서 오는 경로의 개수의 합입니다.
   $$DP[i][j] = DP[i-1][j] + DP[i][j-1]$$
2. **좌표 맞추기 (주의)**: 문제에서 주어지는 격자의 크기는 가로가 $m$, 세로가 $n$이며, `puddles` 좌표도 `[가로_좌표, 세로_좌표]` 형태로 주어집니다. 일반적인 2차원 배열의 `[행][열]` 구조와 반대이므로, 배열을 선언할 때 `new int[n + 1][m + 1]`로 설정하면 직관적으로 매핑할 수 있습니다.
3. **웅덩이 및 예외 처리**: 웅덩이가 있는 위치는 경로의 수가 `0`이 되도록 설정하고 계산에서 제외합니다.
4. **오버플로우 방지**: 경로의 수가 커질 수 있으므로, 매 연산마다 `1,000,000,007`로 나눈 나머지를 저장해야 합니다.

---

### 정답 코드 (배열 인덱스를 활용한 $O(M \times N)$ DP 풀이)

```java
class Solution {
    public int solution(int m, int n, int [][] puddles) {
        // 주어지는 좌표가 1부터 시작하므로 (n+1) x (m+1) 크기로 선언
        int[][] dp = new int[n + 1][m + 1];
        
        // 웅덩이 위치를 -1로 표시 (가로 m이 열, 세로 n이 행)
        for (int[] puddle : puddles) {
            dp[puddle[1]][puddle[0]] = -1;
        }
        
        // 시작 지점(집) 초기화
        dp[1][1] = 1;
        
        for (int i = 1; i <= n; i++) {
            for (int j = 1; j <= m; j++) {
                // 웅덩이가 있는 곳은 패스 (0으로 만들어 경로가 이어지지 않게 함)
                if (dp[i][j] == -1) {
                    dp[i][j] = 0;
                    continue;
                }
                
                // 위쪽에서 오는 경로 추가
                if (i > 1) {
                    dp[i][j] = (dp[i][j] + dp[i - 1][j]) % 1000000007;
                }
                
                // 왼쪽에서 오는 경로 추가
                if (j > 1) {
                    dp[i][j] = (dp[i][j] + dp[i][j - 1]) % 1000000007;
                }
            }
        }
        
        return dp[n][m];
    }
}
```

### 배운 점 && 보완할 점
- 이 문제는 내 스스로 풀지 못했다...
- 좌표계 매핑 주의: 알고리즘 문제에서 (m, n)이 (가로, 세로)로 주어질 때 코딩 시 2차원 배열의 행/열 인덱스 [row][col] 체계와 헷갈리기 쉽다. 문제의 입력 형태를 확실히 확인하고 dp[세로][가로] 구조로 안전하게 매핑하는 연습이 필요.
- 모듈러 연산의 위치: 최종 결과에만 % 1000000007을 해주면 중간 과정에서 int 범위를 초과하는 오버플로우가 발생할 수 있다. DP 테이블을 갱신하는 모든 더하기 연산 직후에 모듈러 연산을 적용해 주어야 안전하다.
