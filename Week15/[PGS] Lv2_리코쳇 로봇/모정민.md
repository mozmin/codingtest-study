# [Level 2] 리코쳇 로봇

[문제 링크](https://school.programmers.co.kr/learn/courses/30/lessons/169199)

`#BFS` `#너비우선탐색` `#시뮬레이션`

### 제출 일자
2026년 5월 21일

### 문제 설명 (요약)
보드판 위에서 로봇이 최소 움직임으로 목표 위치(`G`)까지 도달하는 최단 거리를 구하는 문제입니다.

- **이동 규칙**: 로봇은 상, 하, 좌, 우 4방향으로 미끄러져 이동하며, **장애물(`D`)이나 보드판의 끝에 부딪힐 때까지 멈추지 않고 한 번에 쭉 직진**합니다. 부딪혀서 멈춘 위치가 하나의 이동 단위가 됩니다.
- **조건**: 
  - 목표 위치에 도달할 수 없다면 `-1`을 반환합니다.
  - 보드판의 가로, 세로 길이는 최대 100입니다.

---

### 핵심 풀이 전략
1. **최단 거리 탐색(BFS)**: 미끄러져 이동하는 횟수의 최솟값을 구해야 하므로 가중치가 없는 그래프의 최단 거리를 찾는 BFS(너비 우선 탐색)가 가장 적합합니다.
2. **미끄러짐 구현 (While문)**: 방향 전환 후 장애물(`D`)을 만나거나 경계선 밖으로 나가기 전까지 `while` 루프를 돌며 좌표를 계속 갱신합니다. 벽에 부딪히기 직전의 마지막 유효한 좌표가 로봇이 멈추는 지점입니다.
3. **방문 처리(Visited)**: 로봇이 미끄러져서 **최종적으로 멈춰 선 위치**를 기준으로 방문 여부를 체크해야 무한 루프를 방지하고 효율적으로 탐색할 수 있습니다.

---

### 내가 구현한 구현 코드
```java
import java.util.*;
class Solution {
    
    int[] dy = {0, 0, 1, -1};
    int[] dx = {1, -1, 0, 0};
    int min;
    public int solution(String[] board) {
        min = -1;
        
        int m = board.length;
        int n = board[0].length();
        
        String[][] map = new String[m][n];
        
        int startY = 0;
        int startX = 0;
        
        // map 초기화
        for(int i = 0; i < m; i++){
            for(int j = 0; j < n; j++){
                char c = board[i].charAt(j);
                
                if(c == 'R'){
                    startY = i;
                    startX = j;
                }
                
                map[i][j] = String.valueOf(c);
            }
        }
        
        bfs(map, startY, startX, m, n);
        
        return min;
    }
    
    public void bfs(String[][] map, int startY, int startX, int m, int n){
        
        Deque<int[]> q = new ArrayDeque<>();
        
        boolean[][] visited = new boolean[m][n];
        
        q.offer(new int[]{startY, startX, 0});
        visited[startY][startX] = true;
        
        while(!q.isEmpty()){
            
            int[] poll = q.poll();
            int curY = poll[0];
            int curX = poll[1];
            int curCount = poll[2];
            
            if(map[curY][curX].equals("G")){
                min = curCount;
            }
            
            for(int i = 0; i < 4; i++){
                int nY = curY;
                int nX = curX;
                
                while (true) {
                    int nextY = nY + dy[i];
                    int nextX = nX + dx[i];

                    // 다음 칸이 범위를 벗어나거나 장애물이라면 더 이상 갈 수 없음
                    if (nextY < 0 || nextY >= m || nextX < 0 || nextX >= n || map[nextY][nextX].equals("D")) {
                        break;
                    }

                    // 조건이 만족하면 해당 방향으로 계속
                    nY = nextY;
                    nX = nextX;
                }

                // 멈춰 선 최종 위치(nY, nX)를 방문한 적이 없다면 큐에 삽입
                if (!visited[nY][nX]) {
                    
                    // [삽질 포인트] 처음 구현 시 이 방문 처리 코드를 누락하여 
                    // 동일한 좌표를 계속해서 큐에 담아 메모리 초과/시간 초과 위험 및 무한 루프 발생 가능성이 있었음
                    visited[nY][nX] = true;
                    q.offer(new int[]{nY, nX, curCount + 1});
                }
            }
        }
        
        return;
    } 
}
```

### 회고 및 삽질기록 🧠
- 미끄러져서 최종적으로 멈춘 위치(nY, nX)에 대해 visited 방문 체크 조건문과 실질적인 visited[nY][nX] = true; 상태 갱신을 순간적으로 깜빡했다. BFS에서 중복 방문 제거(visited 처리)를 누락하면 이미 탐색한 경로를 다시 큐에 집어넣어 큐가 무한히 증식하거나 무한 루프에 빠지게 됨을 다시금 명심하자.

### 배운 점 && 보완할 점
- BFS에서의 방문 처리 시점: 탐색 유형에 따라 '이동하는 과정의 모든 칸'을 체크해야 할 때가 있고, 이 문제처럼 '최종적으로 정지하는 칸'만 체크해야 할 때가 있다. 문제의 규칙(미끄러져서 멈춘 후 재출발)에 맞는 정확한 단위를 식별하는 안목을 유지하자.
- 공간 최적화: String을 2차원 배열로 전부 쪼개서 담는 것보다, String[].charAt()을 활용하는 것이 100x100 크기에서도 불필요한 객체 생성을 줄이는 깔끔한 방법이다.
