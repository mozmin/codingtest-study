### 문제 설명 (요약)
창고에 쌓인 알파벳(A~Z) 컨테이너들을 정해진 출고 요청(`requests`) 규칙에 따라 꺼낸 뒤, 최종적으로 남은 컨테이너의 개수를 구하는 문제.
- **storage**: 컨테이너의 초기 상태를 담은 1차원 문자열 배열 (최대 50x50)
- **requests**: 출고 요청을 담은 1차원 문자열 배열
    - 길이가 1인 경우 (지게차): 외부와 연결된(상하좌우 중 하나라도 뚫려 있는) 해당 알파벳 컨테이너만 제거.
    - 길이가 2인 경우 (크레인): 외부 연결 여부와 상관없이 해당 알파벳 컨테이너를 모두 제거.

---

### 핵심 풀이 전략
1. 배열 크기를 `m+2`, `n+2`로 하여 컨테이너 주변에 빈 공간을 만든다.
2. `(0,0)` 에서 BFS()를 사용하여 접근 가능한 컨테이너를 조사한다.
3. 만약 접근이 안된다면 해당 컨테이너는 사방이 막혀있다는 뜻이고, 접근 가능하다면 지게차가 들어갈 수 있다는 의미이다.
4. `BB` 처럼 길이가 2인 경우는 for문을 돌면서 모든 값들을 `""`로 바꾼다.
5. 마지막으로 전체 for문을 돌면서 값이 있는 컨테이너만 카운트한다.

---

### Java 구현 코드

```java
import java.util.*;
class Solution {
    
    int[] dy = {0, 0, 1, -1};
    int[] dx = {1, -1, 0, 0};
    public int solution(String[] storage, String[] requests) {
        int answer = 0;
        
        String[][] containers = new String[storage.length + 2][storage[0].length() + 2];
        for(int i = 0; i < containers.length; i++){
            Arrays.fill(containers[i], "");
        }
        
        // 배열 초기화
        for(int i = 1; i <= storage.length; i++){
            for(int j = 1; j <= storage[0].length(); j++){
                containers[i][j] = String.valueOf(storage[i-1].charAt(j-1));
            }
        }
        
        // 탐색
        for(int r = 0; r < requests.length; r++){
            String value = requests[r];
            
            // 지게차로 접근 가능한 컨테이너만
            if(value.length() == 1){
                List<int[]> removeList = new ArrayList<>();

                for(int i = 1; i <= storage.length; i++){
                    for(int j = 1; j <= storage[0].length(); j++){
                        if(value.equals(containers[i][j]) && check4Walls(containers, i, j)) {
                            removeList.add(new int[]{i, j});
                        }
                    }
                }

                // 한 번에 제거
                for(int[] pos : removeList){
                    containers[pos[0]][pos[1]] = "";
                }
            }
            
            // 모든 컨테이너
            else if (value.length() == 2){
                value = String.valueOf(value.charAt(0));
                for(int i = 1; i <= storage.length; i++){
                    for(int j = 1; j <= storage[0].length(); j++){
                        if(value.equals(containers[i][j])) containers[i][j] = "";
                    }
                }
            }
                  
        }
        
        // 비어있지 않은 컨테이너 카운트
        for(int i = 1; i < containers.length; i++){
            for(int j = 1; j < containers[0].length; j++){
                if(!"".equals(containers[i][j])) answer++;
            }
        }
        
        return answer;
    }
    
    // (0, 0) 부터 시작해서 해당 컨테이너에 접근이 가능한지 파악
    // true -> 해당 컨테이너 주변에 빈 공간이 있다.
    // false -> 해당 컨테이너 주변이 전부 막혀있다.
    public boolean check4Walls(String[][] containers, int y, int x){
        
        int m = containers.length;
        int n = containers[0].length;
        
        boolean[][] visited = new boolean[m][n];
        Deque<int[]> que = new ArrayDeque<>();
        que.offer(new int[]{0, 0});
        
        visited[0][0] = true;
        
        while(!que.isEmpty()){
            
            int[] poll = que.poll();
            int curY = poll[0];
            int curX = poll[1];
            
            if(curY == y && curX == x) return true;
            
            for(int i = 0; i < 4; i++){
                int nextY = curY + dy[i];
                int nextX = curX + dx[i];
                
                if(nextY >= 0 && nextY < m && nextX >= 0 && nextX < n 
                   && !visited[nextY][nextX] 
                   && (containers[nextY][nextX].equals("") || (nextY == y && nextX == x))) {

                    que.offer(new int[]{nextY, nextX});
                    visited[nextY][nextX] = true;
                }
                
            }
        }
        
        
        return false;
    }
}
```

### 삽질 기록 🧠
1. 배열 인덱스 범위 초과 <= length 사용
- 배열은 0 ~ length-1까지만 가능
- i < length로 써야 함
  
2. 문자열 비교를 ==로 함
containers[i][j] == value
- 주소 비교라서 틀림
- value.equals(containers[i][j]) 로 해야함

3. 빈 문자열 비교 실수
containers[i][j] != ""
- 이것도 주소 비교
- !"".equals(containers[i][j])
  
4. 배열 초기값 null 처리 안 함
String[][] 기본값은 null
- "" 비교 시 문제 발생
- Arrays.fill(..., "")로 초기화
  
5. BFS 범위 조건 오류 nextY <= m, nextX <= n
- 범위 초과, nextY < m, nextX < n 로 해야 함.
- m과 n은 length이기 때문이다.
  
6. BFS에서 목표 지점 도달 불가 구조
- 빈 칸("")만 이동 가능하게 함
- 목표 컨테이너로 못 들어감
- 목표 좌표는 예외 허용

8. 탐색하면서 배열 바로 수정 **(핵심)**
-조건 만족하면 즉시 containers[i][j] = ""
- 이후 탐색에 영향 → 연쇄 제거 발생
- 삭제 대상 모아서 한 번에 처리


### 보완할 점
1. 문자열 비교 실수 (== vs equals)
- 자바에서 문자열은 반드시 equals()로 비교해야 한다는 기본기를 다시 점검할 필요 있음

2. 배열 인덱스 범위 관리 미흡
- <= length 같은 실수로 인해 런타임 에러 발생
- 반복문 작성 시 항상 범위를 명확히 확인하는 습관 필요

3. 탐색 중 상태 변경으로 인한 오류
- 탐색과 동시에 값을 변경하여 연쇄적인 영향 발생
- 이후에는 `탐색 → 결과 수집 → 일괄 반영` 패턴을 유지해야 함

4. 초기값(null) 처리에 대한 고려 부족
- 배열 초기화 없이 비교 연산을 수행하여 예외 가능성 존재
- `배열은 null로 초기화` 됨을 잊지말자
