### 성능 요약
메모리: KB, 시간: ms  
시간복잡도: O(r × n × m × n × m)

- `r`: 요청 개수
- `n`, `m`: 창고의 행, 열 크기

### 제출 일자
2026년 4월 23일

### 문제 설명
창고에 보관된 컨테이너들 중에서 요청에 따라 물건을 꺼내는 문제.

- 요청 문자열의 길이가 2이면 크레인을 사용하므로, 해당 종류의 컨테이너를 모두 제거할 수 있다.
- 요청 문자열의 길이가 1이면 지게차를 사용하므로, 바깥과 연결된 통로를 통해 접근 가능한 해당 종류의 컨테이너만 제거할 수 있다.
- 모든 요청을 처리한 뒤, 창고에 남아 있는 컨테이너 수를 반환한다.

### 입력
- `storage`: 창고 상태를 나타내는 문자열 배열
- `requests`: 작업 요청 문자열 배열

### 출력
- 모든 요청 처리 후 남은 컨테이너 개수

---

### 핵심 풀이 전략
- `storage`를 2차원 문자 배열 `table`로 변환해 상태를 관리한다.
- 요청을 하나씩 처리한다.
- 요청 길이가 2인 경우:
  - 크레인 사용이므로 해당 문자를 가진 모든 칸을 즉시 제거한다.
- 요청 길이가 1인 경우:
  - 해당 문자를 가진 칸마다, 현재 위치가 외부와 연결된 빈 공간을 통해 접근 가능한지 확인한다.
  - 접근 가능한 칸만 따로 표시해 두었다가 한 번에 제거한다.
- 최종적으로 전체 칸 수에서 제거된 개수를 빼서 남은 컨테이너 수를 구한다.

---

### 제출 코드
```java
import java.util.*;
class Solution {
    public int solution(String[] storage, String[] requests) {
        int count = 0;
        char[][] table = new char[storage.length][storage[0].length()];
        for(int i = 0 ; i < storage.length; i++){
            for(int j = 0 ; j < storage[0].length(); j++){
                table[i][j] = storage[i].charAt(j);
            }
        }
        
        for(int i = 0 ; i < requests.length; i++){
            char request = requests[i].charAt(0);
            
            // 두 자리
            if(requests[i].length() == 2){
                for(int j = 0; j < table.length; j++){
                    for(int k = 0; k < table[0].length; k++){
                        if(table[j][k] == request){
                            count++;
                            table[j][k] = '.';
                        }
                    }
                }
            }
            
            // 한 자리
            else{
                boolean[][] checkTable = new boolean[table.length][table[0].length];
                for(int j = 0; j < table.length; j++){
                    for(int k = 0 ; k < table[0].length; k++){
                        if(table[j][k] == request){
                            if(check(table, j, k)){
                                count++;
                                checkTable[j][k] = true;
                            }
                        }
                    }
                }
                
                for(int j = 0 ; j < table.length; j++){
                    for(int k = 0; k < table[0].length; k++){
                        if(checkTable[j][k] == true) table[j][k] = '.';
                    }
                }
            }
        }   
        return table.length * table[0].length - count;
    }
    
    boolean check(char[][] table, int j, int k){
        if(j == 0 || k == 0 || j == table.length - 1 || k == table[0].length - 1) return true;
        int[] dx = {1, -1, 0, 0};
        int[] dy = {0, 0, 1, -1};
        boolean[][] visited = new boolean[table.length][table[0].length];
        Queue<int[]> queue = new LinkedList<>();
        queue.offer(new int[]{j, k});
        visited[j][k] = true;
        while(!queue.isEmpty()){
            int[] cur = queue.poll();
            if(cur[0] == 0 || cur[1] == 0 || cur[0] == table.length -1 || cur[1] == table[0].length -1) return true;
            for(int i = 0 ; i < 4; i++){
                int nx = cur[0] + dx[i];
                int ny = cur[1] + dy[i];
                if(nx >= 0 && ny >= 0 && nx < table.length && ny < table[0].length){
                    if(visited[nx][ny] == false && table[nx][ny] == '.'){
                        queue.offer(new int[] {nx, ny});
                        visited[nx][ny] = true;
                    }
                }
            }
        }
        return false;
    }   
}
```

### 삽질 기록🧠
1. 처음에는 BFS로 풀어야한다고 생각을 못해서, 4방향 중에서 빈칸이 있으면 빼내는 방식을 사용했다가 틀렸다. 
2. 테스트 케이스를 보고 BFS로 풀어야한다는 생각을 했다. 
3. BFS를 풀면서, while(!queue.isEmpty())를 실수로 while(queue.isEmpty())로 적어서 어디가 틀린지 계속 헤맸다..

### 배운 점 && 보완할 점
- 너무 비효율 적인 풀이가 아닌가? 라는 생각이 들어서 GPT한테 물어보니, 바깥 패딩 방식을 사용하는게 효율적이라는 답변을 받았다. 
```
....
.AB.
.CD.
....
```
- 이런 형태로 문제를 푸는 초기에 바깥에 패딩을 깔아둔다.
- 이후, 패딩된 배열에서 `(0,0)`부터 BFS를 진행한다.
- 외부 공기와 연결된 모든 `'.'`을 방문한다.
- 상하좌우를 보면서 요청 문자가 있으면 제거 대상으로 표시한다.
