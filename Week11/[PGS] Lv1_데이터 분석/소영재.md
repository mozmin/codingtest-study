### 문제 설명 (요약)
- 주어진 2차원 정수 배열 data에서 특정 열(ext)의 값이 val_ext보다 작은 행만 추려낸 뒤, sort_by를 기준으로 오름차순 정렬하여 2차원 배열 형태로 반환하는 문제

### 입력
- int[][] data : 여러 개의 데이터 행이 담긴 2차원 정수 배열 [code, date, maximum, remain]
- String ext : 필터링 기준이 되는 열 이름
- int val_ext : 필터링 기준 값
- String sort_by : 정렬 기준이 되는 열 이름

### 출력
- ext에 해당하는 값이 val_ext보다 작은 행들만 골라 sort_by 기준으로 오름차순 정렬한 2차원 정수 배열 반환

<hr>

### 핵심 풀이 전략
1. ext가 어떤 열을 의미하는지 문자열 비교를 통해 인덱스로 변환한다.
2. data를 순회하면서 해당 열의 값이 val_ext보다 작은 행만 List에 담는다.
3. sort_by도 동일하게 인덱스로 변환한 뒤, 해당 열 기준으로 오름차순으로 정렬한다.
4. List를 int[][]형태로 변환하여 반환한다.

---

```java
import java.util.*;

class Solution {
    public int[][] solution(int[][] data, String ext, int val_ext, String sort_by) {
        
        int condition = 0;
        int sort_Idx = 0;
        
        if(ext.equals("code")){
            condition = 0;
        } else if(ext.equals("date")){
            condition = 1;
        } else if(ext.equals("maximum")){
            condition = 2;
        } else {
            condition = 3;
        }
        
        List<int[]> list = new ArrayList<>();
        
        for(int i = 0; i < data.length; i++){
            if(data[i][condition] < val_ext){
                list.add(data[i]);
            }
        }
        
         if(sort_by.equals("code")){
            sort_Idx = 0;
        } else if(sort_by.equals("date")){
            sort_Idx = 1;
        } else if(sort_by.equals("maximum")){
            sort_Idx = 2;
        } else {
            sort_Idx = 3;
        }
        
        final int idx = sort_Idx;
        
        list.sort((a, b) -> Integer.compare(a[idx], b[idx]));

        return list.toArray(new int[list.size()][]);   
        
    }
}
```

### 배운 점 && 보완할 점
- 마지막 List -> 2차원 정수 배열로 변환하는 과정과 람다식의 조건 변수에서 에러를 겪었다.
- 먼저 람다 안에서는 "변하지 않는 값"(effectively final)만 사용할 수 있다는 점을 배웠다.
- 따라서 값이 변경된 변수를 그대로 사용하는 것이 아니라, final 변수에 한 번 더 할당하여 고정된 값으로 만들어 사용해야 한다는 것을 배웠다.
- 또한 list.toArray(new int[list.size()][])를 통해 리스트에 담긴 1차원 배열들을 그대로 2차원 배열로 변환하는 방식도 배우게 되었다.
- 아쉬운 점으로는 리스트를 정렬할 기준, data 2차원 배열 내의 비교 값을 선택하는 기준을 if문으로 작성했는데, Map<String, Integer>구조를 사용하면 굳이 조건문을 2번 작성하지 않아도 됐을 것 같다.
