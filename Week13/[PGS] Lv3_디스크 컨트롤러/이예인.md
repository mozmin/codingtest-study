### 문제 설명
작업마다 요청 시간과 작업 소요 시간이 주어진다.<br>
하드디스크가 한 번에 하나의 작업만 처리할 수 있을 때, 각 작업의 요청부터 종료까지 걸린 시간의 평균을 최소로 만들어야 한다.<br>
작업의 반환 시간은 다음과 같이 계산한다.<br>
반환 시간 = 작업 완료 시간 - 작업 요청 시간<br>
모든 작업의 반환 시간 평균을 구해 반환한다.<br>

### 입력
- jobs: 각 작업의 요청 시간과 소요 시간을 담은 2차원 배열
- jobs[i][0]: 작업 요청 시간
- jobs[i][1]: 작업 소요 시간

### 출력
- 작업 요청부터 종료까지 걸린 시간의 평균의 최솟값

---

### 핵심 풀이 전략
- 먼저 jobs 배열을 요청 시간 기준으로 정렬한다.
- 현재 시간 t까지 요청된 작업만 PriorityQueue에 넣는다.
- PriorityQueue는 작업 소요 시간이 짧은 순서로 정렬한다.
- 현재 처리 가능한 작업 중 가장 짧은 작업을 먼저 처리하면 평균 반환 시간을 줄일 수 있다.
- 처리 가능한 작업이 없다면 현재 시간을 다음 작업의 요청 시간으로 이동시킨다.
- 작업을 하나 처리할 때마다 완료 시간 t를 갱신하고, 반환 시간인 `t - 요청시간`을 answer에 더한다.

---

### 제출 코드
```java
import java.util.*;

class Solution {
    public int solution(int[][] jobs) {
        int answer = 0;
        int t = 0;
        int idx = 0;
        int count = 0;

        // 요청 시간 기준 정렬
        Arrays.sort(jobs, (a, b) -> {
            if (a[0] == b[0]) {
                return Integer.compare(a[1], b[1]);
            }
            return Integer.compare(a[0], b[0]);
        });

        // 소요 시간이 짧은 순서
        PriorityQueue<int[]> pq = new PriorityQueue<>(
            (a, b) -> Integer.compare(a[1], b[1])
        );

        while (count < jobs.length) {

            // 현재 시간 t까지 요청된 작업들을 pq에 넣음
            while (idx < jobs.length && jobs[idx][0] <= t) {
                pq.offer(jobs[idx]);
                idx++;
            }

            if (!pq.isEmpty()) {
                int[] job = pq.poll();

                int start = job[0]; // 요청 시간
                int time = job[1];  // 작업 소요 시간

                t += time;          // 현재 시간 갱신
                answer += t - start; // 반환 시간 = 완료 시간 - 요청 시간

                count++;
            } else {
                // 현재 처리 가능한 작업이 없으면 다음 작업 요청 시간으로 이동
                t = jobs[idx][0];
            }
        }

        return answer / jobs.length;
    }
}
```

### 삽길 기록 🧠
처음에는 PriorityQueue를 요청 시간 기준으로 정렬해서 모든 작업을 한 번에 넣고 처리하려고 했다.<br>
하지만 이렇게 하면 아직 요청되지 않은 작업까지 먼저 처리될 수 있어서 문제 조건에 맞지 않았다.<br>
<br>
또한 반환 시간을 계산할 때 `start - t + time`처럼 계산했는데, 이는 올바른 방식이 아니었다.<br>
작업의 반환 시간은 작업이 끝난 시점에서 요청 시간을 뺀 값이므로 `t += time`으로 완료 시간을 갱신한 뒤 `answer += t - start`로 계산해야 한다.<br>
<br>
`idx`와 `count`의 역할도 처음에는 헷갈렸다.<br>
`idx`는 아직 PriorityQueue에 넣지 않은 jobs 배열의 위치를 의미하고, `count`는 실제로 처리 완료한 작업 개수를 의미한다.<br>

### 배운 점 && 보완할 점 
- `PriorityQueue<int[]>`를 사용할 때도 배열의 어떤 인덱스를 기준으로 정렬할지 Comparator를 작성해야 한다.<br>
