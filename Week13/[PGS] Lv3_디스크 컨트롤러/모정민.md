`#우선순위큐` `#정렬` `#스케줄링`

### 제출 일자
2026년 5월 6일 12.13ms

### 문제 설명 (요약)
하드디스크가 작업을 처리하는 순서를 최적화하여, 모든 작업의 '요청부터 종료까지 걸린 평균 시간(반환 시간)'을 최소화하는 문제. 

- **jobs**: `[작업이 요청되는 시점, 작업의 소요시간]`을 원소로 가진 2차원 정수 배열
- **조건**: 하드디스크는 한 번에 하나의 작업만 처리할 수 있다. 평균 시간의 소수점 이하는 버린다.

---

### 핵심 풀이 전략
1. **SJF(Shortest Job First) 스케줄링**: 대기 중인 작업들 중 **소요 시간이 가장 짧은 작업**을 먼저 처리해야 전체 평균 반환 시간을 최소화할 수 있다.
2. 입력 배열 `jobs`를 작업의 **요청 시점**을 기준으로 오름차순 정렬한다.
3. 현재 시간(`time`)을 기준으로, 요청 시점이 도달한 모든 작업을 우선순위 큐(소요 시간이 짧은 순으로 정렬)에 삽입한다.
4. 우선순위 큐에서 작업을 하나씩 꺼내 처리하며 현재 시간과 누적 반환 시간을 업데이트한다. 처리 중 새로운 작업이 요청 시간을 넘기면 다시 큐에 넣는 과정을 모든 작업이 완료될 때까지 반복한다.

---

### 내가 구현한 구현 코드
```java
import java.util.*;

class Solution {
    public int solution(int[][] jobs) {
        ArrayList<Integer> totalTime = new ArrayList<>();
        
        Arrays.sort(jobs, Comparator.comparingInt((int[] a) -> a[0]));
            
        // jobs[] {작업이 요청되는 시점, 작업의 소요시간}
        // que -> int[] {작업 소요시간, 작업 요청시간, 작업 번호}
        PriorityQueue<int[]> que = new PriorityQueue<>(
        Comparator.comparingInt((int[] a) -> a[0])
            .thenComparingInt((int[] a) -> a[1])
            .thenComparingInt((int[] a) -> a[2])
        );
        
        int time = 0;
        int index = 0;
        int count = 0;
        while(count < jobs.length){
            
            // 현재 시간 전에 들어온 모든 요청 큐에 삽입
            // 작업 큐에 삽입
            for(int i = index; i < jobs.length; i++){
                
                if(jobs[i][0] <= time){
                    que.offer(new int[]{jobs[i][1], jobs[i][0], i});
                    index++;
                }
            }
            
            // 만약 작업할게 없다면
            // 다음 작업으로 이동
            if(que.isEmpty()){
                time = jobs[index][0];
            }else{
                int[] poll = que.poll();
                time += poll[0];

                // 총 작업시간 저장
                int takenTime = time - poll[1];
                totalTime.add(takenTime);   
                count++; 
            }
        }
        
        int totalValue = 0;
        
        for(int i = 0; i < totalTime.size(); i++){
            totalValue += totalTime.get(i);
        }

        return totalValue / jobs.length;
    }
}
```

### 정답 코드
1. **공간 복잡도 최적화**: 반환 시간의 평균을 구하기 위해 굳이 개별 작업의 반환 시간을 `ArrayList`에 모두 보관할 필요가 없다. `totalResponseTime`이라는 하나의 변수에 값을 누적시키는 방식으로 메모리와 시간을 절약할 수 있다.
2. **우선순위 큐 삽입 로직 간소화**: 큐에 데이터를 넣을 때 순서를 뒤집어 새로운 배열을 생성(`new int[]{jobs[i][1], jobs[i][0], i}`)하는 대신, 원본 `jobs`의 배열 형태 그대로 큐에 넣고 정렬 조건만 소요 시간(`a[1]`) 기준으로 맞춰주면 코드가 훨씬 간결해진다.
```java
import java.util.*;

class Solution {
    public int solution(int[][] jobs) {
        // 1. 요청 시간 순으로 정렬 (람다식으로 간결하게 표현)
        Arrays.sort(jobs, (a, b) -> a[0] - b[0]);

        // 2. 소요 시간 순으로 우선순위 큐 설정 (SJF 핵심)
        PriorityQueue<int[]> que = new PriorityQueue<>((a, b) -> a[1] - b[1]);

        int totalResponseTime = 0; // ArrayList 대신 정수형 변수에 누적
        int time = 0;
        int index = 0;
        int count = 0;

        while (count < jobs.length) {
            
            // 현재 시간 이하에 요청된 작업을 모두 큐에 삽입 (for문 대신 while문으로 깔끔하게 처리)
            while (index < jobs.length && jobs[index][0] <= time) {
                que.offer(jobs[index]);
                index++;
            }

            // 처리할 작업이 없다면 다음 작업의 요청 시점으로 시간 점프 (Idle Time 처리)
            if (que.isEmpty()) {
                time = jobs[index][0];
            } else {
                // 소요 시간이 가장 짧은 작업 수행
                int[] job = que.poll();
                time += job[1];  // 시간 경과 (소요 시간만큼)
                
                // 총 작업 시간 누적 (현재 종료 시간 - 최초 요청 시간)
                totalResponseTime += time - job[0];
                count++;
            }
        }

        // 전체 누적 시간을 작업 개수로 나누어 평균 반환
        return totalResponseTime / jobs.length;
    }
}
```

### 삽질 기록 🧠
- 초기 풀이에서 `jobs` 배열이 요청 시간 순으로 정렬되어 있을 것이라 착각하여 `Arrays.sort`를 누락했었다. 이로 인해 작업이 순차적으로 큐에 들어가지 않는 문제가 발생했다.
  - 즉, 1번~n번 작업이 모두 시간순대로 jobs 배열에 들어있지 않다.
  - `심지어 작업 번호는 문제 결과에 영향을 주지 않기 때문에 배제`해도 된다.
- `time`을 1씩 증가시키는 `time++` 방식을 사용했을 때, 요청이 없는 빈 시간(Idle Time) 구간이 길 경우 불필요한 반복문이 수없이 돌아가며 시간 초과가 발생했었다.
- 큐에 있는 모든 작업을 한 번에 처리하려 다가(`while(!que.isEmpty())`), 긴 작업을 처리하는 동안 새로 요청된 짧은 작업이 우선순위를 배정받지 못하게 되는 실수를 했었다.

### 배운 점 && 보완할 점
- 평균이나 총합만을 요구하는 문제에서는 `ArrayList`에 `모든 데이터를 담아 마지막에 순회하며 더하는 것보다`, `변수 하나에 즉시 누적`(`+=`)시키는 방식이 메모리 최적화와 가독성 면에서 훨씬 우수하다.
- 데이터를 다룰 때 새롭게 배열을 생성해(`new int[]{...}`) 구조를 바꾸기보다는, 원본 구조를 유지하되 `Comparator`의 기준점만 변경(`(a, b) -> a[1] - b[1]`)하는 것이 코드를 훨씬 깔끔하고 헷갈리지 않게 만들어준다.
- `while`문 내부에서 `index`를 제어할 때, 굳이 `for`문을 쓰지 않고 조건식(`index < jobs.length && jobs[index][0] <= time`)을 잘 활용하면 인덱스 관리 로직이 더욱 직관적이게 된다.
