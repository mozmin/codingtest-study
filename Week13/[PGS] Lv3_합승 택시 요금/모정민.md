### 문제 설명 (요약)
출발지 S에서 A와 B가 각각의 목적지까지 택시를 타고 갈 때, 함께 합승하여 갈 수 있는 구간은 최대한 합승하여 전체 택시 요금을 최소화하는 문제. 

- **fares**: `[지점, 지점, 예상 택시 요금]`을 원소로 가진 2차원 정수 배열
- **조건**: 노드 개수 n은 3 이상 200 이하. 요금은 방향성이 없는 무방향 그래프(양방향 통행) 형태.

---

### 핵심 풀이 전략
1. 특정 지점 `i`까지 두 사람이 합승한 뒤, `i`에서부터 A와 B가 각자 목적지로 흩어지는 경우의 수를 모두 탐색한다. (비용 산출 공식: `S -> i 비용` + `i -> A 비용` + `i -> B 비용`)
2. 노드의 개수가 최대 200개로 작으므로, **플로이드-워셜** 알고리즘(3중 for문)을 사용해 모든 노드 간의 최단 거리를 한 번에 구하거나,
3. 출발지 `S`, A의 목적지 `A`, B의 목적지 `B`를 시작점으로 하는 **다익스트라**를 딱 3번만 수행하여 최단 거리 배열을 미리 완성해둔다.
4. 반복문을 돌며 임의의 분기점 `i`에 대한 총합 비용이 가장 작은 값을 갱신하여 도출한다. (이때 `i`가 S, A, B 중 하나인 경우도 자연스럽게 포함된다.)

### 내가 구현한 구현 코드 (오답 코드)
```java
import java.util.*;

class Solution {
    int totalFee;
    public int solution(int n, int s, int a, int b, int[][] fares) {
        totalFee = Integer.MAX_VALUE;
        
        // 배열 초기화
        ArrayList<int[]>[] load = new ArrayList[n+1];
        for(int i = 1; i <= n; i++) load[i] = new ArrayList<>();
        
        // 요금 초기화
        for(int[] fare : fares){
            int to = fare[0];
            int from = fare[1];
            int fee = fare[2];
            
            load[to].add(new int[]{from, fee});
            load[from].add(new int[]{to, fee});
        }
        
        // 모든 노드 탐색
        for(int i = 0; i < n; i++){ // 🚨 오류: 1부터 n까지 돌아야 함
            boolean[] visited = new boolean[n+1];
            visited[s] = true;
            
            // start -> i 까지 가는 최단 경로
            int allFee = bfs(s, i, load, visited);
            int toA = bfs(i, a, load, visited);
            int toB = bfs(i, b, load, visited);
            
            int value = allFee + toA + toB;
            
            totalFee = Math.min(totalFee, value);
        }
        
        return totalFee;
    }
    
    // 공통 -> A or B 까지 가는 최단 경로
    public int bfs(int start, int end, ArrayList<int[]>[] load, boolean[] visited){
        
        boolean[] copy = new boolean[visited.length];
        for(int i = 0; i < visited.length; i++){
            if(visited[i]) copy[i] = true;
        }
        
        // int[] {방문 노드, 방문 비용}
        PriorityQueue<int[]> que = new PriorityQueue<>(Comparator.comparingInt((int[] arr) -> arr[1]));
        que.offer(new int[]{start, 0});
        
        int curTotalFee = 0; // 🚨 치명적 오류: 갈래길의 요금들이 모두 이곳에 누적 합산됨 -> dist[]를 써야 함.
        while(!que.isEmpty()){
            
            int[] poll = que.poll();
            int node = poll[0];
            int fee = poll[1];
            
            curTotalFee += fee; 
            
            if(node == end) return curTotalFee;
            
            // 인접 노드 중 방문하지 않은 노드 방문
            // 방문 비용이 적은 노드 먼저 방문
            for(int i = 0; i < load[node].size(); i++){
                
                int[] next = load[node].get(i);
                int nextNode = next[0];
                int nextFee = next[1];
                
                if(!copy[nextNode]){
                    que.offer(new int[]{nextNode, nextFee});
                    copy[nextNode] = true;
                }
            }
        }
        
        return 0;
    }
}
```

### 정답 코드 1 (플로이드-워셜 / N이 200 이하일 때 강력 추천)
노드 개수 N이 작을 때 가장 간결하고 빠르게 풀 수 있는 3중 for문 기반의 O(N^3) 접근법.
```java
import java.util.*;

class Solution {
    static final int INF = 20000000; // 최대 요금(10만) * 최대 노드(200)로 넉넉하게 무한대 설정 (오버플로우 방지)

    public int solution(int n, int s, int a, int b, int[][] fares) {
        int[][] dist = new int[n + 1][n + 1];
        
        // 1. 인접 행렬 초기화
        for (int i = 1; i <= n; i++) {
            Arrays.fill(dist[i], INF);
            dist[i][i] = 0; // 자기 자신으로 가는 비용은 0
        }
        
        for (int[] fare : fares) {
            dist[fare[0]][fare[1]] = fare[2];
            dist[fare[1]][fare[0]] = fare[2];
        }
        
        // 2. 플로이드-워셜 핵심 알고리즘 (k: 거쳐가는 노드, i: 출발 노드, j: 도착 노드)
        for (int k = 1; k <= n; k++) {
            for (int i = 1; i <= n; i++) {
                for (int j = 1; j <= n; j++) {
                    if (dist[i][j] > dist[i][k] + dist[k][j]) {
                        dist[i][j] = dist[i][k] + dist[k][j];
                    }
                }
            }
        }
        
        // 3. 최적의 환승 지점 탐색 (i 지점에서 헤어지는 모든 경우의 수 중 최솟값)
        int totalFee = INF;
        for (int i = 1; i <= n; i++) {
            totalFee = Math.min(totalFee, dist[s][i] + dist[i][a] + dist[i][b]);
        }
        
        return totalFee;
    }
}
```

### 정답 코드 2 (다익스트라 최적화)
N이 1000 이상으로 클 경우 플로이드-워셜은 시간초과가 발생하므로, 다익스트라를 필요한 곳(S, A, B)에서만 딱 3번 호출하는 것이 최적이다.
```java
import java.util.*;

class Solution {
    public int solution(int n, int s, int a, int b, int[][] fares) {
        ArrayList<Node>[] graph = new ArrayList[n + 1];
        for (int i = 1; i <= n; i++) graph[i] = new ArrayList<>();
        
        for (int[] fare : fares) {
            graph[fare[0]].add(new Node(fare[1], fare[2]));
            graph[fare[1]].add(new Node(fare[0], fare[2]));
        }
        
        // S, A, B를 시작점으로 하는 다익스트라 (딱 3번만 실행)
        int[] distS = dijkstra(s, graph, n);
        int[] distA = dijkstra(a, graph, n);
        int[] distB = dijkstra(b, graph, n);
        
        int totalFee = Integer.MAX_VALUE;
        // 모든 분기점 i 탐색
        for (int i = 1; i <= n; i++) {
            if (distS[i] == Integer.MAX_VALUE || distA[i] == Integer.MAX_VALUE || distB[i] == Integer.MAX_VALUE) continue;
            totalFee = Math.min(totalFee, distS[i] + distA[i] + distB[i]);
        }
        
        return totalFee;
    }
    
    public int[] dijkstra(int start, ArrayList<Node>[] graph, int n) {
        int[] dist = new int[n + 1];
        Arrays.fill(dist, Integer.MAX_VALUE);
        dist[start] = 0;
        
        PriorityQueue<Node> pq = new PriorityQueue<>(Comparator.comparingInt(node -> node.cost));
        pq.offer(new Node(start, 0));
        
        while (!pq.isEmpty()) {
            Node curr = pq.poll();
            if (dist[curr.idx] < curr.cost) continue; // 비효율적 경로 필터링
            
            for (Node next : graph[curr.idx]) {
                if (dist[next.idx] > curr.cost + next.cost) {
                    dist[next.idx] = curr.cost + next.cost; // 최단 거리 갱신
                    pq.offer(new Node(next.idx, dist[next.idx]));
                }
            }
        }
        return dist;
    }
    
    // 독립적이고 가벼운 객체 생성을 위한 static 내부 클래스
    static class Node {
        int idx, cost;
        Node(int idx, int cost) {
            this.idx = idx; this.cost = cost;
        }
    }
}
```

### 삽질 기록 🧠
- B로 가는 최단 경로를 구하는 `toB` 변수 계산 시, 목적지로 `b`가 아닌 `a`를 넘겨주는 치명적인 오타(`bfs(i, a...)`)가 있었다.
- 최단 거리 탐색(`bfs`)을 반복문 안에 넣어 모든 노드 `i`마다 매번 3번씩 반복 호출하는 치명적인 구조로 작성하여 시간 효율성이 매우 떨어졌다. (다익스트라는 한 번만 돌리면 모든 노드까지의 최단 거리를 알 수 있다는 점을 간과함)
- 다익스트라(우선순위 큐)를 흉내 내었으나, 큐에서 경로를 뽑을 때마다 요금을 별도 변수에 단순 누적합(`curTotalFee += fee`) 해버려서, 다른 갈래길을 탐색할 때 발생하는 요금까지 전부 하나로 섞여버리는 심각한 논리적 오류가 발생했다. -> 이는 dist[]를 사용하여 dist[] 값보다 작은지 확인 후 갱신하는 방법을 써야했다.

### 배운 점 && 보완할 점
- **플로이드-워셜의 위력:** `N`이 400 이하로 작고 "모든 노드 간의 거리"가 필요할 때는 복잡한 큐 구현 없이 3중 for문(플로이드-워셜)만으로 훨씬 간결하게 풀 수 있음을 명심하자. `O(N^3)`의 시간 복잡도도 N=200이면 800만 번 연산이므로 1초 내에 충분히 통과한다.
- **다익스트라의 본질:** 다익스트라는 단일 도착지가 아니라 '시작점에서 다른 모든 노드까지의 최단 거리'를 한 번에 배열로 계산해 준다는 점을 잊지 말자. N이 커서 플로이드-워셜을 못 쓸 때는 필요한 시작점만 골라서 밖에서 미리 돌려두는 것이 핵심이다.
- 큐 값: 지금은 int[]를 큐에 넣는데, 이는 매번 `new int[]`로 새로운 값을 만들어낸다. class Node를 사용하는 것이 메모리 효율면에서 더 좋다.
- **경로 누적 합산 방식:** 큐를 이용한 최단 거리 탐색 시 별도의 누적 변수를 쓰면 안 되고, 큐에 넣는 객체 자체에 '시작점부터 현재까지의 누적 비용'을 함께 담아서 넘겨줘야 독립적인 경로 계산이 가능하다.
