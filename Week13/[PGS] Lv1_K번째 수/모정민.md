### 문제 설명 (요약)
배열 `array`의 `i`번째 숫자부터 `j`번째 숫자까지 자르고 정렬했을 때, `k`번째에 있는 수를 구하는 문제.

- **array**: 1차원 정수 배열 (길이 1 이상 100 이하)
- **commands**: `[i, j, k]`를 원소로 가진 2차원 정수 배열 (길이 1 이상 50 이하)
- **조건**: 
  - 배열의 인덱스는 0이 아닌 1부터 시작하는 것으로 간주하여 계산해야 한다.

---

### 핵심 풀이 전략
1. `commands` 배열을 순회하며 주어진 `[i, j, k]` 값에 따라 배열의 특정 범위를 자른다.
2. 문제의 조건은 1-based 인덱스이므로, 자바의 0-based 인덱스에 맞게 `i-1` 등의 변환을 거쳐야 한다.
3. 잘라낸 부분 배열을 `Arrays.sort()`를 이용해 오름차순으로 정렬한다.
4. 정렬된 배열에서 `k-1` 위치에 있는 값을 찾아 정답 배열에 순서대로 담아 반환한다.

---

### 내가 구현한 구현 코드
```java
import java.util.*;

class Solution {
    public int[] solution(int[] array, int[][] commands) {
        ArrayList<Integer> result = new ArrayList<>();
        
        for(int[] command: commands){
            int[] cut = new int[command[1] - command[0] + 1];
            for(int a = command[0]-1; a <= command[1]-1; a++) cut[a-(command[0]-1)] = array[a];
            
            Arrays.sort(cut);
            result.add(cut[command[2]-1]);
        }
        
        return result.stream().mapToInt(Integer::intValue).toArray();
    }
}
```

### 정답 코드
1. **`Arrays.copyOfRange` 활용**: `for`문을 이용해 수동으로 배열을 복사하고 인덱스를 계산하는 대신, 자바 내장 메서드를 사용하여 가독성과 안정성을 높인다.
2. **배열 크기 사전 할당**: 반환해야 할 정답 배열의 크기는 결국 `commands`의 길이와 동일하다. `ArrayList`를 사용하고 마지막에 `stream`으로 변환하는 오버헤드를 줄이기 위해, 처음부터 크기가 정해진 `int[]`를 생성하여 사용한다.
```java
import java.util.Arrays;

class Solution {
    public int[] solution(int[] array, int[][] commands) {
        // 결과 배열의 크기는 commands 배열의 길이와 동일함
        int[] answer = new int[commands.length];

        for (int i = 0; i < commands.length; i++) {
            // 1-based index를 0-based index에 맞게 변환
            int start = commands[i][0] - 1; 
            int end = commands[i][1];       
            int k = commands[i][2] - 1;     

            // 1. Arrays.copyOfRange로 배열 자르기 (시작 인덱스 포함, 끝 인덱스 미포함)
            int[] temp = Arrays.copyOfRange(array, start, end);

            // 2. 배열 정렬하기
            Arrays.sort(temp);

            // 3. k번째 수 구해서 정답 배열에 저장
            answer[i] = temp[k];
        }

        return answer;
    }
}
```

### 삽질 기록 🧠
- result.add(cut[command[2]-1]); 에서 처음에는 `-1`를 하지 않았다. 인덱스 번호를 신경쓰지 않았다.

### 배운 점 && 보완할 점
- 특정 범위의 배열을 복사할 때는 직접 `for`문을 돌려 복잡하게 인덱스를 계산하는 것보다, 자바에서 제공하는 `Arrays.copyOfRange(array, start, end)` 메서드를 적극 활용하자. 코드가 훨씬 직관적이게 되고 인덱스 계산 실수(Off-by-one error)를 방지할 수 있다.
- 길이를 미리 알 수 있는 결과값임에도 `ArrayList`에 값을 담은 뒤 `stream().mapToInt(Integer::intValue).toArray()`를 사용하여 원시 타입 배열로 변환했다. 이로 인해 불필요한 객체 생성과 스트림 연산에 따른 시간 오버헤드가 발생했다.
