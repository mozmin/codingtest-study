### 제출 일자
2026년 5월 3일 8.17ms

### 문제 설명 (요약)
스마트폰 전화 키패드의 각 칸을 누를 때, 왼손과 오른손 중 어떤 손가락을 사용해야 하는지 순서대로 문자열 형태로 반환하는 문제.

- **numbers**: 누를 번호가 담긴 배열 (1 이상 1,000 이하의 크기, 각 원소는 0~9)
- **hand**: 주로 사용하는 손 ("left" 또는 "right")
- **조건**: 
  - 왼쪽 열(1, 4, 7)은 무조건 왼손 엄지 사용
  - 오른쪽 열(3, 6, 9)은 무조건 오른손 엄지 사용
  - 가운데 열(2, 5, 8, 0)은 두 엄지손가락 중 현재 키패드의 위치에서 더 가까운 손가락 사용
  - 거리가 같을 경우 `hand`에 주어진 주 사용 손가락 사용
  - 처음 시작 위치는 왼손은 `*`, 오른손은 `#`

---

### 핵심 풀이 전략
1. 키패드 각 숫자와 기호(`*`, `#`)에 2차원 좌표(y, x)를 부여하여 HashMap에 저장한다. (숫자 0은 `*`과 `#` 사이인 11번째 위치에 해당)
2. 시작할 때 왼손과 오른손의 초기 좌표를 각각 `*`와 `#`의 좌표로 설정한다.
3. 주어진 `numbers` 배열을 순회하며 다음 과정을 반복한다.
   - `1, 4, 7`이면 무조건 결과에 "L"을 추가하고 왼손의 좌표를 해당 숫자의 좌표로 갱신한다.
   - `3, 6, 9`이면 무조건 결과에 "R"을 추가하고 오른손의 좌표를 해당 숫자의 좌표로 갱신한다.
   - `2, 5, 8, 0`이면 목표 숫자의 좌표와 현재 두 손가락 위치 간의 '맨해튼 거리(|y1-y2| + |x1-x2|)'를 각각 계산한다.
   - 거리가 짧은 쪽의 손가락을 사용하고, 거리가 같다면 초기 파라미터 `hand`를 기준으로 결정한다. 이후 사용한 손가락의 좌표를 갱신한다.

---

### 내가 구현한 구현 코드 (오답 - 삽질 기록 참고)
```java
import java.util.*;

class Solution {
    
    int[] dy = {0, 0, 1, -1};
    int[] dx = {1, -1, 0, 0};
    public String solution(int[] numbers, String hand) {
        ArrayList<String> result = new ArrayList<>();
        
        HashMap<Integer, int[]> map = new HashMap<>();
        
        String defaultHand = hand.equals("right") ? "R" : "L";
        
        int index = 1;
        for(int i = 0; i < 4; i++){
            for(int j = 0; j < 3; j++){
                if(index == 11) map.put(0, new int[]{i, j});
                else{
                    map.put(index, new int[]{i, j});
                }
                index++;
            }
        }
        
        int[] left = new int[]{3, 0};
        int[] right = new int[]{3, 2};
        for(int number : numbers){
            
            if(number == 1 || number == 4 || number == 7){
                result.add("L");
                int[] pos = map.get(number);
                left[0] = pos[0];
                left[1] = pos[1];
            }
            else if(number == 3 || number == 6 || number == 9){
                result.add("R");
                int[] pos = map.get(number);
                right[0] = pos[0];
                right[1] = pos[1];
            }
            else{
                int[] targetPos = map.get(number);
                
                String touch = "";
                if(bfs(left, targetPos[0], targetPos[1]) == bfs(right, targetPos[0], targetPos[1])){
                    result.add(defaultHand);
                    touch = defaultHand;
                }
                else{
                    String value = bfs(left, targetPos[0], targetPos[1]) < bfs(right, targetPos[0], targetPos[1]) ? "L" : "R";
                    result.add(value);
                    touch = value;
                }
                
                if(touch.equals("L")){
                    left[0] = targetPos[0];
                    left[1] = targetPos[1];
                }else{
                    right[0] = targetPos[0];
                    right[1] = targetPos[1];
                }
            }
        }
        
        String[] resultList = result.stream().map(i -> i).toArray(String[]::new);
        
        return String.join("", resultList);
    }
    
    public int bfs(int[] pos, int targetY, int targetX){
        boolean[][] keyboard = new boolean[4][3];
        ArrayDeque<int[]> que = new ArrayDeque<>();
        que.offer(new int[]{pos[0], pos[1], 0});
        
        while(!que.isEmpty()){
        
            int[] poll = que.poll();
            
            if(poll[0] == targetY && poll[1] == targetX) return poll[2];
            
            for(int i = 0; i < 4; i++){
                
                int nY = poll[0] + dy[i];
                int nX = poll[1] + dx[i];
                
                if(nY >= 0 && nY < 4 && nX >= 0 && nX < 3 && !keyboard[nY][nX]){
                    que.offer(new int[]{nY, nX, poll[2] + 1});
                    keyboard[nY][nX] = true;
                }            
            }
        }
        
        return 0;
    }
}
```
### 삽질 기록 🧠
- number가 0일 때의 예외 처리를 누락하여 초기에 NullPointerException이 발생했다. (키패드 좌표를 생성할 때 index가 11이 되는 순간에 숫자 0을 매핑해 주어야 한다.)
- 초기 키패드 좌표 초기화 시 if(index == 11) map.put(...)을 수행한 후 index++를 누락하여 0과 #의 위치가 꼬이는 논리 오류를 범했다.
- else 블록(가운데 열 2, 5, 8, 0 처리) 안에서 손가락이 키패드를 누른 후 해당 위치로 좌표를 갱신하지 않아 오답이 발생했다.
- 자바에서 매개변수로 주어진 hand 문자열을 비교할 때 == 연산자를 사용하여 의도하지 않은 결과가 나왔다. (.equals()를 사용해야 함)
- 목표 지점까지의 최단 거리를 구하기 위해 BFS를 사용했지만, 큐에서 값을 뺄 때마다 카운팅을 하여 '거리'가 아닌 '총 탐색 칸 수'를 반환하는 로직을 작성해 오답이 나왔다.

### 배운 점 && 보완할 점
- **보완할 점**: 2차원 배열 등에서 상하좌우로 단순 이동하는 거리(장애물이 없는 경우)를 구할 때는 복잡한 BFS/DFS가 아니라 맨해튼 거리(Manhattan Distance) 공식(|x1 - x2| + |y1 - y2|)을 사용해야 시간 복잡도와 논리 오류를 획기적으로 줄일 수 있다.
- 문자열 조합 연산이 많을 때는 ArrayList<String>에 add한 뒤 String.join을 쓰는 것보다 StringBuilder의 append()를 사용하는 것이 메모리나 성능 면에서 더 좋다.
- 자바에서 문자열 객체의 값을 비교할 때는 무조건 .equals()를 사용해야 한다는 기본을 잊지 말자.
