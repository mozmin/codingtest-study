### 문제 설명 (요약)
1번부터 $n$번까지 트리 구조로 연결된 배양체들 중 하나가 바이러스에 감염되어 있습니다. 파이프는 A(1), B(2), C(3)의 3가지 종류가 있으며 초기에는 모두 닫혀 있습니다. 특정 종류의 파이프를 한 번에 모두 열었다가 닫는 행동을 최대 $k$번 반복할 수 있을 때, 최종적으로 감염될 수 있는 배양체 개수의 최댓값을 구하는 문제입니다. 단, 한 종류의 파이프를 연 후 다시 닫기 전에 다른 종류의 파이프를 열 수 없습니다.

---

### 핵심 풀이 전략
1. **제한사항을 통한 접근법 선택**: 배양체의 개수 $n$은 최대 100개로 작고, 파이프를 조작하는 횟수 $k$도 최대 10회로 매우 작습니다. 매 단계마다 선택할 수 있는 파이프의 종류는 3가지(A, B, C)이므로, 최대 경우의 수는 $3^{10} = 59,049$가지입니다. 따라서 모든 파이프 개폐 순서를 완전탐색(DFS)하는 방법이 유효합니다.
2. **상태 전이와 탐색 분리**: 
   - **DFS (조작 순서 선택)**: 깊이(`k`)를 내려가며 A, B, C 중 어떤 파이프를 열지 결정합니다. 최대 $k$번에 도달하거나 모든 노드가 감염되면 탐색을 종료합니다.
   - **BFS (바이러스 확산)**: 특정 파이프 종류가 열렸을 때, **이미 감염된 노드들(`visited == true`)을 시작점**으로 삼아 큐에 넣고 연결된 파이프 중 해당 타입의 파이프를 타고 이동 가능한 인접 노드들을 새로 감염시킵니다.
3. **배열 깊은 복사(Deep Copy)를 통한 백트래킹**: 하나의 파이프를 열어 바이러스가 퍼진 상태는 다음 선택에 누적되어 영향을 줍니다. 따라서 다음 DFS 분기로 넘어갈 때 현재까지의 감염 상태 배열(`visited`)을 복사하여 전달함으로써 이전 상태를 안전하게 보존합니다.

---

### 내가 구현한 구현 코드
```java
import java.util.*;

class Solution {
    
    static int maxAns;
    
    public int solution(int n, int infection, int[][] edges, int k) {
        maxAns = 0;
        
        // 트리 그래프 인접 리스트 초기화 (노드 번호, 파이프 타입)
        ArrayList<int[]>[] nodes = new ArrayList[n + 1];
        for (int i = 1; i <= n; i++) {
            nodes[i] = new ArrayList<>();
        }
        
        for (int[] edge : edges) {
            int x = edge[0];
            int y = edge[1];
            int type = edge[2];
            nodes[x].add(new int[]{y, type});
            nodes[y].add(new int[]{x, type});
        }
        
        boolean[] visited = new boolean[n + 1];
        visited[infection] = true; // 최초 감염지 설정
        
        dfs(n, nodes, visited, k);
        
        return maxAns;
    }
    
    public void dfs(int n, ArrayList<int[]>[] nodes, boolean[] visited, int k) {
        // 현재 감염된 배양체 수 계산
        int currentCount = 0;
        for (int i = 1; i <= n; i++) {
            if (visited[i]) currentCount++;
        }
        
        maxAns = Math.max(maxAns, currentCount);
        
        // 모든 배양체가 감염되었거나 행동 기회를 다 쓴 경우 종료
        if (currentCount == n || k == 0) {
            return;
        }
        
        // 1: A, 2: B, 3: C 파이프 선택 기로
        for (int i = 1; i <= 3; i++) {
            // 깊은 복사를 위해 visited 배열 복사
            boolean[] copyVisited = new boolean[n + 1];
            for (int j = 1; j <= n; j++) {
                copyVisited[j] = visited[j];
            }
            
            // 먼저 bfs()로 파이프 확산 작업을 한 후 다음 dfs()로 분기처리
            bfs(i, nodes, copyVisited);
            
            dfs(n, nodes, copyVisited, k - 1);
        }
        
        return;
    }
    
    public void bfs(int pipe, ArrayList<int[]>[] nodes, boolean[] visited) {
        Deque<Integer> q = new ArrayDeque<>();
        
        // 현재 감염된 노드들 모두 큐에 삽입
        for (int i = 1; i < visited.length; i++) {
            if (visited[i]) q.offer(i);
        }
        
        while (!q.isEmpty()) {
            int curr = q.poll();
            
            for (int[] nextInfo : nodes[curr]) {
                int nextNode = nextInfo[0];
                int pipeType = nextInfo[1];
                
                // 아직 감염되지 않았고, 현재 열어둔 파이프 종류와 일치한다면 확산
                if (!visited[nextNode] && pipeType == pipe) {
                    visited[nextNode] = true;
                    q.offer(nextNode);
                }
            }
        }
    }
}
```

## 회고 및 삽질기록 🧠
- 이 문제는 일반적인 트리 탐색과 달리, 특정 시점에 '종류가 같은 모든 파이프'가 한꺼번에 열린다는 특징이 있다. 따라서 매번 한 칸씩 이동하는 DFS가 아니라, **경로 선택은 DFS(조작 조합 탐색)로 하고, 바이러스의 동시 다발적 확산은 BFS로 처리하는 혼합형 방식**이 핵심 설계 포인트임을 깨달았다.
- 단순 백트래킹을 하려고 `visited[nextNode] = false` 형태로 원상복구 하려다가는, 이전 단계에서 이미 감염되었던 노드들까지 잘못 리셋되어 격리가 깨지는 대참사가 날 수 있었다. 상태가 복잡하게 누적되는 탐색에서는 이전 방문 배열을 훼손하지 않도록 `.clone()`을 통해 독립적인 상태 스냅샷을 들고 들어가는 설계가 훨씬 안전하다는 것을 다시 확인했다.
- `currentCount == n` 조건문을 통해 조기에 모든 배양체가 감염되었을 때 불필요한 하위 재귀를 타지 않도록 가지치기(Pruning)를 적절히 배치하여 효율성을 높일 수 있었다.
