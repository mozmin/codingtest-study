# [Medium] CoinChange

[문제 링크](https://leetcode.com/problems/coin-change/description/)

`#DP` `#시뮬레이션`

### 제출 일자

### 문제 설명 (요약)
- 주어진 동전 배열 coins와 목표 금액 amount가 있을 때,
- amount를 만들기 위해 필요한 동전의 최소 개수를 구하는 문제이다.
- 각 동전은 무한히 사용할 수 있으며, 만약 주어진 동전들로 amount를 만들 수 없다면 -1을 반환한다.

### 입력
- int coins[] = 사용할 수 있는 코인이 담겨있는 1차원 정수 배열 
- int amount = 목표 금액

### 출력
- 목표 금액을 만들기 위한 최소한의 동전 개수, 만들 수 없다면 -1을 반환

<hr>

### 핵심 풀이 전략
1. 모든 값을 amount + 1로 초기화 한다. (정답이 나올 수 없는 수)
2. dp[i]를 i원을 만들기 위한 최소 동전 개수로 정의한다.
3. dp[0] = 0으로 초기화 한다.
4. 1원부터 amount원까지 순서대로 최소 동전 개수를 계산한다.
5. 현재 금액(target)에 대해 모든 동전을 하나씩 사용해본다.
6. 현재 동전을 마지막으로 사용한다고 가정한다. (남은 금액은 target-coin)
7. 이전에 계산된 dp[target-coin]을 활용하여 새로운 경우의 수를 계산한다.

---

```java
import java.util.*;

class Solution {
    public int coinChange(int[] coins, int amount) {

        int[] dp = new int[amount + 1];
        
        Arrays.fill(dp, amount + 1);

        dp[0] = 0;

        for (int target = 1; target <= amount; target++) {
            for (int coin : coins) {
                if (target >= coin) {
                     dp[target] = Math.min(dp[target], dp[target - coin] + 1);
                }
            }
        }

        if (dp[amount] == amount + 1) {
            return -1;
        }

        return dp[amount];
    }
}
```
