`#완전탐색` `#깊이우선탐색(DFS)` `#백트래킹`

### 제출 일자 
2026년 6월 25일 0.39ms

### 문제 설명 (요약)
유저의 현재 피로도 k와 각 던전의 ["최소 필요 피로도", "소모 피로도"]가 주어질 때, 유저가 탐험할 수 있는 최대 던전 수를 구하는 문제입니다. 각 던전은 하루에 한 번만 탐험할 수 있으며, 던전을 시작하려면 현재 피로도가 "최소 필요 피로도" 이상이어야 하고, 탐험 후에는 "소모 피로도"만큼 피로도가 감소합니다.

---

### 핵심 풀이 전략
1. **제한사항 확인을 통한 접근**: 던전의 개수(dungeons의 세로 길이)가 1 이상 8 이하로 매우 작습니다. 이는 모든 경우의 수를 전부 탐색하는 **완전탐색(Brute Force)**이 가능함을 시사합니다. 최대 $8! = 40,320$가지의 경로만 존재하므로 DFS를 활용해 모든 순열을 탐색해도 시간 초과가 나지 않습니다.
2. **방문 배열을 통한 순열 생성**: 던전의 탐험 순서가 결과에 영향을 주므로, `visited` 배열을 사용하여 이미 방문한 던전을 제외하고 다음 던전을 고르는 순열(Permutation) 구조로 탐색합니다.
3. **백트래킹 및 최댓값 갱신**: 탐색을 진행하며 매 단계마다 탐험한 던전의 수(`count`)와 전역 변수 `max`를 비교하여 최댓값을 갱신합니다. 조건(현재 피로도 점검)에 맞지 않으면 더 이상 깊이 들어가지 않고 이전 상태로 되돌아옵니다.

---

### 내가 구현한 구현 코드
``` java
import java.util.*;

class Solution {
    
    static int max;
    public int solution(int k, int[][] dungeons) {
        
        max = 0;
        boolean[] visited = new boolean[dungeons.length];
        dfs(k, dungeons, 0, visited);
        
        return max;
    }
    
    public void dfs(int k, int[][] dungeons, int count, boolean[] visited){
        

        max = Math.max(max, count);

        for(int i = 0; i < dungeons.length; i++){
            if(!visited[i] && k >= dungeons[i][0] && k >= dungeons[i][1]){
                visited[i] = true;
                dfs(k-dungeons[i][1], dungeons, count+1, visited);
                visited[i] = false;
            }
        }
    }
    
}
```


## 회고 및 삽질기록 🧠
- 제한사항에서 던전의 개수가 최대 8개라는 점을 보고 바로 완탐/DFS 설계로 방향을 잡을 수 있었다.
- 처음 조건을 작성할 때 최소 필요 피로도와 소모 피로도를 모두 체크(`k >= dungeons[i][0] && k >= dungeons[i][1]`)했으나, 문제 제한사항에 "최소 필요 피로도는 항상 소모 피로도보다 크거나 같다"는 조건이 명시되어 있었다. 따라서 최소 필요 피로도만 확인해도 충분하다는 점을 간과하여 불필요한 연산을 추가했었다. 문제를 꼼꼼히 읽는 습관을 들여야겠다.
- 코딩 테스트 환경에서는 `static` 변수 초기화를 깜빡하면 이전 테케의 값이 누적되는 대참사가 날 수 있으므로, 가급적 재귀 함수의 인자나 리턴값을 활용해 상 결괏값을 주고받는 연습을 지속해야겠다.
