### 문제 설명 (요약)
- 현재 피로도 k와 여러 던전 정보가 주어진다.
- 각 던전은 [최소 필요 피로도, 소모 피로도]와 같은 정보를 가지고 있다.
- 현재 피로도가 최소 필요 피로도 이상이면 해당 던전에 입장할 수 있고, 입장 후에는 소모 피로도만큼 피로도가 감소한다.
- 이때 탐험할 수 있는 던전의 최대 개수를 구하는 문제이다.

### 입력
- int k : 시작 필요도
- int[][] dungeons : 던전 정보가 담겨져 있는 2차원 정수 배열

### 출력
- 최대로 탐험할 수 있는 던전의 개수를 반환

<hr>

### 핵심 풀이 전략
1. 던전의 방문 여부를 체크하는 boolean[] visited 배열을 선언한다.
2. DFS를 통해 현재 갈 수 있는 던전을 하나씩 선택한다.
3. DFS에 들어올 때마다 현재까지 탐험한 최대 횟수를 갱신한다.
4. 현재 피로도 k가 던전의 최소 필요 피로도 이상이면 입장한다.
5. 던전에 입장하면 방문 처리 후, 소모 피로도를 차감하고 다음 DFS를 호출한다.
6. 재귀 호출이 끝나면 방문 여부를 되돌린다.
---

```java
class Solution {
    int maxCount = 0;
    boolean[] visited;
    public int solution(int k, int[][] dungeons) {
        visited = new boolean[dungeons.length];
        
        dfs(k, dungeons, 0);
        
        return maxCount;
    }
    
    public void dfs(int k, int[][] dungeons, int sum){
        maxCount = Math.max(maxCount, sum);
        
        for(int i = 0; i < dungeons.length; i++){
            if(visited[i] == true){
                continue;
            }
            
            if(k >= dungeons[i][0]){
                visited[i] = true;
                dfs(k - dungeons[i][1], dungeons, sum + 1);
                visited[i] = false;
            }
        }
    }
}
```
