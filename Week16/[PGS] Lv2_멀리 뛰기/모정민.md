# [Level 2] 멀리 뛰기

`#다이나믹프로그래밍(DP)` `#수학`

### 제출 일자
2026년 5월 31일 0.06ms

### 문제 설명 (요약)
효진이는 한 번에 1칸, 또는 2칸을 뛸 수 있습니다. 칸이 총 `n`개 있을 때, 효진이가 끝에 도달하는 방법이 몇 가지인지 알아내어, 그 값에 1234567을 나눈 나머지를 반환합니다.

---

### 핵심 풀이 전략
1. **패턴 파악 (피보나치 수열)**: 1칸을 갈 때(1), 2칸을 갈 때(2), 3칸을 갈 때(3), 4칸을 갈 때(5)... 와 같이 나열해 보면 전형적인 피보나치 수열의 형태를 띱니다.
2. **점화식 도출**: 현재 칸(`n`)에 도달하기 위해서는 한 칸 전(`n-1`)에서 1칸을 뛰어오거나, 두 칸 전(`n-2`)에서 2칸을 뛰어오는 두 가지 경우밖에 없습니다. 따라서 `dp(n) = dp(n-1) + dp(n-2)`가 성립합니다.
3. **모듈러 연산**: 수가 기하급수적으로 커지므로, 오버플로우를 방지하기 위해 배열에 값을 저장할 때마다 `1234567`로 나눈 나머지를 저장합니다.

---

### 내가 구현한 구현 코드
```java
class Solution {
    
    public long solution(int n) {
        
        if(n == 1) return 1;
        else if(n == 2) return 2;
        
        long result = 0;
        
        // 점화식: dp(n) = dp(n-1) + dp(n-2)
        int[] dist = new int[n+1];
        
        dist[1] = 1;
        dist[2] = 2;
        
        if(n <= 2) return dist[n];
        else{
            for(int i = 3; i <= n; i++){
                dist[i] = (dist[i-1] + dist[i-2]) % 1234567;
            }
        }
        
        return dist[n];
    }

}
```

### 정답 코드 (리팩토링 - 공간 복잡도 $O(1)$ 최적화 및 중복 조건문 제거)
코드 초반에 `n == 1`과 `n == 2`일 때 이미 `return`으로 빠져나가기 때문에, 뒤에 나오는 `if(n <= 2)` 조건문은 불필요한 중복. 
또한 이 문제는 바로 직전의 두 개의 값만 알면 다음 값을 구할 수 있으므로, 크기가 $O(N)$인 배열(`dist`)을 선언할 필요 없이 **두 개의 변수만으로** 공간 복잡도를 $O(1)$로 줄일 수 있다.

```java
class Solution {
    public long solution(int n) {
        // 예외 처리 및 초기값 설정
        if (n == 1) return 1;
        if (n == 2) return 2;
        
        // 배열 대신 변수 두 개로 이전 상태만 기억
        long prev1 = 1; // n-2 칸
        long prev2 = 2; // n-1 칸
        long current = 0;
        
        for (int i = 3; i <= n; i++) {
            current = (prev1 + prev2) % 1234567; // 다음 값 계산
            
            // 한 칸씩 앞으로 전진
            prev1 = prev2;
            prev2 = current;
        }
        
        return current;
    }
}
```

### 삽질기록 🧠
- 반복문 내부에 `% 1234567` 모듈러 연산을 까먹어서 틀렸다
- dist[1] 과 dist[2]에 값이 못 들어갈 수도 있어서(배열의 크기가 2보다 작을경우) 이를 위한 조건 처리에서 애를 먹었다.

### 배운 점 && 보완할 점
- **공간 복잡도(Space Complexity) 개선 (Sliding Window for DP)**: DP 문제에서 점화식이 직전 $K$개의 항만 참조한다면, 배열 전체를 유지할 필요 없이 변수 $K$개만 업데이트하며 진행하면 메모리를 크게 아낄 수 있다.
- **불필요한 조건 분기 제거**: Early Return 패턴을 적용해 함수 상단에서 예외 처리를 끝냈다면, 하위 로직에서는 해당 케이스들을 완전히 배제하고(예: 무조건 `n >= 3`이라고 가정하고) 코드를 깔끔하게 전개할 수 있다.
