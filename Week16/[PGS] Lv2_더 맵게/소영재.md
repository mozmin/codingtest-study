### 문제 설명 (요약)
- 모든 음식의 스코빌 지수를 K 이상으로 만들기 위해 가장 맵지 않은 두 음식을 반복적으로 섞는다.
- 음식을 섞는 공식은 새로운 음식 = 가장 맵지 않은 음식 + (두 번째로 맵지 않은 음식 * 2)이다.
- 모든 음식의 스코빌 지수가 K 이상이 될 때까지 섞은 횟수를 반환한다.
- 만약 모든 음식의 스코빌 지수를 K 이상으로 만드는 것이 불가능하다면 -1을 반환한다.

### 입력
- int[] scoville : 음식들의 스코빌 지수가 담긴 1차원 정수 배열
- int K : 목표 스코빌 지수

### 출력
- 모든 음식의 스코빌 지수를 K 이상으로 만들기 위해 섞어야 하는 최소 횟수, 불가능한 경우는 -1 반환

<hr>

### 핵심 풀이 전략
1. 최소값을 빠르게 반환할 수 있는 PriorityQueue를 활용하여 모든 음식을 저장한다.
2. 가장 작은 스코빌 지수가 K 이상이 될 때까지 다음 과정을 반복한다.
    - 가장 작은 음식 꺼내기
    - 두 번째로 작은 음식 꺼내기
    - 새로운 음식 (스코빌 조합) 생성
    - PriorityQueue에 다시 삽입 후 카운트 증가
3. 최종적으로 섞은 횟수인 count를 반환한다.

---

```java
import java.util.*;
class Solution {
    public int solution(int[] scoville, int K) {
        int count = 0;
        PriorityQueue<Integer> lowScoville = new PriorityQueue<>();
        PriorityQueue<Integer> highScoville = new PriorityQueue<>();
        
        for(int i = 0; i < scoville.length; i++){
            if(scoville[i] < K){
                lowScoville.add(scoville[i]);
            } else {
                highScoville.add(scoville[i]);
            }
        }
        
        while (!lowScoville.isEmpty()) {
            int firstFood = lowScoville.poll();

            int secondFood;

            if (!lowScoville.isEmpty()) {
                secondFood = lowScoville.poll();
            } else if (!highScoville.isEmpty()) {
                secondFood = highScoville.poll();
            } else {
                return -1;
            }

            int addFood = firstFood + (secondFood * 2);
            count++;

            if (addFood < K) {
                lowScoville.add(addFood);
            } else {
                highScoville.add(addFood);
            }
        }

        return count;
    }
}
```

### 삽질 기록 🧠
```java
import java.util.*;
class Solution {
    public int solution(int[] scoville, int K) {
        int count = 0;
        List<Integer> lowScoville = new ArrayList<>();
        List<Integer> highScoville = new ArrayList<>(); 
        for(int i = 0; i < scoville.length; i++){
            if(scoville[i] < K){
                lowScoville.add(scoville[i]);
            } else {
                highScoville.add(scoville[i]);
            }
        }
        
        Collections.sort(lowScoville);
        Collections.sort(highScoville);
        
        for(int j = 0; j < lowScoville.size(); j++){
            int firstFood = lowScoville.get(j);
            lowScoville.remove(j);
            int secondFood = lowScoville.get(j);
            lowScoville.remove(j);
            
            int conditionFood = firstFood + (secondFood * 2);
            
            if(conditionFood > K){
                highScoville.add(conditionFood);
            } else{
                lowScoville.add(conditionFood);
                count++;
            }
        }
        return count;
    }
}
```
- 처음에는 가변적인 길이를 다루기 위해 List를 사용했다. 하지만 문제를 풀다보니 스코빌 지수를 계산하고 다시 넣을 때 정렬을 계속 해줘야 한다는 점에서 의문이 들게 되었다.
- 또한, 리스트 내부의 값을 삭제하면서 예를 들어 리스트 내부의 하나의 값만 남아있다면 IndexOutOfBoundsException 예외가 발생한다는 점도 알게 되었다.
- 따라서 PriorityQueue를 활용하는 방식으로 다시 접근하였다.

### 배운 점 && 보완할 점
- PriorityQueue를 다시 학습하게 되었다. PriorityQueue는 내부적으로 전체가 정렬된 상태를 유지하는 것이 아닌, 가장 작은 값이 항상 먼저 나오도록 관리하는 자료구조임을 깨닫게 되었다.
- PriorityQueue는 기본적으로 오름차순 정렬이지만, 내림차순으로 만들 수도 있다.
- PriorityQueue<Integer> pq = new PriorityQueue<>(Collections.reverseOrder());
- 정리를 하면서 코드를 다시 보니, 굳이 PriorityQueue를 두 개로 나눠서 푸는 방법보다, 한 개로 푸는 방법이 편했을 것 같다..
