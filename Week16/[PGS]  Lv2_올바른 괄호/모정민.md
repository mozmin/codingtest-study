# [Level 2] 올바른 괄호

[문제 링크](https://school.programmers.co.kr/learn/courses/30/lessons/12909)

`#스택/큐` `#문자열`

### 제출 일자
2026년 6월 1일 0.09ms

### 문제 설명 (요약)
괄호가 바르게 짝지어졌다는 것은 '(' 문자로 열렸으면 반드시 짝지어서 ')' 문자로 닫혀야 한다는 의미입니다.
문자열 `s`가 주어질 때, 올바른 괄호이면 `true`를, 아니면 `false`를 반환합니다.

- **조건**:
  - 문자열 `s`의 길이: 100,000 이하의 자연수
  - 문자열 `s`는 '(' 또는 ')' 로만 이루어져 있습니다.

---

### 핵심 풀이 전략
1. **스택(Stack)의 원리 활용**: LIFO(Last In First Out) 구조를 이용해 괄호의 짝을 맞춥니다.
2. **여는 괄호 처리**: `(`가 나오면 저장(Push)합니다.
3. **닫는 괄호 처리**: `)`가 나오면 가장 최근에 저장된 `(`를 제거(Pop)합니다. 이때 저장된 여는 괄호가 없다면 올바르지 않은 괄호입니다.
4. **최종 검증**: 문자열을 모두 순회한 후, 저장소에 남은 괄호가 없어야(Empty) 올바른 괄호입니다.

---

### 내가 구현한 구현 코드
```java
import java.util.*;
class Solution {
    boolean solution(String s) {

        Deque<Character> q = new ArrayDeque<>();
        
        for(int i = 0; i < s.length(); i++){
            
            char c = s.charAt(i);
            
            if(c == '(') q.push(c);
            else if(c == ')'){
                if(q.isEmpty()) return false;
                q.pop();
            }
        }

        return q.isEmpty();
    }
}
```

### 정답 코드 (리팩토링 - 카운팅 변수를 활용한 공간 복잡도 $O(1)$ 최적화)
작성하신 코드는 스택(자바에서는 `Stack` 대신 `ArrayDeque`를 사용하는 것이 권장됨)을 잘 활용한 정석적인 풀이입니다. 로직상 완벽하고 시간 초과도 발생하지 않습니다. 
다만, 이 문제는 괄호의 종류가 `()` 한 가지뿐이기 때문에 실제 `Deque` 자료구조를 할당하지 않고 **정수형 변수 하나(카운터)만으로** 스택의 역할을 완벽히 대신할 수 있습니다. 이를 통해 공간 복잡도를 $O(N)$에서 $O(1)$로 획기적으로 줄일 수 있습니다.

```java
class Solution {
    boolean solution(String s) {
        int count = 0; // 스택의 사이즈(여는 괄호의 개수)를 대신할 변수
        
        // toCharArray()를 사용해 향상된 for문으로 가독성 확보
        for (char c : s.toCharArray()) {
            if (c == '(') {
                count++;
            } else {
                if (count == 0) return false; // 짝을 맞출 여는 괄호가 없는 경우
                count--;
            }
        }
        
        // 모든 순회 후 여는 괄호가 남아있지 않아야 함
        return count == 0;
    }
}
```
