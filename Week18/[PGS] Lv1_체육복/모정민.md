### 문제 설명 (요약)
도난당한 학생들에게 여벌 체육복이 있는 학생들이 체육복을 빌려주어 최대한 많은 학생이 체육수업을 들을 수 있도록 하는 문제입니다. 학생들은 바로 앞번호나 뒷번호의 학생에게만 체육복을 빌려줄 수 있습니다. 단, 여벌 체육복을 가져온 학생도 도난을 당했을 수 있으며, 이 경우 남은 체육복이 하나뿐이므로 다른 학생에게 체육복을 빌려줄 수 없습니다. 최종적으로 체육수업을 들을 수 있는 학생 수의 최댓값을 반환합니다.

---

### 핵심 풀이 전략
1. **여벌복 도난 학생 선처리 (중복 제거)**: 여벌 체육복을 가져왔지만 도난당한 학생은 빌려줄 수도, 빌릴 필요도 없는 평범한 상태(체육복 1개)가 됩니다. 다른 로직을 수행하기 전에 이 조건을 가장 먼저 반영해야 합니다.
2. **선택의 기준 (Greedy)**: 체육복이 없는 학생을 기준으로 **앞번호(`i-1`) 학생에게 먼저 빌릴 수 있는지 확인**하는 것이 무조건 이득입니다. 앞번호 학생은 뒷번호 학생(`i`) 외에는 다른 대안이 없을 확률이 높기 때문에, 앞번호부터 챙기는 것이 전체 생존자를 극대화하는 그리디 핵심 탐색 방향입니다.
3. **배열을 이용한 상태 관리**: `HashSet`을 사용해 순회하면서 데이터를 직접 지우면 런타임 에러가 발생할 수 있습니다. 대신 전체 학생 수 크기의 `int[]` 배열을 생성하여 각 학생의 체육복 상태 수치(`-1`, `0`, `1`)를 직접 증감시키는 방식이 직관적이고 안전합니다.

---

### 내가 구현한 구현 코드
```java
import java.util.*;
class Solution {
    public int solution(int n, int[] lost, int[] reserve) {
        int answer = 0;
        
        int[] student = new int[n+1];
        
        // 도난 당한 학생 정리
        for(int l : lost){
            student[l]++;
        }
        
        for(int r : reserve){
            student[r]--;
        }
        
        
        // 앞번호 우선
        for(int i = 1; i <= n; i++){
            
            if(i-1 > 0 && student[i] == 1 && student[i-1] == -1){
                student[i-1]++;
                student[i]--;
            }
            else if(i+1 <= n && student[i] == 1 && student[i+1] == -1){
                student[i+1]++;
                student[i]--;
            }
        }
        
        // 0인 학생 갯수
        for(int i = 1; i <= n; i++){
            if(student[i] <= 0) answer++;
        }  
            
        return answer;
    }
}
```

## 정답 코드 (리팩토링 - 직관적인 상태 값 부여 및 가독성 개선)
도난당한 학생은 `1 - 1 = 0`개, 여벌이 있는 학생은 `1 + 1 = 2`개와 같이 실제 가지고 있는 체육복 개수를 직관적으로 대입하여 배열을 초기화하면 코드의 가독성이 훨씬 높아집니다. 

경계값 체크(`i - 1 > 0`, `i + 1 <= n`) 시 인덱스 에러를 방지하기 위해 배열 크기를 `n + 2`로 여유 있게 선언하면 조건문 내부를 더 단순화할 수 있습니다.

```java
import java.util.*;

class Solution {
    public int solution(int n, int[] lost, int[] reserve) {
        // 인덱스 경계 초과 방지를 위해 n + 2 크기로 선언
        int[] students = new int[n + 2];
        
        // 모든 학생은 기본적으로 1벌씩 가지고 있음
        Arrays.fill(students, 1);
        
        // 도난당한 학생은 차감 (0벌)
        for (int l : lost) {
            students[l]--;
        }
        
        // 여벌이 있는 학생은 추가 (도난과 중복된 경우 자동으로 1벌이 됨)
        for (int r : reserve) {
            students[r]++;
        }
        
        // 1번 학생부터 n번 학생까지 탐색하며 빌려주기 진행
        for (int i = 1; i <= n; i++) {
            if (students[i] == 0) { // 내가 체육복이 없다면
                if (students[i - 1] == 2) { // 앞번호가 여벌이 있는가?
                    students[i - 1]--;
                    students[i]++;
                } else if (students[i + 1] == 2) { // 뒷번호가 여벌이 있는가?
                    students[i + 1]--;
                    students[i]++;
                }
            }
        }
        
        // 체육복이 1벌 이상인 학생 수 카운트
        int answer = 0;
        for (int i = 1; i <= n; i++) {
            if (students[i] >= 1) {
                answer++;
            }
        }
        
        return answer;
    }
}
```

## 회고 및 삽질기록 🧠
- Java의 `for-each` 루프 컬렉션을 순회하는 도중, 대상을 직접 삭제(`HashSet.remove()`)하려고 시도하다가 `ConcurrentModificationException` 런타임 에러를 마주했다. 컬렉션을 순회할 때는 순회 기준 객체와 삭제 대상 객체를 분리해야 하며, 이 문제처럼 단순 선형 구조를 다룰 때는 컬렉션 대신 일반 `int[]` 배열을 활용하는 것이 에러를 방지하고 상태를 안전하게 관리하는 좋은 대안임을 깨달았다.
- Set 풀이법을 포기하고, 단순 int[] 배열을 사용하였다.
