### 문제 설명 (요약)
직원들이 각자 설정한 출근 희망 시각에 늦지 않고 출근한 날을 계산하여, 일주일 동안 평일에 모두 제시간(출근 희망 시각 + 10분 이내)에 출근한 직원에게 상품을 주는 문제입니다. 단, 토요일과 일요일의 출근 시각은 이벤트에 영향을 주지 않습니다. 모든 시각은 시에 100을 곱하고 분을 더한 정수로 표현되며, 최종적으로 상품을 받을 직원의 수를 반환합니다.

---

### 핵심 풀이 전략
1. **출근 인정 시각 계산**: 출근 희망 시각에 10분을 더할 때, 분(minute)이 60분 이상이 되는 케이스를 정확히 처리해야 합니다. 시각 표현에서 분은 59를 넘을 수 없으므로, 60분 이상이 되면 시간에 1을 더하고 분에서 60을 빼주어야 합니다.
2. **요일(주말) 처리**: 이벤트를 시작한 요일(`startday`)을 기준으로 각 일차(`j`)의 요일을 계산해야 합니다. 나눗셈 연산자(`% 7`)를 활용하여 주말(토요일, 일요일)인 날은 출근 기록 검사에서 제외(`continue`)합니다.
3. **지각 여부와 카운트 관리**: 각 직원의 일주일 출근 기록을 순회하며 평일에 한 번이라도 출근 인정 시각을 초과하면 해당 직원은 탈락이므로 반복을 멈추고(`break`), 지각하지 않은 평일의 수가 총 평일 수(5일)와 같을 때만 상품 수령 대상자로 카운트합니다.

---

### 내가 구현한 구현 코드
```java
import java.util.*;

class Solution {
    public int solution(int[] schedules, int[][] timelogs, int startday) {
        int answer = 0;
        
        // 희망 출근시간
        int[] converted = new int[schedules.length];
        for(int i = 0; i < schedules.length; i++){
            int minute = (schedules[i] % 100) + 10;
            
            if(minute >= 60){
                converted[i] = ((schedules[i] / 100) * 100) + 100 + (minute-60); 
            }else{
                converted[i] = ((schedules[i] / 100) * 100) + minute; 
            }
        }

        // 지각 여부 판단
        // timelogs.length -> n명
        for(int i = 0; i < timelogs.length; i++){
            
            // 일주일
            int count = 0;
            for(int j = 0; j < 7; j++){
                
                int d = (startday + j) % 7;
                
                // 주말은 패스
                if(d == 6 || d == 0) continue;
                
                // 만약 출근시간을 지키 못했다면,
                if(timelogs[i][j] > converted[i]) break;
                
                count++;
            }
            
            if(count >= 5) answer++;
        }
        
        return answer;
    }
}
```

## 정답 코드 (리팩토링 - 가독성 및 시간 계산 로직 단순화)
시와 분을 분리하여 계산할 때, 기존의 값에 직접 연산하는 방식을 사용하면 코드가 한눈에 들어오고 실수를 줄일 수 있습니다. 또한 `converted` 배열을 따로 두지 않고 직원을 순회할 때 즉시 제한 시각을 계산하면 메모리와 가독성 측면에서 더 깔끔해집니다.

```java
import java.util.*;

class Solution {
    public int solution(int[] schedules, int[][] timelogs, int startday) {
        int answer = 0;
        
        for (int i = 0; i < schedules.length; i++) {
            // 1. 해당 직원의 출근 인정 시각 계산
            int hour = schedules[i] / 100;
            int minute = (schedules[i] % 100) + 10;
            
            if (minute >= 60) {
                hour += 1;
                minute -= 60;
            }
            int limitTime = (hour * 100) + minute;
            
            // 2. 일주일 동안 지각 여부 체크
            boolean isPerfect = true;
            for (int j = 0; j < 7; j++) {
                int day = (startday + j) % 7;
                
                // 토요일(6)과 일요일(0)은 제외
                if (day == 6 || day == 0) {
                    continue;
                }
                
                // 평일에 한 번이라도 지각한 경우
                if (timelogs[i][j] > limitTime) {
                    isPerfect = false;
                    break;
                }
            }
            
            // 모든 평일을 무사히 통과했다면 상품 수령
            if (isPerfect) {
                answer++;
            }
        }
        
        return answer;
    }
}
```

## 회고 및 삽질기록 🧠
- 조건문에서 경계값인 `60분` 처리를 누락하는 실수를 했다. 예를 들어 `9시 50분`인 경우 10분을 더하면 정확히 `60분`이 되는데, 처음에 조건식을 `minute > 60`으로 설정하는 바람에 `960`이라는 잘못된 시각 포맷이 그대로 통과되어 오답이 발생했다. 이상적인 시각 단위를 다룰 때는 이상/이하(`>=`, `<=`) 경계 조건 검사가 얼마나 치명적인지 다시금 깨달았다. `minute >= 60`으로 단 한 줄만 수정하여 명확하게 해결할 수 있었다.
