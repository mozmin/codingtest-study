### 제출 일자

### 문제 설명
- 각 직원은 희망 출근 시각을 정하고, 일주일 동안 실제 출근 기록을 가진다.

- 직원이 평일에 희망 출근 시각보다 10분 이내로 출근하면 해당 날짜는 정상 출근으로 인정된다.
- 토요일과 일요일은 이벤트 대상 출근일에서 제외된다.

- 각 직원별로 평일 5일 동안 모두 정상 출근했는지 확인하고, 조건을 만족한 직원 수를 구하는 문제이다.

### 입력
- schedules: 각 직원의 희망 출근 시각 배열
- timelogs: 각 직원의 7일간 실제 출근 시각 기록
- startday: 이벤트가 시작한 요일
  - 1: 월요일
  - 2: 화요일
  - 3: 수요일
  - 4: 목요일
  - 5: 금요일
  - 6: 토요일
  - 7: 일요일

### 출력
- 평일 5일 동안 모두 정상 출근한 직원 수

---

### 핵심 풀이 전략
- HHMM 형태의 시간을 분 단위로 변환한다.
- 각 직원마다 희망 출근 시각을 분 단위로 변환한다.
- 실제 출근 시각도 분 단위로 변환해서 비교한다.
- 희망 출근 시각 + 10분 이내이면 정상 출근으로 처리한다.
- 토요일과 일요일은 검사하지 않는다.
- 직원마다 시작 요일은 동일해야 하므로, 반복문 안에서 startday를 직접 변경하지 않고 별도의 day 변수를 사용한다.
- 평일 정상 출근 횟수가 5번이면 정답을 증가시킨다.

---

### 제출 코드
```java
class Solution {
    public int solution(int[] schedules, int[][] timelogs, int startday) {
        int answer = 0;
        for(int i = 0 ; i < schedules.length; i++){
            int scheduleMinChange = changeMin(schedules[i]);
            int count = 0;
            for(int j = 0; j < 7; j++){
                if(startday == 6) {
                    startday++;
                    continue;
                }
                if(startday == 7) {
                    startday = 1;
                    continue;
                }
                int timelogsMin = changeMin(timelogs[i][j]);
                if(timelogsMin <= scheduleMinChange + 10) count++;
                startday++;
            }
            if(count == 5) answer++;
            count = 0;
            
        }
        return answer;
        
    }
    
    int changeMin(int schedules){
        int hour = schedules / 100;
        int min = schedules % 100;
        return hour * 60 + min;
    }
}
```
