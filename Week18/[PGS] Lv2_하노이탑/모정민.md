### 문제 설명 (요약)
하노이의 탑은 세 개의 기둥(1, 2, 3번)을 이용해 한 기둥에 쌓인 `n`개의 원판을 크기 순서를 유지하며 다른 기둥으로 모두 옮기는 퍼즐입니다. 한 번에 하나의 원판만 옮길 수 있고, 큰 원판이 작은 원판 위에 올 수 없다는 제약 조건이 있습니다. 1번 기둥에 있는 `n`개의 원판을 최소 횟수로 3번 기둥으로 옮기는 모든 이동 경로를 순서대로 2차원 배열에 담아 반환하는 문제입니다.

---

### 핵심 풀이 전략
1. **분할 정복과 재귀 구조**: `n`개의 원판을 `from`에서 `to`로 옮기기 위한 과정은 크게 세 단계로 쪼갤 수 있습니다.
   * **1단계**: 맨 아래의 가장 큰 원판을 제외한 위의 `n-1`개 원판을 비어있는 보조 기둥(`empty`)으로 옮깁니다.
   * **2단계**: 맨 아래에 남은 가장 큰 원판 1개를 목적지 기둥(`to`)으로 옮깁니다.
   * **3단계**: 보조 기둥(`empty`)에 있던 `n-1`개의 원판을 다시 목적지 기둥(`to`)으로 옮깁니다.
2. **기저 조건 (Base Case)**: 옮겨야 할 원판의 개수가 `n == 1`일 때는 다른 원판을 고려할 필요가 없으므로, 즉시 `from`에서 `to`로 원판을 이동시키고 재귀를 종료합니다.
3. **결과 배열 평탄화**: 재귀 과정에서 몇 번째 단계에 몇 번 이동할지 동적으로 배열 크기를 할당하기 어렵기 때문에, 가변 리스트(`ArrayList<int[]>`)에 경로를 누적한 뒤 마지막에 2차원 배열 형태로 변환(`toArray`)합니다.

---

### 내가 구현한 구현 코드
```java
import java.util.*;

class Solution {
    // 이동 경로를 순서대로 저장할 리스트
    ArrayList<int[]> result = new ArrayList<>();
    
    public int[][] solution(int n) {

        hanoi(n, 1, 3, 2);
        
        return result.toArray(new int[result.size()][]);
    }
    
    private void hanoi(int n, int from, int to, int empty) {

        if (n == 1) {
            result.add(new int[]{from, to});
            return;
        }

        // 1. n-1개를 보조 기둥으로 이동
        hanoi(n - 1, from, empty, to);
        
        // 2. 가장 큰 원판을 목적지 기둥으로 이동
        result.add(new int[]{from, to});

        // 3. 보조 기둥의 n-1개를 목적지 기둥으로 이동
        hanoi(n - 1, empty, to, from);
    }
}
```
