### 문제 설명 (요약)
밑변의 길이와 높이가 `n`인 삼각형에서 맨 위 꼭짓점부터 반시계 방향으로 달팽이 모양 채우기를 진행합니다. 숫자는 1부터 시작하여 빈칸을 순차적으로 채워 나가며, 첫 행부터 마지막 행까지 정렬된 모든 숫자를 하나의 1차원 배열로 합쳐서 반환하는 문제입니다.

---

### 핵심 풀이 전략
1. **방향 전환 구조 파악 (반시계 달팽이)**: 움직임의 방향은 크게 세 가지로 순환합니다:
   * **아래로 이동** (`row++`)
   * **오른쪽으로 이동** (`col++`)
   * **대각선 위로 이동** (`row--`, `col--`)
   방향 전환은 총 `n`번 일어나며, 각 방향으로 이동할 때마다 채워야 하는 칸의 개수가 `n`개부터 시작해 `1`개까지 1씩 줄어듭니다.
2. **2차원 배열을 활용한 좌표 매핑**: 삼각형 모양의 데이터 구조를 `[n+1][n+1]` 크기의 2차원 배열로 표현하면 직관적으로 아래/오른쪽/대각선 위 연산을 수행할 수 있습니다. 
3. **1차원 배열로의 순차 변환**: 달팽이 채우기가 끝난 후, 2차원 배열의 첫 번째 행부터 각 행의 유효한 열(`j <= i`)까지 순서대로 탐색하며 정답 배열에 값을 채워 넣습니다. 총 숫자의 개수는 1부터 `n`까지의 합인 `n * (n + 1) / 2`개입니다.

---

### 내가 구현한 구현 코드
```java
class Solution {
    public int[] solution(int n) {
        
        int lastPoint = (n * (n+1)) / 2;
        int[][] snail = new int[n+1][n+1];
        int[] answer = new int[lastPoint];
        
        int row = 0;
        int col = 1;
        int start = 1;
        // 방향 전환을 n번 수행
        for(int i = 1; i <= n; i++){
            
            // 이동거리가 갈수록 -1씩 줄어들음
            for(int j = i; j <= n; j++){
                
                // 첫번째 방향
                if(i % 3 == 1){
                    row++;
                }
                // 두번째 방향
                else if(i % 3 == 2){
                    col++;
                }
                // 세번째 방향
                else if(i % 3 == 0){
                    row--;
                    col--;
                }
                snail[row][col] = start++;
            }
        }
        
        int index = 0;
        for (int i = 1; i <= n; i++) {
            for (int j = 1; j <= i; j++) { // 삼각형 모양만큼만 돌기
                answer[index++] = snail[i][j];
            }
        }
            
        return answer;
    }
}
```

## 정답 코드 (리팩토링 - 방향 배열을 활용한 깔끔한 시뮬레이션)
조건문 분기(`if (i % 3 == ... )`) 대신 방향 벡터(dx, dy) 배열을 활용하면 회전 로직을 더욱 모듈화하고 가독성을 높일 수 있습니다. 또한, 배열 인덱스를 0번부터 활용하도록 맞추면 직관적인 메모리 관리가 가능합니다.

```java
class Solution {
    public int[] solution(int n) {
        int[][] matrix = new int[n][n];
        int totalElements = (n * (n + 1)) / 2;
        int[] answer = new int[totalElements];
        
        // 하, 우, 대각선 좌상향 방향 벡터
        int[] dr = {1, 0, -1};
        int[] dc = {0, 1, -1};
        
        int row = -1, col = 0; // 시작 지점 유도를 위한 초기화
        int num = 1;
        int dir = 0; // 현재 방향 인덱스
        
        // i번 전환하며 갈수록 채우는 칸수가 n개부터 1개까지 줄어듦
        for (int i = 0; i < n; i++) {
            for (int j = i; j < n; j++) {
                row += dr[dir];
                col += dc[dir];
                matrix[row][col] = num++;
            }
            // 한 방향 이동이 끝나면 다음 방향으로 전환 (0 -> 1 -> 2 -> 0)
            dir = (dir + 1) % 3;
        }
        
        // 2차원 삼각형 구조 순차 탐색 후 1차원 평탄화
        int idx = 0;
        for (int i = 0; i < n; i++) {
            for (int j = 0; j <= i; j++) {
                answer[idx++] = matrix[i][j];
            }
        }
        
        return answer;
    }
}
```

## 회고 및 삽질기록 🧠
- 주어지는 입력을 2차원 직각삼각형 구조로 추상화하여 규칙을 찾는 부분이 핵심이었다. 처음에는 대각선 방향으로 올라가는 좌표 제어가 까다롭게 느껴졌으나, `row`와 `col`을 동시에 감소(`row--, col--`)시키는 규칙을 도출해 내면서 시뮬레이션을 단순화할 수 있었다.
- 회전할 때마다 수행 횟수가 `n`에서 시작해 `1`까지 하나씩 줄어드는 중첩 루프 구조(`for j = i; j <= n`)의 설계를 확실히 익혔다. `i % 3` 방식의 조건 분기도 훌륭하게 작동하지만, 방향 벡터 배열(`dr`, `dc`)을 인덱스 연산(`(dir + 1) % 3`)으로 갱신하는 패턴이 구현 실수를 방지하는 데 더 유리하다는 점을 인지하자.
