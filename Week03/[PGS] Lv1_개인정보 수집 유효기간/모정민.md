### 문제 설명 (요약)

- 고객의 약관 동의 날짜와 약관별 유효기간이 주어질 때, **오늘 날짜를 기준으로 파기해야 할 개인정보의 번호를 구하는 문제**.
- **핵심 조건:** 모든 달은 28일까지 있다고 가정한다. (실제 달력 적용 불가)

### 입력

- `today` : 오늘 날짜를 의미하는 문자열 (예: "2022.05.19")
- `terms` : 약관의 유효기간을 담은 1차원 문자열 배열 (예: "A 6")
- `privacies` : 수집된 개인정보 정보를 담은 1차원 문자열 배열 (예: "2021.05.02 A")

### 출력

- 파기해야 할 개인정보의 번호를 오름차순으로 담은 1차원 정수 배열

<hr>

### 핵심 풀이 전략

1. 연, 월, 일을 모두 **'일(Day)' 단위로 환산**하여 하나의 정수로 만들어 비교.
2. **해시맵을 이용:** 약관 종류(String)에 따른 유효기간(Integer)을 `HashMap`에 저장하여 시간 복잡도를 $O(1)$로 최적화.
3. **만료일 계산 :** `(수집 일자 환산 값) + (약관 유효기간 환산 값)`이 `(오늘 날짜 환산 값)`보다 작거나 같으면 list.add().

---

```java
import java.util.*;

class Solution {
    public int[] solution(String today, String[] terms, String[] privacies) {
        List<Integer> list = new ArrayList<>();
        
        // 1. 오늘 날짜를 일수로 변환
        int totalToday = dateToDays(today);
        
        // 2. 약관 정보를 HashMap에 저장 (월 단위 -> 일 단위로 변환)
        Map<String, Integer> termMap = new HashMap<>();
        for (String t : terms) {
            String[] split = t.split(" ");
            termMap.put(split[0], Integer.parseInt(split[1]) * 28);
        }
        
        // 3. 개인정보 파기 여부 확인
        for (int i = 0; i < privacies.length; i++) {
            String[] p = privacies[i].split(" ");
            String date = p[0];
            String type = p[1];
            
            // 수집 일자를 일수로 변환 + 약관 유효기간(일수) 더하기
            int expireDay = dateToDays(date) + termMap.get(type);
            
            // 오늘 날짜가 파기일과 같거나 지났다면 파기 대상
            if (totalToday >= expireDay) {
                list.add(i + 1);
            }
        }
        
        // ArrayList를 배열로 변환하여 반환
        return list.stream().mapToInt(i -> i).toArray();
    }
    
    // 날짜 문자열("YYYY.MM.DD")을 총 일수로 변환해주는 헬퍼 메서드
    private int dateToDays(String date) {
        String[] split = date.split("\\.");
        int year = Integer.parseInt(split[0]);
        int month = Integer.parseInt(split[1]);
        int day = Integer.parseInt(split[2]);
        
        return (year * 12 * 28) + (month * 28) + day; // 💡 핵심 실수 포인트: day에는 28을 곱하지 않음!
    }
}
```

### 삽질 기록 🧠
- 처음에 terms[]에서 "A 12" 처럼 약관 월이 2자리 수를 어떻게 나누어야할지 감을 못잡아서 시간을 오래 씀. -> 결국 split()을 쓰기로 함
- 문자열 분리: 날짜를 . 기준으로 분리하려 할 때, split(".")을 사용함. .은 정규식에서 문자 쓸 때는 split("\\.")으로 이스케이프 처리를 하는걸 잊어버림.
- 일 수까지 x28을 해버림: 연, 월, 일을 전부 일수로 변환하는 계산식에서 day 부분에도 28을 곱해버림.
- 컬렉션 → 배열 변환: 반환형이 int[]라서 ArrayList를 변환해야 했는데, 단순히 toArray()만 써서 Object[] 타입으로 반환되어 에러가 발생함. list.stream().mapToInt(i -> i).toArray() 패턴을 사용해야 함을 다시 상기함.
