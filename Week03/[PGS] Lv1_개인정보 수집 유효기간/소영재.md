### 문제 설명 (요약)

고객의 개인 정보 유효기간이 현재 날짜와 동일하거나 지났다면 파기해야 하는 문제

### 입력
String today : 오늘 날짜가 담긴 문자열
String[] terms : 약관 종류와 유효 개월 수가 담긴 배열
String[] privacies : 수집된 일자와 약관 종류가 담긴 배열

### 출력
파기해야 하는 개인정보의 번호 배열
<hr>

### 핵심 풀이 전략

1. 유효기간을 통한 날짜 계산을 해야하기 때문에 먼저 LocalDate(YYYY.MM.DD) 형식으로 변환
2. 문제의 조건이 달은 28일로 고정되어 있지만, 입력 데이터는 월로만 들어오기 때문에 LocalDate 타입을 활용하여 계산 (만약 MM형식이 아닌, MM,DD or DD형식이라면 접근 X)
3. 약관 종류마다 유효기간이 정해져있기 때문에 Map을 통해 매핑
4. 약관마다의 유효 기간을 약관 종류의 월(Month)를 더한 뒤, 현재 날짜와 비교
5. 정답 크기가 가변적이기 때문에 array가 아닌 list를 통해 정답 크기를 선언, 리턴 타입을 int[]배열로 변환

---

```java
import java.util.*;
import java.time.LocalDate;
import java.time.format.DateTimeFormatter;
class Solution {
    public int[] solution(String today, String[] terms, String[] privacies) {
        
        //정답 크기가 가변적이기 때문에 리스트로 선언
        List<Integer> list = new ArrayList<>();
        
        LocalDate localDate = LocalDate.parse(today, DateTimeFormatter.ofPattern("yyyy.MM.dd"));
        
        Map<String, Integer> map = new HashMap<>();
        
        //A -> 6 , B -> 12 매핑
        for(String term : terms){
            String[] str = term.split(" ");
            map.put(str[0], Integer.parseInt(str[1]));
        }
        
        //배열을 돌면서 유효기간이 지나거나 같은 날짜를 리스트에 해당 번호를 붙여 추가
        for(int i = 0; i < privacies.length; i++){
            //날짜와 약관 종류를 분리 후 map에 매핑된 월을 날짜에 더함
            String[] str1 = privacies[i].split(" ");
            LocalDate date = LocalDate.parse(str1[0], DateTimeFormatter.ofPattern("yyyy.MM.dd"));
            LocalDate calc = date.plusMonths(map.get(str1[1]));
            
            //현재 날짜와 비교하여 유효기간이 지났다면 추가
            if (calc.isBefore(localDate) || calc.isEqual(localDate)) {
                 list.add(i + 1);
            }                   
        }
        
        //
        return list.stream().mapToInt(Integer::intValue).toArray();
    }
}

```

### 배운 점 && 보완할 점
얼마 전에 자바 공부를 하면서 날짜와 시간 타입을 다루는 LocalDate..등 여러 클래스를 정리하였는데 이를 활용하여 문제를 풀 수 있겠다 생각하여 접근하였다.
만약 유효기간이 달이 아닌, 일로 주어지고 달이 28일로 고정될 경우에는 LocalDate 방식으로 접근하면 까다로울 것 같아서 다른 방식도 생각해봐야겠다. (LocalDate는 실제 달력 기준으로 계산하기 때문)
