### 문제 설명 (요약)
정해진 순서(**빵 – 야채 – 고기 – 빵**)로 쌓인 재료만 하나의 햄버거로 포장

- 1: 빵, 2: 야채, 3: 고기
- 조리법 순서: **[1, 2, 3, 1]**
- 재료가 쌓이는 순서대로 들어올 때, 만들 수 있는 총 햄버거의 개수 반환

### 입력
- `ingredient`: 재료의 번호가 순서대로 담긴 정수 배열

### 출력
- 완성되어 포장된 햄버거의 개수

---

### 핵심 풀이 전략
1. 같은 크기의 int[] hamburger 선언
2. 재료를 반복하면서 하나씩 hamburger에 넣는다.
3. index도 각 반복마다 +!씩 증가시킨다.
4. index >= 4라면 이전 값 4개를 확인해서 햄버거를 완성할 수 있는지 확인한다.
5. 완성할 수 있다면 answer++ 하고 index를 -4 로 이동시킨다.
6. 어차피 새로운 값이 이전 값을 덮을 거기 때문에 굳이 `= 0` 으로 만드는 로직은 넣지않았다.

---

### Java 구현 코드

```java
import java.util.*;

class Solution {
    public int solution(int[] ingredient) {
        int answer = 0;
        
        int[] hamburger = new int[ingredient.length];
        
        // 0. 인덱스 초기화
        int index = 0;
        
        // 1. ingredient 반복
        for(int i : ingredient){
            
            // 2. 재료 넣기
            hamburger[index++] = i;
            
            // 3. 4번째부터는 햄버거 완성이 되는지 검사
            if(index >= 4
              && hamburger[index-1] == 1 // 빵
              && hamburger[index-2] == 3 // 고기
              && hamburger[index-3] == 2 // 야채
              && hamburger[index-4] == 1 // 빵
            ) { 
                answer++;
                index -= 4;
            }
        }
        
        return answer;
    }
}
```

### 삽질 기록 🧠
1. 후순위 연산
- index++ 하면 그 뒤에 실행되는 코드는 index + 1 이 된 상태인데, 나는 반복문 한번 돌 때마다 index + 1 이 될 것이라고 무의식적으로 생각하고 작업했다.
- 4번째부터 검사하는 if문을 처음에는 `index >= 3` 이라 했는데 index++ 이기 때문에 `index >= 4`로 해야했다.
