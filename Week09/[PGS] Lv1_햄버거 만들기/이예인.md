### 문제 설명
재료가 순서대로 주어질 때, 빵(1) - 야채(2) - 고기(3) - 빵(1) 순서가 만들어질 때마다 햄버거 1개를 포장한다.  <br>
포장한 재료는 즉시 제거되고, 이후 남은 재료들로 다시 햄버거를 만들 수 있다.  <br>
최종적으로 만들 수 있는 햄버거의 개수를 구하는 문제이다.<br>

### 입력
- `ingredient`: 재료의 순서를 나타내는 정수 배열  
  - `1` : 빵
  - `2` : 야채
  - `3` : 고기

### 출력
- 만들 수 있는 햄버거의 개수


### 핵심 풀이 전략
- `ArrayList`를 스택처럼 활용했다.
- 재료를 하나씩 리스트에 넣으면서,  
  리스트의 마지막 4개가 `1, 2, 3, 1`인지 확인했다.
- 햄버거 패턴이 완성되면, 해당 4개를 제거하고 `answer`를 증가시켰다.
- 새 재료는 항상 뒤에 쌓이므로, 매번 전체를 검사할 필요 없이 맨 뒤 4개만 확인하면 된다.


### 제출 코드
```java
import java.util.*;

class Solution {
    public int solution(int[] ingredient) {
        int answer = 0;
        List<Integer> list = new ArrayList<>();
        for(int i = 0; i < ingredient.length; i++){
            list.add(ingredient[i]);
            
            if(list.size() >= 4){
                int n  = list.size();
                
                if(list.get(n-4) == 1 && list.get(n-3) == 2 &&
                   list.get(n-2) == 3 && list.get(n-1) == 1) {
                    
                    list.remove(n-1);
                    list.remove(n-2);
                    list.remove(n-3);
                    list.remove(n-4);
                    
                    answer++;     
                }
            }
        }
        return answer;
    }
}

```

### 삽길 기록 🧠
- 처음에는 햄버거가 만들어질 수 있는 모든 구간을 매번 검사해야 하나 고민했다.
- 하지만 실제로는 새 재료가 맨 뒤에만 추가되기 때문에, 햄버거가 새로 만들어진다면 항상 마지막 부분에서만 확인하면 된다는 점을 놓치고 있었다.
- 삭제할 때 앞에서부터 지우면 인덱스가 밀릴 수 있어서, 뒤에서부터 제거해야 안전하다는 점도 체크했다.
