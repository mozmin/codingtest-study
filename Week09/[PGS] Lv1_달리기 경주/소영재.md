### 문제 설명 (요약)
- 선수들이 등수 순서대로 players[]에 주어진다.
- 해설진이 특정 선수 이름을 부르면, 그 선수는 자기 바로 앞 선수와 자리 교체(추월)를 한다.
- 모든 과정이 끝난 뒤, 등수대로 최종 선수들이 담긴 문자열 배열을 반환한다.
### 입력
- String[] players : 등수 순서대로 선수들 이름이 담긴 배열
- calling[] : 해설진이 부른 이름을 담은 문자열 배열

### 출력
- 경주가 끝난 후 등수 순서대로 선수들 이름이 담긴 문자열 배열을 반환
<hr>

### 핵심 풀이 전략
1. 현재의 순위를 map에 저장한다.
2. 해설진이 부른 이름을 담긴 배열을 순회하면서 players[]과 map의 순위를 스왑하며 갱신한다.
---

```java
import java.util.*;
class Solution {
    public String[] solution(String[] players, String[] callings) {
        
        Map<String, Integer> map = new HashMap<>();
        
        int idx = 0;
        
        for(String player : players){
            map.put(player, idx);
            idx++;
        }
        
       
        for(String player : callings){
            
            int grade = map.get(player);
            String s = players[grade];
            String s2 = players[grade-1];
            map.put(player, grade-1);
            map.put(players[grade-1], grade);
            players[grade-1] = s;
            players[grade] = s2;
            
        }
        return players;
    }
}
```

### 삽질 기록 🧠
```java
import java.util.*;
class Solution {
    public String[] solution(String[] players, String[] callings) {
        for (String player : callings) {
            for (int i = 0; i < players.length; i++) {
                if (players[i].equals(player)) {
                    String s = players[i];
                    String s2 = players[i - 1];

                    players[i] = s2;
                    players[i - 1] = s;
                }
            }
        }

        return players;
    }
}
```
- 처음에 단순히 2중 for문으로 돌아서 시간 초과가 났다.
- 문제를 살펴보니 시간 복잡도가 players 5만, callings 100만이라 이중 for문을 돌게되면 최악의 경우 500억번의 연산 결과가 이뤄진다는 것을 알게되었다.
