### 문제 설명
현재 선수들의 등수가 players 배열에 순서대로 주어지고, <br>
해설진이 선수를 한 명씩 부를 때마다 그 선수는 바로 앞 선수와 자리를 바꾼다. <br>
모든 호출이 끝난 뒤 최종 선수 순서를 반환하는 문제이다. <br>

### 입력
- players: 현재 등수 순서대로 담긴 선수 이름 배열
- callings: 해설진이 부른 선수 이름 배열

### 출력
- 모든 호출이 끝난 뒤의 최종 선수 순서 배열

### 핵심 풀이 전략
처음에는 단순히 players 배열에서 매번 해당 선수를 찾아 앞 선수와 바꾸는 방식도 생각할 수 있다. <br>
하지만 이 경우 호출마다 선수를 찾기 위해 배열을 탐색해야 하므로 비효율적이다. <br>

그래서 선수 이름과 현재 등수를 바로 찾을 수 있도록 HashMap<String, Integer> 를 함께 사용했다. <br>

1. players 배열을 순회하며 선수 이름 -> 현재 등수(index) 를 해시맵에 저장한다.
2. callings를 하나씩 확인한다.
3. 불린 선수의 현재 등수를 해시맵에서 바로 가져온다.
4. 그 선수와 바로 앞 선수의 위치를 players 배열에서 서로 바꾼다.
5. 바뀐 결과를 반영해서 해시맵의 등수 정보도 함께 갱신한다.

이렇게 하면 매 호출마다 선수를 찾기 위해 배열 전체를 탐색하지 않아도 되므로 빠르게 처리할 수 있다. <br>

### 제출 코드
```java
import java.util.*;

class Solution {
    public String[] solution(String[] players, String[] callings) {
        String[] answer = {};
        
        HashMap<String, Integer> map = new HashMap<>();
        for(int i = 0 ; i < players.length; i++){
            map.put(players[i], i);
        }
        for(int i = 0; i < callings.length; i++){
            int rank = map.get(callings[i]); 
            int prev_rank = rank - 1; 
            
            String prev_player = players[prev_rank]; 
            
            players[prev_rank] = callings[i]; 
            players[rank] = prev_player; 
            
            map.put(callings[i], prev_rank); 
            map.put(prev_player, rank);
             
        }
        
        return players;
    }
}

```

### 삽길 기록 🧠
- 처음에는 HashMap에 값을 넣을 때 add()를 써서 오류가 났다. HashMap은 add()가 아니라 put() 을 사용해야 한다.
