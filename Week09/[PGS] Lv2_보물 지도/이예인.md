### 성능 요약

메모리:  KB, 시간:  ms <br>
시간복잡도: O(n × m × 2)

### 제출 일자

### 문제 설명
- `n x m` 크기의 지도에서 시작점에서 도착점까지 이동해야 한다.  
- 일반적으로는 상하좌우 한 칸씩 이동할 수 있고,  
- 추가로 한 번 상하좌우 방향으로 두 칸 점프할 수 있다.  
- 함정이 있는 칸은 밟을 수 없으며  
- 도착점까지 가는 최소 이동 횟수를 구해야 한다.  
- 도달할 수 없다면 `-1`을 반환한다.

### 입력
- `n`: 세로 길이
- `m`: 가로 길이
- `hole`: 함정 좌표가 담긴 2차원 배열  
  - 문제 좌표는 `(1, 1)`부터 시작하므로 배열 인덱스로 바꿀 때 `-1` 처리 필요

### 출력
- 도착점까지의 최소 이동 횟수

### 핵심 풀이 전략
- hole에 주어진 함정 좌표를 blocked 배열에 표시한다.
- `visited[x][y][jumpUsed]` 배열로 방문 여부를 관리한다.
- BFS 큐에는 (좌표, 점프 사용 여부, 거리)를 함께 저장한다.
- 현재 위치에서 상하좌우로 한 칸 이동을 시도한다.
- 아직 점프를 사용하지 않았다면, 상하좌우로 두 칸 점프도 시도한다.
- 목적지에 처음 도착한 순간의 거리가 최단거리이므로 즉시 반환한다.
- 끝까지 도달하지 못하면 -1을 반환한다.

### 제출 코드
```java
import java.util.*;

class Solution {
    public int solution(int n, int m, int[][] hole) {
        
        // 상하좌우 이동 배열
        int[] dx = {1, -1, 0, 0};
        int[] dy = {0, 0, 1, -1};
        
        // 함정 위치 표시 배열
        boolean[][] blocked = new boolean[n][m];
        
        // 배열 인덱스에 맞게 -1 gowna 
        for(int i = 0; i < hole.length; i++){
            int x = hole[i][0] - 1;
            int y = hole[i][1] - 1;
            blocked[x][y] = true;
        }
        
        
        // visited[x][y][jumpUsed]
        // jumpUsed = 0 : 점프를 아직 사용하지 않고 (x, y)에 도착
        // jumpUsed = 1 : 점프를 이미 사용하고 (x, y)에 도착
        // 점프 사용한 상태로 방문한건지, 안쓴 상태로 방문한건지
        boolean[][][] visited = new boolean[n][m][2];
        
        // new int[]{x, y, jumpUsed, dist}
        Queue<int[]> queue = new LinkedList<>();

        queue.offer(new int[]{0, 0, 0, 0});
        visited[0][0][0] = true;
        
        //BFS
        while(!queue.isEmpty()){
            
            int[] now = queue.poll();
            int x = now[0];
            int y = now[1];
            int jumpUsed = now[2];
            int dist = now[3];
            
            if(x == n-1 && y == m -1) return dist;
            
            for(int i = 0; i < 4; i++){
                int nx = x + dx[i];
                int ny = y + dy[i];
                
                if(nx >= 0 && ny >= 0 && nx < n && ny < m 
                   && !blocked[nx][ny] && !visited[nx][ny][jumpUsed]){
                    
                    visited[nx][ny][jumpUsed] = true;
                    queue.offer(new int[]{nx, ny, jumpUsed, dist+1});
                    
                }
                
                // 점프 이동 (두 칸 이동)
                if(jumpUsed == 0){
                    int jx = x + dx[i] * 2;
                    int jy = y + dy[i] * 2;
                    
                    if(jx >= 0 && jy >= 0 && jx < n && jy < m
                       && !blocked[jx][jy] && !visited[jx][jy][1]){
                        visited[jx][jy][1] = true;
                        queue.offer(new int[]{jx, jy, 1, dist+1});
                    }
                }
            }   
        }
        return -1;
    }
}

```

### 삽길 기록 🧠
- 처음에는 그냥 일반적인 BFS처럼 (x, y) 좌표만 큐에 넣으면 된다고 생각했다.
- 하지만 이 문제는 점프 사용 여부도 상태로 관리해야 하는 문제라서, 같은 칸이라도 서로 다른 경우로 봐야 했다.
- 그래서 visited를 2차원이 아니라 3차원 배열로 만들어야 했다.
- 또한 큐에도 좌표만 넣는 것이 아니라 jumpUsed, dist까지 함께 넣어야 BFS가 제대로 동작했다.
