# [Lv 2] 보물 지도

[문제 링크](https://school.programmers.co.kr/learn/courses/15009/lessons/121690)

`#BFS` `#다차원방문체크` `#상태관리` `#최단거리`

### 제출 일자
2026년 4월 8일 214.57ms

### 문제 설명 (요약)
가로 `n`, 세로 `m` 크기의 지도가 주어집니다. `(1, 1)`에서 출발하여 우측 하단의 `(n, m)` 보물 위치까지 이동하는 최단 시간 반환하는 문제
- 상하좌우로 1칸씩 이동 가능.
- **신비로운 신발**을 사용하면 상하좌우로 **단 한 번, 2칸**을 한 번에 뛰어넘을 수 있다. (이때 중간에 있는 함정은 무시)
- `hole` 배열로 주어지는 함정 칸은 밟을 수 없다.

### 입력
- `n`, `m`: 지도의 가로, 세로 크기
- `hole`: 함정의 위치가 담긴 2차원 배열

### 출력
- 보물까지 가는 최단 시간 (도달할 수 없다면 -1 반환)

---

### 핵심 풀이 전략

1. **최단 시간 = BFS**
   - 가중치가 없는 격자 그래프에서 최단 거리를 구하는 문제이므로 BFS를 사용.

2. **상태 공간의 확장 (3차원 방문 체크)**
   - "신발을 아직 안 쓴 상태"와 "신발을 이미 써버린 상태"는 알고리즘 입장에서 완전히 다른 평행세계.
   - 따라서 방문 체크 배열을 `boolean[][][] visited = new boolean[n][m][2]`로 선언하여 두 상태의 발자국이 섞이지 않도록 분리.

3. **이동 분기 처리**
   - **점프 이동**: 현재 큐에서 꺼낸 상태가 `신발 미사용(0)`일 때만 시도하며, 이동 후에는 상태를 `신발 사용(1)`로 고정하여 큐에 삽입.
   - **기본 이동**: 현재 나의 상태(0 또는 1)를 그대로 유지하면서 이동.

---

### Java 구현 코드
```java
import java.util.*;

class Solution {
    
    int[] dy = {1, -1, 0, 0};
    int[] dx = {0, 0, 1, -1};
    
    public int solution(int n, int m, int[][] hole) {
        int answer = 0;
        boolean[][][] visited = new boolean[n][m][2];
        
        
        // 함정은 방문 표시
        for(int[] h : hole){
            int holeY = h[0] - 1;
            int holeX = h[1] - 1;
            
            visited[holeY][holeX][0] = true;
            visited[holeY][holeX][1] = true;
        }
            
        // 큐 초기화
        Deque<int[]> que = new ArrayDeque<>();
        que.add(new int[]{0, 0, 0, 0});
        visited[0][0][0] = true;
        
        while(!que.isEmpty()){
            
            int[] poll = que.poll();
            int curY = poll[0];
            int curX = poll[1];
            int curTime = poll[2];
            int isJump = poll[3];
            
            // 보물(N-1, M-1)을 찾았다면 종료
            if(curY == n-1 && curX == m-1) return curTime;
            
            // 만약 점프를 아직 안했다면
            if(isJump == 0){
                
                for(int i = 0; i < 4; i++){
                    
                    int nextY = curY + dy[i] + dy[i];
                    int nextX = curX + dx[i] + dx[i];
                
                    if(nextY >= 0 && nextX >= 0 && nextY < n && nextX < m && !visited[nextY][nextX][1]){
                        que.add(new int[]{nextY, nextX, curTime+1, 1});
                        visited[nextY][nextX][1] = true;
                    }
                }
            }
            
            // 기본 이동
            for(int i = 0; i < 4; i++){
                int nextY = curY + dy[i];
                int nextX = curX + dx[i];
                
                if(nextY >= 0 && nextX >= 0 && nextY < n && nextX < m && !visited[nextY][nextX][isJump]){
                    que.add(new int[]{nextY, nextX, curTime+1, isJump});
                    visited[nextY][nextX][isJump] = true;
                }
            }
        }
        
        
        return -1;
    }
}
```

---

### 삽질 기록 🧠

1. `isJump` 상태 갱신 누락`
   - 점프를 한 뒤에 큐에 값을 넣을 때, 현재 상태인 `isJump` (0)을 그대로 넣었다. 
   - 이렇게 되면 점프를 썼는데도 "아직 안 쓴 상태"로 취급되어 신발을 무한정 쓸 수 있게 된다. 
   - 점프 분기를 탔다면 반드시 **`1`을 하드코딩해서 넣어줘야** 했다.

2. 인덱스 고정
   - 기본 이동 시 방문 체크를 무조건 `!visited[nextY][nextX][0]`으로 고정해서 풀었다.
   - 이렇게 하면 신발을 쓴 상태(`isJump == 1`)로 이동 중인 사람이, 신발을 안 쓴 사람이 지나간 경로(`[0]`)를 검사하게 되어 길이 막혀버린다.

### 배운 점 && 보완할 점

- **BFS의 다차원 상태 관리**: '한 번만 쓸 수 있는 특수 능력', '벽 부수기' 같은 조건이 붙으면 무조건 방문 배열의 **차원(Dimension)을 늘려서 관리**해야 한다는 것을 배웠다.
- 상태 전이를 코드로 옮길 때, **"이 행동을 하면 내 상태가 어떻게 변하는가?"**를 큐 삽입부에 정확히 반영해야 한다.
