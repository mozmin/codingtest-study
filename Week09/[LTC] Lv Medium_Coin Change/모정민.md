# [Medium] Coin Change

[문제 링크](https://leetcode.com/problems/coin-change/)

`#다이나믹프로그래밍(DP)` `#배열` `#Bottom-Up`

### 제출 일자

### 문제 설명 (요약)
다양한 액면가를 가진 동전들의 배열 `coins`와 만들어야 하는 목표 금액 `amount`가 주어짐.
주어진 동전들을 조합하여 `amount`를 만드는 데 필요한 **동전의 최소 개수**를 반환하는 문제.
(각 동전은 무한히 사용할 수 있다고 가정)

- **coins**: 동전의 액면가 배열 (1 <= coins.length <= 12)
- **amount**: 목표 금액 (0 <= amount <= 10^4)
- 금액을 만들 수 없는 경우 **-1** 반환.

### 입력
- `coins`: 사용할 수 있는 동전의 종류
- `amount`: 달성해야 하는 목표 금액

### 출력
- 목표 금액을 만들기 위한 최소 동전 개수 (불가능하면 -1)

---

### 핵심 풀이 전략
1. **DP 배열 정의 및 직관적 매핑 (인덱스 = 금액)**
   - `dp[i]`를 "금액 `i`를 만드는 데 필요한 최소 동전 개수"로 정의.
   - 인덱스 계산의 혼란(0-index 매핑의 늪)을 막기 위해 배열의 크기를 `amount + 1`로 설정하여 직관성을 확보.

2. **불가능한 상태(Impossible State) 초기화**
   - 도달 불가능한 상태를 구별하기 위해 `dp` 배열을 절대 나올 수 없는 큰 값(`amount + 1`)으로 모두 초기화. 
   - `Integer.MAX_VALUE`를 쓰면 `+1` 연산 시 오버플로우가 발생할 수 있으므로, 정답의 최댓값보다 살짝 큰 `amount + 1`이 안전함.
   - 단, 0원을 만드는 데 필요한 동전은 0개이므로 `dp[0] = 0`으로 설정.

3. **Bottom-Up 방식의 점화식 적용**
   - 1원부터 `amount`까지 순회하며, 각 금액마다 사용 가능한 동전(`coin`)의 가치를 빼본다.
   - **점화식**: `dp[i] = Math.min(dp[i], dp[i - coin] + 1)` (단, `i - coin >= 0`일 때)

---

### Java 구현 코드

```java
import java.util.*;

class Solution {
    public int coinChange(int[] coins, int amount) {
        
        int[] dp = new int[amount + 1];
        for(int i = 1; i <= amount; i++){
            dp[i] = amount+1;
        }
        
        for(int i = 1; i <= amount; i++){
            
            for(int coin : coins){
                
                if((i - coin) >= 0){
                    dp[i] = Math.min(dp[i-coin] + 1, dp[i]);
                }
            }
            
        }

        if(dp[amount] == amount+1) return -1;
        else return dp[amount];
        
    }
}
```

---

### 삽질 기록 🧠
1. dp[i]를 i+1원으로 간주하고 풀려다 보니, 이전 상태를 참조하기 위한 수식이 i - coin이 아니라 i - coin - 1 등 기형적으로 변했다. 이 과정에서 ArrayIndexOutOfBounds 에러와 자기 자신을 참조하는 오류가 발생
2. 잘못된 초기화: dp 리스트를 초기화할 때, Integer.MAX_VALUE를 넣었는데, 이렇게 하니
```java
if((i - coin) >= 0){
  dp[i] = Math.min(dp[i-coin] + 1, dp[i]);
}
```
이 부분에서 잘못된 계산이 이루어졌다. Integer.MAX_VALUE와 dp[i-coin] + 1을 비교하니 무조건 Integer.MAX_VALUE가 작은 값이 되는 현상이 발생했다.
그래서 초기값을 amount+1로 절대 나올 수 없는 최소한의 값으로 설정했다.


### 배운 점 && 보완할 점
1. 인덱스 매핑의 중요성: DP 문제를 풀 때 인덱스와 구하고자 하는 값을 1:1로 일치시키는 것(배열 크기를 +1 하는 것)이 인덱스 계산 실수와 복잡한 예외 처리를 한 방에 날려버리는 스킬임을 뼈저리게 배웠다.
