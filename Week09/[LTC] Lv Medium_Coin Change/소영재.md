# [Medium] Coin Change

[문제 링크](https://leetcode.com/problems/coin-change/description/)

`#DP`

### 제출 일자

### 문제 설명 (요약)
- 주어진 여러 개의 코인으로 특정 금액 amount를 만들 때 해당 금액을 만들기 위한 최소한의 코인 개수를 구하는 문제
- 단, 어떤 조합으로도 금액을 만들 수 없다면 -1을 반환한다.

### 입력
- int[] coins : 코인들이 들어있는 정수 배열 
- int amount : 코인들로 만들어야 하는 금액

### 출력
- amount를 만들 수 있는 최소 경우의 수, 만약 없다면 -1을 반환
<hr>

### 핵심 풀이 전략
1. 문제를 코인의 경우의 수가 아닌 금액(amount)의 경우의 수로 접근한다.
2. amount원을 만드는 데 필요한 최소 코인의 개수를 memo[] 배열에 선언하고 저장한다.
3. 특정 금액을 만들 때, 마지막에 어떤 코인을 사용했는지를 기준으로 더 작은 금액으로 나눈다.
4. 이미 계산된 더 작은 금액들 중에서 가장 적은 코인을 사용한 경우를 선택한다.
5. 선택된 값에 현재 코인 1개를 더해 해당 금액의 최소 코인 개수를 결정한다
6. 계산한 값은 memo 배열에 저장하여 이후 동일한 금액 계산 시 재사용한다.

---

```java
import java.util.*;

class Solution {
    //아직 아무것도 계산하지 않았다는 것을 나타내기 위한 변수 
    int LIMIT = Integer.MAX_VALUE;

    public int coinChange(int[] coins, int amount) {
        int[] memo = new int[amount + 1];
        Arrays.fill(memo, LIMIT);
        memo[0] = 0;
        return dp(memo, coins, amount);
    }

    public int dp(int[] memo, int[] coins, int amount) {
        //메모이제이션 역할, 이미 계산한 적이 있다면 해당 값을 리턴
        if (memo[amount] != LIMIT) {
            return memo[amount];
        }

        int result = LIMIT;

        for (int coin : coins) {
            if (amount - coin >= 0) {
                int prev = dp(memo, coins, amount - coin);

                if (prev != -1) {
                    result = Math.min(result, prev);
                }
            }
        }

        memo[amount] = (result == LIMIT) ? -1 : result + 1;
        return memo[amount];
    }
}
```

### 삽질 기록 🧠
```java
class Solution {
public int coinChange(int[] coins, int amount) {
int result = dfs(coins, amount, 0, 0, 0);

        if (result == Integer.MAX_VALUE){
            return -1;
        }
        return result;
    }

    public int dfs(int[] coins, int amount, int depth, int sum, int count) {
        
        if (sum == amount) {
            return count;
        }
        
        if (sum > amount || depth == coins.length) {
            return Integer.MAX_VALUE;
        }
        
        int currentCoin = dfs(coins, amount, depth, sum + coins[depth], count + 1);
        
        int nextCoin = dfs(coins, amount, depth + 1, sum, count);
        
        return Math.min(currentCoin, nextCoin);
}
```

### 배운 점 && 보완할 점
- DP의 개념은 이해가 되는데, 어떻게 코드로 적용해야 할지 떠오르지가 않아서 공부법을 좀 찾아보았다.
- DP문제는 풀이 방법이 다양해서 한 문제에 시간을 오래 쓰지말고 풀이 코드를 참고하면서 DP를 공부하는 것이 좋다고 해서 참고하였다.
- 풀이에서 어려웠던 점은 '코인'을 기준으로 경우의 수를 찾아야 한다고 생각했는데, '코인'이 기준이 아닌, '금액'을 기준으로 푸는 점이었다.
- 만약 완전탐색으로 풀 수 있을 것 같은 문제에 접근할 때, 경우의 수가 크다면 DP를 고려하는 것이 좋다는 점을 배웠다.
- DP는 여러 문제에 적용할 수 있다. 특히 경우를 나누어 문제를 풀 수 있으며, 동일한 하위 문제가 반복될 때 사용하면 효율적이다.
- 

- Arrays.fill(대상 배열, 초기값); 메서드를 활용하여 배열의 초기값을 세팅할 수 있다는 점도 배웠다.
