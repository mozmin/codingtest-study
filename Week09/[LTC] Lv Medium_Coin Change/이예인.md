### 문제 설명
서로 다른 동전 금액이 담긴 배열 coins와 목표 금액 amount가 주어진다. <br>
이때 amount를 만들기 위해 필요한 동전 개수의 최솟값을 구하는 문제이다. <br>
만약 어떤 조합으로도 만들 수 없다면 -1을 반환한다. <br>
각 동전은 무한히 사용할 수 있다. <br>

### 입력
- coins: 동전 금액 배열
- amount: 만들어야 하는 금액

### 출력
- 목표 금액을 만들기 위한 최소 동전 개수
- 만들 수 없으면 -1

### 핵심 풀이 전략

dp[i] = i원을 만드는 데 필요한 최소 동전 개수 <br>

점화식
- 어떤 금액 i를 만들 때, 마지막으로 사용한 동전이 coin이라고 생각하면
- 그 전에 만들어져 있어야 하는 금액은 i - coin이다.

따라서 후보는 dp[i - coin] + 1 이 되고, 가능한 모든 동전에 대해 최솟값을 갱신하면 된다. <br>

즉 점화식은 다음과 같다. <br>

`dp[i] = Math.min(dp[i], dp[i - coin] + 1);`

단, i - coin >= 0 인 경우만 계산할 수 있다.<br>

초기값 설정 : 처음에는 모든 금액을 만들 수 없다고 가정하고 아주 큰 값으로 채운다.

`Arrays.fill(dp, amount + 1);`

이후,

`dp[0] = 0;`

0원을 만드는 데 필요한 동전 개수는 0개이기 때문이다. <br>

### 제출 코드
```java
import java.util.*;

class Solution {
    public int coinChange(int[] coins, int amount) {
        int[] dp = new int[amount + 1];
        Arrays.fill(dp, amount+1);
        dp[0] = 0;

        for(int i = 0; i < amount+1; i++){
            for(int j = 0 ; j < coins.length; j++){
                int coin = coins[j];
                if(i - coin >= 0){
                    dp[i] = Math.min(dp[i], dp[i - coin] + 1);
                }
            }
        }
        if(dp[amount] == amount + 1) return -1;
        else return dp[amount];
    }
}

```

### 삽길 기록 🧠
- 처음에는 dp[coins[i]] = 1처럼 동전 금액만 직접 표시하려고 했는데, 이 방식으로는 나머지 금액들을 체계적으로 계산하기 어려웠다.
