### 문제 설명 (요약)
- n개의 신호등이 초록(G) → 노란(Y) → 빨간(R) 순서로 무한 반복됨.
- 각 신호등마다 G, Y, R의 지속 시간이 다름.
- 모든 신호등이 동시에 노란불이 되는 가장 빠른 시각(초)을 구해야 함.
- 1초부터 시작하며, 시작 시 모든 신호등은 초록불의 첫 번째 초 상태임.
- 그런 경우가 없다면 -1을 반환

### 입력
- int[][] signals: 각 행은 [G, Y, R] 정수 배열 (신호 지속 시간)
- 신호등 개수 n (2~5), 주기 합 (3~20)

### 출력
- 모든 신호등이 노란불인 최초의 시간(int), 없을 경우 -1

<hr>

### 핵심 풀이 전략
- 신호등은 G+Y+R을 한 주기로 반복함. 특정 시간 T에서의 인덱스 값은 T % (주기) 를 통해 알 수 있음.
- 신호등 개수와 주기가 매우 작으므로, 시간을 1초씩 증가시키며 모든 신호등이 노란불인지 체크함.
- 모든 신호등의 상태가 다시 처음으로 돌아오는 시점(최소공배수)까지 반복함.
  
---

```java
import java.util.*;

class Solution {
    public int solution(int[][] signals) {
        int answer = 0;
        
        // 신호등 배열 초기화
        ArrayList<String>[] list = new ArrayList[signals.length];
        
        for(int i = 0; i < signals.length; i++){
            list[i] = new ArrayList<>();
        }
        
        for(int i = 0; i < signals.length; i++){
            addListValue(list[i], signals[i][0], "G");
            addListValue(list[i], signals[i][1], "Y");
            addListValue(list[i], signals[i][2], "R");
        }
        
        // 신호등 시뮬레이션
        int count;
        for(int i = 0; i < Math.pow(20, signals.length); i++){
            
            count = 0;
            for(ArrayList<String> l : list){
                if(isYellow(l, i)) count++;
            }    
            
            if(count == signals.length){
                answer = i + 1;
                break;
            }
        }
        
        if(answer == 0) return -1;
        
        return answer;
    }
    
    public boolean isYellow(ArrayList<String> list, int time){
        
        int index = time % list.size();
        
        if(list.get(index).equals("Y")) return true;
        
        return false;
    }
    
    public void addListValue(ArrayList<String> list, int count, String value){
        
        for(int i = 0; i < count; i++){
            list.add(value);
        }
    }
}
```

### 삽질 기록 🧠
- 시간-인덱스 불일치: 문제의 시간은 1초부터 시작하나 배열은 0번 인덱스부터 시작함. 이를 맞추기 위해 i+1 처리가 필요했음.
- 종료 조건 미흡: 처음에는 break를 넣지 않아 가장 빠른 시간이 아닌 가장 늦은 시간이 answer에 저장되었음. 최단/최초 탐색 시 조기 종료의 중요성을 깨달음.
