# [Lv1] 로또의 최고 순위와 최저 순위

[문제 링크](https://school.programmers.co.kr/learn/courses/30/lessons/77484#) 

`#구현` `#조건 분기`

### 제출 일자

2026년 3월 9일 (0.11ms)

### 문제 설명 (요약)
민우의 로또 번호 중 일부가 0(지워진 번호)으로 표시.
당첨 번호와 비교하여 0이 모두 맞았을 때(최고 순위)와 모두 틀렸을 때(최저 순위)를 계산.

### 입력
- lottos: 민우의 번호 배열 (0 포함)
- win_nums: 당첨 번호 배열

### 출력
[최고 순위, 최저 순위]

<hr>

### 핵심 풀이 전략
1. 당첨 번호 셋(Set) 구성: 당첨 번호를 HashSet에 넣어 탐색 속도를 $O(1)$로 만듦.
2. 카운팅: lottos를 돌며 '확정된 맞은 개수(matchCount)'와 '모르는 개수(zeroCount)'를 각각 구함.
3. 순위 매핑: 7 - 맞은 개수 또는, 미리 정의된 순위 배열을 사용하여 결과를 도출.

---

### 핵심 풀이 전략

```java
import java.util.*;

class Solution {
    
    int lowRank;
    int highRank;
    Set<Integer> set;
    int zeroCount;
    public int[] solution(int[] lottos, int[] win_nums) {
        
        lowRank = 0;
        highRank = 0;
        zeroCount = 0;
        set = new HashSet<>();
        
        // Set에 당첨 번호 저장
        for(int win : win_nums) set.add(win);
        
        for(int l : lottos){
            // 만약 추천번호 중에 당첨번호가 있다면 삭제
            if(set.contains(l)) set.remove(l);
            if(l == 0) zeroCount++;
        }
        
        // 최저점 계산
        lowRanking();
        
        // 최고점 계산
        if(lowRank == 1) return new int[]{lowRank, lowRank}; // 추천 번호 모두 당첨번호와 동일할 경우
        else if(lowRank == 6 && zeroCount == 0) return new int[]{lowRank, lowRank}; // 0이 하나도 없고, 당첨번호와 모두 일치하지 않을 경우
        
        int count = 0;
        for(int l : lottos) if(l == 0) count++;
        highRank = set.size() - count + 1;
        
        return new int[]{highRank, lowRank};
    }
    
    public void lowRanking(){
        if(set.size() == 6 || set.size() == 5) lowRank = 6;
        else if(set.size() == 0) lowRank = 1;
        else{
            lowRank = set.size() + 1;
        }
    }

}
```

### 삽질 기록 🧠
- 문제 푸는 아이디어 자체는 쉽게 떠올렸으나 조건 분기 처리하는데 애를 먹음. 여러가지 분기를 생각하다보니 코드가 길어지고 지저분해짐.
- 마지막에 **추첨 번호에 0이 없고**, **모든 값이 당첨 번호와 불일치** 한 **케이스**를 처리하지 못해서 시간을 많이 할애함.

### 배운 점 && 보완할 점
- AI가 현재 풀이는 조건 처리가 너무 많아서 가독성이 떨이지고, `highRank = set.size() - count + 1` 같은 공식은 좋지 않은 공식이라 하였다.
- 아래 코드가 AI가 제시한 깔끔한 풀이 방식이다. 이처럼 case 조건 분기는 `if문을 쓸지, switch문을 쓸지 고민하기 전에 int 배열로 해결할 수 없나 고민을 먼저 해야겠다.`
```java
// 훨씬 깔끔한 버전
// 이러한 방식이 더 좋은 코드이다.
class Solution {
    public int[] solution(int[] lottos, int[] win_nums) {
        Set<Integer> winSet = new HashSet<>();
        for (int num : win_nums) winSet.add(num);

        int matchCount = 0;
        int zeroCount = 0;

        for (int l : lottos) {
            if (l == 0) zeroCount++;
            else if (winSet.contains(l)) matchCount++;
        }

        // 순위 매핑 배열 (0개~6개 맞았을 때의 순위)
        int[] rank = {6, 6, 5, 4, 3, 2, 1};

        return new int[]{rank[matchCount + zeroCount], rank[matchCount]};
    }
}
```
