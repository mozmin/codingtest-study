### 문제 설명 (요약)
- 스테이지에 도달했으나 아직 클리어하지 못한 플레이어의 수/ 스테이지에 도달한 플레이어 수를 통해 실패율을 계산한다.
- 실패율이 높은 스테이지부터 내림차순으로 스테이지의 번호가 담겨있는 배열을 리턴한다.

### 입력
int N : 전체 스테이지의 개수
int[] stages : 사용자가 현재 멈춰있는 스테이지의 번호가 담긴 배열

### 출력
- 실패율이 높은 스테이지부터 내림차순으로 스테이지의 번호가 담겨있는 배열
<hr>

### 핵심 풀이 전략
- stages 배열의 길이를 통해 전체 유저 수를 구한다.
- 각 스테이지를 순회하면서 해당 스테이지에 머물러 있는 사용자 수를 계산한다.
- 실패율은 (현재 스테이지에 머물러 있는 사용자 수 / 해당 스테이지에 도달한 사용자 수)로 계산한다.
- 만약 해당 스테이지에 도달한 사용자가 없다면 실패율을 0으로 처리한다.
- 모든 스테이지의 실패율 계산이 끝나면 실패율을 기준으로 내림차순 정렬한다.
- 실패율이 같을 경우 스테이지 번호를 기준으로 오름차순 정렬한다.
- 정렬된 스테이지 번호를 결과 배열에 담아 반환한다.
---

```java
import java.util.*;
class Solution {
    public int[] solution(int N, int[] stages) {
        int users = stages.length;
        int index = users;
        double failure[] = new double[N];
        Integer[] stage = new Integer[N];
        
        for(int i = 0; i < N; i++){
            int count = 0;
           for(int j = 0; j < users; j++){
               if(i+1 == stages[j]){
                   count++;
               }
           }
           if(index == 0){
               failure[i] = 0;
           } else {
               failure[i] = (double)count/index;
           
            index -= count;
           }
        }
    
        for(int i = 0; i < N; i++){
            stage[i] = i + 1;
        }
        
        Arrays.sort(stage, (a,b)->{
            if(failure[b-1] == failure[a-1]){
                return a - b;
            }
            return Double.compare(failure[b-1], failure[a-1]);
        });
        
        int[] result = new int[N];
        
        for(int i = 0; i < N; i++){
            result[i] = stage[i];
        }
        
        return result;
    }
}
```

### 삽질 기록 🧠
- 처음에는 실패율을 굳이 계산하지 않고 사람 수를 통해 구현하려고 방향을 잡았으나, 이후 사람 수가 동일할 때 실패율이 다를 수 있다는 점을 깨닫게 되었다.

### 배운 점 && 보완할 점
- 실패율 계산한 뒤, 정렬을 할 때 도저히 방법이 생각이 나지 않아 AI의 도움을 받았다.
- compare(a,b) 함수는 반환 값(음수, 0, 양수)에 따라 어떤 값이 먼저 와야 하는지 판단하는 메서드이다.
- 또한 Arrays.sort()에서 비교함수를 사용하려면 Wrapper 객체를 사용해야 한다는 점도 배웠다. (Int[]배열로는 안됨)
