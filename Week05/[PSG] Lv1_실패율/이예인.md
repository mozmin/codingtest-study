### 문제 설명
전체 스테이지 수 N과 사용자가 현재 도전 중인 스테이지 정보가 담긴 stages 배열이 주어진다. <br>
각 스테이지에 대해 실패율을 구한 뒤, 실패율이 높은 스테이지부터 내림차순으로 정렬하여 스테이지 번호를 반환하는 문제이다.<br>

실패율은 다음과 같이 정의된다.<br>
- 스테이지에 도달했으나 아직 클리어하지 못한 사용자 수 / 해당 스테이지에 도달한 전체 사용자 수
단, 실패율이 같다면 스테이지 번호가 작은 순서대로 정렬해야 한다.<br>

### 입력
- N: 전체 스테이지의 개수
- stages: 사용자가 현재 도전 중인 스테이지 번호가 담긴 배열

### 출력
- 실패율이 높은 스테이지부터 정렬한 스테이지 번호 배열

### 핵심 풀이 전략
이 문제는 각 스테이지의 실패율을 계산한 뒤, 실패율 내림차순 / 스테이지 번호 오름차순 기준으로 정렬하면 된다. <br>

내 풀이에서는 먼저 stages 배열을 오름차순 정렬한 후, 같은 스테이지 번호가 몇 번 나오는지를 세면서 실패율을 계산했다.<br>

1. stages를 오름차순 정렬한다.

2. 현재 보고 있는 스테이지 번호(curStage)와 그 스테이지에 머물러 있는 사람 수(count)를 관리한다.

3. 스테이지 번호가 바뀌는 시점에 이전 스테이지의 실패율을 계산한다.

4. 중간에 아무도 없는 스테이지가 있다면 실패율을 0으로 채운다.

5. [스테이지 번호, 실패율] 형태로 2차원 배열에 저장한다.

6. Arrays.sort()와 Comparator를 이용해

- 실패율 기준 내림차순

- 실패율이 같으면 스테이지 번호 기준 오름차순으로 정렬한다.

7. 정렬된 결과에서 스테이지 번호만 꺼내 정답 배열에 담는다.

### 제출 코드
```java
import java.util.*;

class Solution {
    public int[] solution(int N, int[] stages) {
        int[] answer = new int[N];
        
        // stage 오름차순 정렬
        Arrays.sort(stages);
        
        // [0] -> stage 번호, [1] -> 실패율
        double[][] failPercent = new double[N][2];
        int playerNum = stages.length;
        
        int curStage = 1;
        int count = 0;
        int idx = 0;
        
        for(int i = 0 ; i < stages.length; i++){
            if(stages[i] > N) break; // N+1은 제외
            
            if(curStage == stages[i]){
                count++;
            }
            else{
                while(curStage < stages[i] && stages[i] <= N){
                    failPercent[idx][0] = curStage;
                    if(count == 0){
                        failPercent[idx][1] = 0;
                    }
                    else{
                        failPercent[idx][1] = (double)count/playerNum;
                        playerNum -= count;
                        count = 0;
                    }
                    idx++;
                    curStage++;
                }
                count = 1;
            }
        }
        
        // 마지막 stage 처리 (마지막 count 처리 안됨)
        while(curStage <= N){
            failPercent[idx][0] = curStage;
            if(count == 0 || playerNum == 0){
                failPercent[idx][1] = 0;
            }
            else{
                failPercent[idx][1] = (double)count / playerNum;
                playerNum -= count;
                count = 0;
            }
            idx++;
            curStage++;
        }
        
        // 실패율을 기준으로 내림차순
        Arrays.sort(failPercent, (a, b) -> {
            // 실패율이 똑같을 때 -> 오름차순
            if(Double.compare(b[1], a[1]) == 0){
                return Double.compare(a[0], b[0]);
            }
            else{
                return Double.compare(b[1], a[1]);
            }
        });
        
        for(int i = 0 ; i < N; i++){
            answer[i] = (int)failPercent[i][0];
        }
        
        return answer;
    }
}
```

### 삽길 기록 🧠
이 문제를 풀면서 가장 많이 헷갈렸던 부분은 실패율 계산 자체보다 배열 처리와 정렬 기준 설정이었다.<br>

처음에는 실패율을 저장할 때 단순히 1차원 배열로 처리하려고 했지만, 정렬 후에도 어떤 스테이지의 실패율인지 알아야 하므로<br>
결국 [스테이지 번호, 실패율] 형태의 2차원 배열이 필요했다.<br>

또한 count / playerNum 계산에서 둘 다 int라서 정수 나눗셈이 되어 결과가 계속 0이 나오는 문제도 있었다. <br>
이 부분은 (double) count / playerNum으로 형변환해서 해결했다. <br>

그리고 Arrays.sort()로 2차원 배열을 정렬할 때 단순히 Arrays.sort(stage[0])처럼 쓰면 되는 줄 알았는데, <br>
그건 한 행 내부만 정렬하는 것이고, 실제로는 각 행 전체를 비교하는 Comparator가 필요하다는 점을 알게 되었다. <br>

마지막으로 이 문제는 단순히 등장한 스테이지만 처리하면 안 되고, 아무도 없는 스테이지도 실패율 0으로 포함해야 한다는 점도 중요한 포인트였다. <br>

### 배운 점 && 보완할 점
특히 이번 문제에서 새롭게 정리할 수 있었던 부분은 Comparator를 이용한 정렬 방법이었다. <br>
기존에는 Arrays.sort()를 사용할 때 1차원 배열을 단순히 오름차순 정렬하는 경우가 대부분이었지만, <br>
이번 문제에서는 [스테이지 번호, 실패율] 형태의 2차원 배열을 특정 열 기준으로 정렬해야 했기 때문에 Comparator를 사용해야 했다. <br>

예를 들어 다음 코드처럼 Comparator를 사용하면 <br>
```
Arrays.sort(failPercent, (a, b) -> Double.compare(b[1], a[1]));
```
a와 b는 배열의 각 행(row)을 의미한다. a[1], b[1]은 실패율 값을 의미한다. <br>

Double.compare(b[1], a[1])을 사용하면 실패율 기준 내림차순 정렬이 된다. <br>

또한 문제 조건에서 실패율이 같은 경우 스테이지 번호가 작은 순서로 정렬해야 한다는 조건이 있기 때문에 Comparator 내부에서 정렬 기준을 두 단계로 나누어 처리했다. <br>
```
Arrays.sort(failPercent, (a, b) -> {
    if(Double.compare(b[1], a[1]) == 0){
        return Double.compare(a[0], b[0]);
    }
    return Double.compare(b[1], a[1]);
});
```


2차원 배열을 정렬할 때는 Comparator를 사용해 비교 기준을 직접 정의해야 한다. <br>

Comparator 내부에서 여러 조건을 이용한 다단계 정렬도 가능하다. <br> 

Double.compare()를 사용하면 a[1] - b[1] 방식보다 오버플로우 없이 안전하게 비교할 수 있다. <br>
