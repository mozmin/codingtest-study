### 문제 설명 (요약)
각 스테이지의 실패율을 계산한 뒤 실패율이 높은 스테이지부터 내림차순으로 정렬하는 문제.  
실패율은 특정 스테이지에 도달했으나 아직 클리어하지 못한 사용자 수를 해당 스테이지에 도달한 전체 사용자 수로 나눈 값으로 정의. 
실패율이 같을 경우 스테이지 번호가 작은 순으로 정렬.

### 입력
- N: 전체 스테이지 수  
- stages: 각 사용자가 현재 도달한 스테이지 번호가 담긴 배열

### 출력
실패율이 높은 스테이지부터 정렬된 스테이지 번호 배열을 반환

<hr>

### 핵심 풀이 전략
1. 스테이지별 사람 수 세기 → stages 배열 순회
2. 실패율 계산 → 실패자 / 남은 플레이어 수 (남은 플레이어가 0이면 실패율 0으로 계산)
3. 배열/객체에 저장 → [스테이지 번호, 실패율]
4. 정렬 → 실패율 내림차순, 같으면 스테이지 번호 오름차순
5. 결과 반환 → 정렬된 스테이지 번호 배열

---

### 핵심 풀이 전략

```java
import java.util.*;
class Solution {
    public int[] solution(int N, int[] stages) {
        double[][] answer = new double[N][2];
        
        for(int i = 0; i < stages.length; i++){
            
            int s = stages[i] - 1;
            
            if(s == N) continue;
            answer[s][1]++;
        }
        
        double sum = (double) stages.length;
        for(int i = 0; i < answer.length; i++){
            // 스테이지 i 실패율
            answer[i][0] = i+1;
            
            double fail = (sum == 0 ? 0 : (double) answer[i][1] / sum);
            sum -= answer[i][1];
            answer[i][1] = fail;
        }
        
        Arrays.sort(answer, Comparator.comparingDouble((double[] a) -> a[1]).reversed()
                    .thenComparingDouble((double[] a) -> a[0]));
        
        int[] result = new int[N];
        for(int i = 0; i < answer.length; i++){
            result[i] = (int) answer[i][0];
        }
        return result;
    }
}
```

### 삽질 기록 🧠
1. 문제를 이해하는데 시간이 좀 걸렸다.
2. 남은 플레이어 수가 0일 경우를 생각 못해서 조건 분기처리를 못했다. -> double fail = (sum == 0 ? 0 : (double) answer[i][1] / sum);
3. 이중 체인 정렬을 아직도 어려워한다.
4. double을 안쓰고 int로 해서 에러가 났다. 소수점까지 정확하게 비교해야 하므로 double을 써야한다.
---

### 배운 점 && 보완할 점
1. **Comparator의 `static 메서드들`(comparingDouble 등)에서만 체인**을 쓸 수 있다. **(a, b) -> a[1] - b[1] 하는건 직접 `Comparator 객체를 new`로 쓰는거다**.
2. **백분율이나 나눗셈을 활용하는 문제에서는 `double`이 필요한지 생각해보자.**
