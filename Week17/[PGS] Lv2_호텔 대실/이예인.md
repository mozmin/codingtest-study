### 문제 설명
- 호텔 객실 예약 시간이 주어졌을 때, 모든 예약을 처리하기 위해 필요한 최소 객실 수를 구하는 문제이다.
- 각 예약은 시작 시간과 종료 시간으로 주어지며, 객실은 퇴실 후 바로 사용할 수 없다.
- 퇴실 후에는 10분의 청소 시간이 필요하므로, 실제로 해당 객실을 다시 사용할 수 있는 시간은 종료 시간에서 10분이 지난 시점이다.
- 따라서 예약 시간이 겹치거나, 이전 예약의 청소가 끝나기 전에 다음 예약이 시작된다면 새로운 객실이 필요하다.

### 입력
- 예약 시작 시간과 종료 시간이 담긴 2차원 문자열 배열 book_time
- 각 시간은 "HH:MM" 형식으로 주어진다.

### 출력
- 모든 예약을 처리하기 위해 필요한 최소 객실 수

---

### 핵심 풀이 전략
- "HH:MM" 형식의 시간은 비교하기 쉽도록 분 단위 정수로 변환한다.
- 각 예약의 시작 시간과 종료 시간을 분 단위로 저장한다.
- 종료 시간에는 청소 시간 10분을 더해 객실이 다시 사용 가능한 시간으로 처리한다.
- 예약은 시작 시간이 빠른 순서대로 정렬한다.
- 객실별로 언제 다시 사용할 수 있는지를 관리하기 위해 우선순위 큐를 사용한다.
- 우선순위 큐에는 각 객실의 사용 가능 시간이 저장된다.
- 현재 예약의 시작 시간이 가장 빨리 비는 객실의 시간보다 크거나 같다면 해당 객실을 재사용할 수 있다.
- 재사용 가능한 경우 우선순위 큐에서 기존 시간을 제거하고, 현재 예약의 종료 시간에 청소 시간을 더한 값을 다시 넣는다.
- 재사용할 수 없다면 새로운 객실이 필요하므로 현재 예약의 종료 시간을 우선순위 큐에 추가한다.
- 모든 예약을 처리한 뒤 우선순위 큐의 크기가 필요한 최소 객실 수가 된다.

---

### 제출 코드
```java
import java.util.*;
class Solution {
    public int solution(String[][] book_time) {
        int[][] booking = new int[book_time.length][2];
        
        for(int i = 0 ; i < book_time.length; i++){
            int start = changeMin(book_time[i][0]);
            int end = changeMin(book_time[i][1]) + 10; // 청소 시간 10분 추가
            
            booking[i][0] = start;
            booking[i][1] = end;
        }
        
        // 시작 시간 기준 정렬
        Arrays.sort(booking, (a, b) -> a[0] - b[0]);
        
        PriorityQueue<Integer> pq = new PriorityQueue<>();
        
        for(int i = 0 ; i < booking.length; i++){
            int start = booking[i][0];
            int end = booking[i][1];
            
            // 가장 빨리 비는 방을 현재 예약이 사용할 수 있다면 재사용
            if(!pq.isEmpty() && pq.peek() <= start){
                pq.poll();
            }
            
            pq.offer(end);
        }
        
        return pq.size();

    }
    
    int changeMin (String time){
        String hour = time.substring(0,2);
        String min = time.substring(3);
        int hour_i = Integer.parseInt(hour);
        int min_i = Integer.parseInt(min);
        return hour_i * 60 + min_i;
    }
}
```

### 삽길 기록 🧠
- 처음에는 특정 시간대가 사용 중인지 확인하기 위해 HashSet을 사용할 수 있을 것 같다고 생각할 수 있다.
- 하지만 이 문제는 특정 시간이 존재하는지만 확인하는 문제가 아니다.
- 현재 예약 시작 시간보다 작거나 같은 사용 가능 시간이 있는지 확인해야 한다.
- 그중에서도 가장 빨리 비는 객실을 먼저 확인해야 하므로 HashSet보다는 우선순위 큐가 적합하다.
