### 제출 일자
2026년 6월 11일 13.29ms

### 문제 설명 (요약)
호텔을 운영하는 도중 최소한의 객실만 사용하여 예약 손님들을 매칭하는 문제입니다. 각 예약은 입실 시간과 퇴실 시간인 `["HH:MM", "HH:MM"]` 형태로 주어집니다. 한 번 사용한 방은 퇴실 후 **10분간 청소**를 해야 다음 손님이 사용할 수 있습니다. 필요한 최소 객실의 수를 구합니다.

---

### 핵심 풀이 전략
1. **시간 단위 통일 및 청소 시간 미리 누적**: "HH:MM" 형식의 문자열을 분(`minute`) 단위의 정수로 변환합니다. 이때 퇴실 시간에 **청소 시간 10분을 미리 더해두면** 다음 손님의 입실 시간과 단순 대소 비교(`퇴실시간 <= 입실시간`)가 가능해져 조건 처리가 단순해집니다.
2. **입실 시간 기준 정렬**: 순서대로 방을 배정하기 위해 원본 예약 배열을 **대실 시작(입실) 시간 기준 오름차순**으로 정렬합니다.
3. **우선순위 큐(Min-Heap)를 이용한 방 재사용 관리**: 큐에는 **현재 사용 중인 방들의 퇴실(+청소포함) 시간**만 정수 형태로 넣어 관리합니다. 큐의 가장 앞(가장 빨리 비는 방)의 시간보다 현재 손님의 입실 시간이 크거나 같다면 기존 방을 재사용(`q.poll()`)하고, 그렇지 않다면 새 방을 개설해야 합니다.

---

### 내가 구현한 구현 코드
```java
import java.util.*;

class Solution {
    public int solution(String[][] book_time) {
        int answer = 0;
        
        // 1. 시간 파싱 및 퇴실 시간에 청소 시간 10분 미리 더하기
        int[][] book_time_int = new int[book_time.length][2];
        for (int i = 0; i < book_time.length; i++) {
            book_time_int[i][0] = converTime(book_time[i][0], "in");
            book_time_int[i][1] = converTime(book_time[i][1], "out");
        }
        
        // 2. 대실 시작 시간 기준으로 오름차순 정렬
        Arrays.sort(book_time_int, (o1, o2) -> Integer.compare(o1[0], o2[0]));
        
        // 3. 가장 빨리 비는 방의 퇴실 시간을 관리할 우선순위 큐
        PriorityQueue<Integer> q = new PriorityQueue<>();
        
        for (int[] book : book_time_int) {
            // 가장 빨리 비는 방의 청소 완료 시간이 현재 손님의 입실 시간 이하이면 방 재사용 가능
            if (!q.isEmpty() && q.peek() <= book[0]) {
                q.poll();
            }
            // 기존 방을 이어 쓰든 새 방을 파든 새로운 퇴실 시간을 큐에 주입
            q.offer(book[1]);
        }
        
        // 최종적으로 사용 중인(큐에 남아있는) 방의 개수가 필요한 최소 객실 수
        return q.size();
    }
    
    public int converTime(String time, String type) {
        String[] split = time.split(":");
        int total = (Integer.parseInt(split[0]) * 60) + Integer.parseInt(split[1]);
        
        if (type.equals("out")) {
            total += 10; // 퇴실 시 청소 시간 10분 추가
        }
        return total;
    }
}
```

### 정답 코드 (리팩토링 - 조건문 최적화 및 람다 오버헤드 축소)
1) `converTime` 분기 로직을 직관적으로 개선하여 불필요한 `String` 매개변수 비교 연산을 제거한다.
2) 대규모 데이터에서 람다 표현식의 정렬 오버헤드를 방지하기 위해 `Comparator.comparingInt` 또는 정수 비교 정석을 사용한다.

```java
import java.util.*;

class Solution {
    public int solution(String[][] book_time) {
        int[][] times = new int[book_time.length][2];
        
        for (int i = 0; i < book_time.length; i++) {
            times[i][0] = toMinutes(book_time[i][0]);
            // 퇴실 시간에 청소 시간 10분을 더해 저장
            times[i][1] = toMinutes(book_time[i][1]) + 10;
        }

        // 입실 시작 시간 기준 오름차순 정렬
        Arrays.sort(times, (o1, o2) -> Integer.compare(o1[0], o2[0]));

        // 각 객실의 청소 완료(퇴실) 시간을 오름차순으로 관리하는 큐
        PriorityQueue<Integer> rooms = new PriorityQueue<>();

        for (int[] time : times) {
            // 비어있는 가장 빠른 방의 청소 완료 시간 <= 현재 대실 시작 시간이면 기존 방 사용
            if (!rooms.isEmpty() && rooms.peek() <= time[0]) {
                rooms.poll();
            }
            rooms.offer(time[1]);
        }

        return rooms.size();
    }

    private int toMinutes(String time) {
        String[] split = time.split(":");
        return Integer.parseInt(split[0]) * 60 + Integer.parseInt(split[1]);
    }
}
```
