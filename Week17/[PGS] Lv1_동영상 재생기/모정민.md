### 문제 설명 (요약)
동영상 재생기 기능을 구현하는 문제입니다. 10초 전으로 이동하는 `prev`, 10초 후로 이동하는 `next` 명령 배열이 주어집니다. 단, 영상의 처음(00:00)이나 마지막(video_len) 범위를 벗어날 수 없으며, 현재 위치가 오프닝 구간(`op_start` ≤ 현재 재생 위치 ≤ `op_end`)에 포함되면 자동으로 오프닝이 끝나는 위치(`op_end`)로 점프합니다. 모든 명령을 수행한 후의 최종 위치를 "mm:ss" 형식으로 반환합니다.

---

### 핵심 풀이 전략
1. **시간 단위 통일 (초 단위 변환)**: "mm:ss" 형식의 문자열을 다루기 쉽게 `분 * 60 + 초`를 통해 전체 초(second) 정수형태로 변환하여 계산합니다.
2. **오프닝 건너뛰기 예외 처리 타임라인**: 오프닝 건너뛰기는 명령어를 수행할 때뿐만 아니라, **비디오가 시작하는 최초 시점**에도 적용되어야 합니다. 또한 `prev`나 `next`로 인해 위치가 바뀔 때마다 매번 즉시 체크해야 합니다.
3. **출력 포맷팅**: 한 자리 숫자가 나올 경우 앞에 `0`을 붙여 2자리로 만들어야 하므로, 단순 타입 변환(`String.valueOf`) 대신 자릿수를 지정할 수 있는 `String.format("%02d:%02d", minute, second)`를 활용합니다.

---

### 내가 구현한 구현 코드
```java
import java.util.*;

class Solution {
    public String solution(String video_len, String pos, String op_start, String op_end, String[] commands) {
        String answer = "";
        
        int video_len_int = (Integer.parseInt(video_len.substring(0, 2)) * 60) + Integer.parseInt(video_len.substring(3, 5));
        int pos_int = (Integer.parseInt(pos.substring(0, 2)) * 60) + Integer.parseInt(pos.substring(3, 5));
        int op_start_int = (Integer.parseInt(op_start.substring(0, 2)) * 60) + Integer.parseInt(op_start.substring(3, 5));
        int op_end_int = (Integer.parseInt(op_end.substring(0, 2)) * 60) + Integer.parseInt(op_end.substring(3, 5));
        
        for (String c : commands){
            
            // 오프닝 조건
            if (op_start_int <= pos_int && pos_int <= op_end_int) pos_int = op_end_int;
            
            // 10초 후
            // 비디오 길이를 넘는다면 비디오 길이로 이동
            // 비디오 길이 안에있다면 10초 후로 이동
            if (c.equals("next")){
                if (pos_int + 10 >= video_len_int) pos_int = video_len_int;
                else if (pos_int + 10 < video_len_int) pos_int += 10;
            }
            // 10초 전
            // 0초보다 작다면 0초로 이동
            // 0초보다 크다면 10초 전으로 이동
            else if (c.equals("prev")){
                if (pos_int - 10 <= 0) pos_int = 0;
                else if (pos_int - 10 > 0) pos_int -= 10;
            }
        }
        
        // 오프닝 조건
        if (op_start_int <= pos_int && pos_int <= op_end_int) pos_int = op_end_int;

        int minute = pos_int / 60;
        int second = pos_int % 60;
        
        answer = String.format("%02d:%02d", minute, second);
        return answer;
    }
}
```

## 정답 코드 (리팩토링 - 가독성 개선 및 조건문 단순화)
split(":")을 활용하여 문자열 파싱 과정을 더 명확하고 보기 쉽게 개선할 수 있다.

for문 내부의 복잡한 범위 체크 if-else문 대신 Math.max와 Math.min을 활용하면 범위를 제한하는 코드를 대폭 단순화할 수 있다.

```Java
import java.util.*;

class Solution {
    public String solution(String video_len, String pos, String op_start, String op_end, String[] commands) {
        int videoLen = toSeconds(video_len);
        int currentPos = toSeconds(pos);
        int opStart = toSeconds(op_start);
        int opEnd = toSeconds(op_end);

        for (String command : commands) {
            // 1. 명령 수행 전 오프닝 구역인지 체크 (최초 진입 시점 포함 역할)
            if (currentPos >= opStart && currentPos <= opEnd) {
                currentPos = opEnd;
            }

            // 2. 명령 수행 및 범위 제한(Math 클래스 활용으로 분기 최소화)
            if (command.equals("next")) {
                currentPos = Math.min(videoLen, currentPos + 10);
            } else if (command.equals("prev")) {
                currentPos = Math.max(0, currentPos - 10);
            }
        }

        // 3. 모든 명령 수행 후 마지막 위치 오프닝 체크
        if (currentPos >= opStart && currentPos <= opEnd) {
            currentPos = opEnd;
        }

        return String.format("%02d:%02d", currentPos / 60, currentPos % 60);
    }

    // 파싱 로직을 별도 메서드로 분리하여 가독성 증대
    private int toSeconds(String time) {
        String[] tokens = time.split(":");
        return Integer.parseInt(tokens[0]) * 60 + Integer.parseInt(tokens[1]);
    }
}
```

## 회고 및 삽질기록 🧠
- String.format(), String.valueOf()의 차이점을 명확히 알지 못해 문자열 포맷 변환 과정에서 IllegalFormatConversionException 컴파일 에러를 겪었다. 데이터 타입(int와 String)에 맞는 올바른 포맷팅 메서드를 선택해야 함을 배웠다.
- 오프닝 건너뛰기 규칙을 루프가 끝난 뒤에만 처리하려다가 틀렸다. 시뮬레이션 문제에서는 문제 조건에 기술된 "예외가 발생하는 타이밍(언제 검사해야 하는가)"을 엄격하게 파악하여 매 이동 단계(루프)마다 조건 검사를 배치해야 함을 깨달았다.
