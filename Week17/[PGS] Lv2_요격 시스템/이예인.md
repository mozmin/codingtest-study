### 문제 설명
- 여러 개의 미사일이 각각 특정 구간을 따라 날아오고, 요격 미사일을 발사해 이 미사일들을 요격해야 하는 문제이다.
- 각 미사일은 [s, e] 형태의 구간으로 주어지며, 요격 미사일은 s < x < e인 위치에서 발사해야 해당 미사일을 요격할 수 있다.
- 하나의 요격 미사일이 여러 미사일의 구간에 동시에 포함된다면, 여러 개의 미사일을 한 번에 요격할 수 있다.
- 모든 미사일을 요격하기 위해 필요한 요격 미사일의 최소 개수를 구해야 한다.

### 입력
- 미사일의 시작 지점과 끝 지점이 담긴 2차원 배열 targets

### 출력
- 모든 미사일을 요격하기 위해 필요한 요격 미사일의 최소 개수

---

### 핵심 풀이 전략
- 이 문제는 여러 구간을 최소 개수의 점으로 커버하는 문제로 볼 수 있다.
- 가장 적은 요격 미사일을 사용하려면, 가능한 한 많은 구간을 하나의 요격 위치로 처리해야 한다.
- 이를 위해 미사일들을 끝나는 지점 기준으로 오름차순 정렬한다.
- 아직 요격하지 못한 미사일을 만나면, 해당 미사일의 끝 지점 바로 앞에 요격 미사일을 쏜다고 생각한다.
- 끝나는 지점이 빠른 미사일부터 처리하면, 현재 미사일을 반드시 요격하면서 이후 미사일들과 겹칠 가능성을 최대한 유지할 수 있다.
- 이전에 설정한 요격 위치로 현재 미사일을 요격할 수 있다면 새 미사일을 쏘지 않는다.
- 반대로 현재 미사일의 시작 지점이 이전 요격 기준점 이상이라면, 기존 요격 위치로는 요격할 수 없으므로 새로운 요격 미사일이 필요하다.
- 이 과정을 모든 미사일에 대해 반복하면서 필요한 요격 미사일 개수를 센다.

---

### 제출 코드
```java
class Solution {
    public int solution(int[][] targets) {
        int answer = 0;
        
        // 끝나는 지점을 기준으로 오름차순으로 정렬
        Arrays.sort(targets, (a, b) -> a[1] - b[1]);
        int end = -1;
        
        for(int[] target: targets){
            int start = target[0];
            int finish = target[1];
            
            // 현재 요격 미사일로 처리할 수 없는 경우
            if(end <= start){
                answer++;
                end = finish;
            }
        }
        
        return answer;
    }
}
```

### 삽길 기록 🧠
- 처음에는 시작 지점 기준으로 정렬해서 겹치는 구간을 묶으려고 생각할 수 있다.
- 하지만 시작 지점 기준으로 보면 어느 위치에 요격 미사일을 쏘는 것이 가장 효율적인지 판단하기 어렵다.
- `s < x < e` 조건 때문에 `end <= start`인 경우에는 새 요격 미사일이 필요하다는 점을 기억해야 한다.
