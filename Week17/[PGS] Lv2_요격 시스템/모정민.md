# [Level 2] 요격 시스템

[문제 링크](https://school.programmers.co.kr/learn/courses/30/lessons/181188)

`#그리디(Greedy)` `#정렬`

### 제출 일자
2026년 6월 11일 240.11ms

### 문제 설명 (요약)
2차원 공간에서 각 폭격 미사일의 x 좌표 범위인 개구간 `(s, e)` 목록이 주어집니다. 특정 x 좌표에서 y 축에 수평이 되도록 요격 미사일을 발사하면 해당 x 좌표를 관통하는 모든 폭격 미사일을 한 번에 요격할 수 있습니다. 단, 개구간의 양 끝점인 `s`와 `e` 위치에서는 요격할 수 없습니다. 모든 폭격 미사일을 요격하기 위해 필요한 요격 미사일 수의 최솟값을 구합니다.

---

### 핵심 풀이 전략
1. **정렬(Sort)을 통한 기준 수립**: 미사일의 구간이 무작위로 주어지므로, 탐욕법(Greedy)을 적용하기 위해 미사일이 **끝나는 지점(`e`)을 기준으로 오름차순 정렬**합니다.
2. **개구간 경계 조건 처리**: 개구간 `(s, e)` 특성상 이전 미사일의 끝점과 다음 미사일의 시작점이 같기만 해도 (`lastPoint == s`) 같은 요격 미사일로 맞출 수 없습니다. 따라서 다음 미사일의 시작점이 현재 요격선보다 크거나 같은 경우 (`lastPoint <= s`)에만 새로운 요격 미사일을 발사합니다.
3. **요격선 갱신**: 새로운 미사일을 쏠 때 요격선의 위치(`lastPoint`)를 해당 미사일의 끝점(`e`)으로 갱신해 나가며 최소 개수를 누적합니다.

---

### 내가 구현한 구현 코드
```java
import java.util.*;

class Solution {
    public int solution(int[][] targets) {
        int answer = 0;
        
        // 끝나는 지점(e)을 기준으로 오름차순 정렬하는 우선순위 큐 활용
        PriorityQueue<int[]> q = new PriorityQueue<>(Comparator.comparingInt((int[] a) -> a[1]));
        
        for (int[] t : targets) q.offer(t);
        
        int lastPoint = 0;
        
        // 큐에서 값을 꺼내고, 요격 가능 경계값(lastPoint) 갱신
        while (!q.isEmpty()){
            
            int[] poll = q.poll();
            
            // 다음 미사일의 시작점이 현재 요격선(lastPoint) 이하에 걸치지 않는다면 새로 발사
            if (lastPoint <= poll[0]){
                lastPoint = poll[1];
                answer++;
            }
        }
        
        return answer;
    }
}
```

### 정답 코드 (리팩토링 - 공간 복잡도 최소화 및 연산 속도 최적화)
1) `PriorityQueue`를 사용하면 삽입/삭제 시 $O(\log N)$의 오버헤드가 발생하므로, 대신 원본 배열을 `Arrays.sort()`로 정렬하여 메모리와 실행 시간을 단축할 수 있다.
2) 람다식 대신 익명 클래스나 기존 정렬 로직을 활용하면 함수형 인터페이스 호출 오버헤드를 줄여 효율성 테스트를 더 빠르게 통과할 수 있다.

```java
import java.util.*;

class Solution {
    public int solution(int[][] targets) {
        int answer = 0;

        // PriorityQueue 대신 Arrays.sort를 활용해 오버헤드 최소화 (끝점 기준 오름차순 정렬)
        Arrays.sort(targets, new Comparator<int[]>() {
            @Override
            public int compare(int[] o1, int[] o2) {
                return Integer.compare(o1[1], o2[1]);
            }
        });

        // 0번 index 미사일 처리를 위해 초기 요격선 값을 -1로 설정
        int lastPoint = -1;

        for (int[] target : targets) {
            // 개구간이므로 다음 미사일의 시작점(target[0])이 현재 요격선과 같거나 오른쪽에 있다면 새로 요격해야 함
            if (lastPoint <= target[0]) {
                lastPoint = target[1];
                answer++;
            }
        }

        return answer;
    }
}
```

### 회고 및 삽질기록 🧠
- 처음에는 데이터의 범위를 보고 `Map`이나 다른 복잡한 자료구조를 써야 하나 고민했으나, 구간 겹침을 최적화하는 전형적인 그리디(Greedy) 문제임을 깨달았다.
- 개구간 `(s, e)`의 경계 조건을 피하기 위해 무리하게 `double` 연산이나 `-0.5` 처리를 하려다 미세한 오차가 발생해 오답을 겪었다. 정수형 범위 그대로 둔 채 다음 시작점(`s`)과 현재 요격선(`lastPoint`)의 대소 관계 비교 연산자(`<=`)만 정확히 정의해 주면 깔끔하게 풀린다는 것을 배웠다.
- 데이터의 개수가 최대 500,000개에 달할 때는 `PriorityQueue`보다 `Arrays.sort()`를 사용하는 것이 시간 및 공간 복잡도 측면에서 훨씬 유리함을 체감했다.
