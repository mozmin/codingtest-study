### 문제 설명 (요약)
- 농부 현서는 1번 헛간에서 출발해 N번 헛간에 있는 농부 찬홍이에게 택배를 배달해야 한다.
- 헛간 사이에는 여러 개의 양방향 길이 있고, 각 길마다 지나갈 때 만나게 되는 소의 수가 정해져 있다.
- 현서는 길의 길이 자체는 신경쓰지 않고 지나가면서 소들에게 줘야 하는 여물의 총합이 최소가 되는 경로를 선택하려고 한다.
- 즉 1번 헛간에서 N번 헛간까지 이동할 때 드는 최소 비용을 구하는 문제이다.

### 입력
- int N, M : 첫째 줄에 헛간의 개수 N과 길의 개수 M이 주어진다.
- 다음 M개의 줄에는 세 정수 A,B,C가 주어진다. A,B는 서로 연결된 두 헛간 번호, C는 길을 지나갈 때 만나게 되는 소의 수(여물 비용)

### 출력
- 1번 헛간에서 N번 헛간까지 이동할 때 필요한 최소 여물 비용을 반환

### 핵심 풀이 전략
- 이 문제는 가중치가 있는 그래프에서 최단거리를 구하는 문제이다.
- 이 문제는 간선마다 비용이 다르므로 BFS로는 해결할 수 없고, 모든 경로를 DFS를 탐색하면 경우의 수가 너무 많아 비효율적이다.
- 따라서 다익스트라 알고리즘을 사용하여 접근한다.

1. 각 헛간과 연결된 길 정보를 인접 리스트로 저장한다.
2. dist[i]를 1번 헛간에서 i번 헛간까지의 최소 비용으로 정의한다.
3. 처음에는 모든 값을 매우 큰 값으로 초기화하고, 시작점인 dist[1] = 0으로 설정한다.
4. 우선순위 큐를 이용해 현재까지 비용이 가장 작은 헛간부터 꺼낸다.
5. 현재 헛간과 연결된 다음 헛간들을 확인하면서, 더 적은 비용으로 갈 수 있다면 dist 값을 갱신한다.
6. 최종적으로 dist[N]이 1번 헛간에서 N번 헛간까지의 최소 여물 비용이 된다.

---

```java
import java.io.*;
import java.util.*;

public class Main {
    public static void main(String[] args) throws Exception {
        BufferedReader br = new BufferedReader(new InputStreamReader(System.in));
        StringTokenizer st = new StringTokenizer(br.readLine());

        int N = Integer.parseInt(st.nextToken());
        int M = Integer.parseInt(st.nextToken());

        //각 노드의 인접 정보(연결 노드, 비용)를 담는 리스트 배열
        List<int[]>[] graph = new ArrayList[N + 1];

        //초기화 작업, 하지 않으면 NullPointerException 발생
        for (int i =1; i <= N; i++) {
            graph[i] = new ArrayList<>();
        }

        //graph[a]에는 b노드와 연결되어 있고, c만큼 비용이 발생
        //graph[b]에는 a노드와 연결되어 있고, c만큼 비용이 발생
        for(int i = 1; i <= M; i++) {
            st = new StringTokenizer(br.readLine());
            int a = Integer.parseInt(st.nextToken());
            int b = Integer.parseInt(st.nextToken());
            int c = Integer.parseInt(st.nextToken());
            graph[a].add(new int[] {b,c});
            graph[b].add(new int[] {a,c });
        }

        //노트 개수만큼 거리 저장 공간 생성, 최소 비용 기록
        int [] dist = new int[N + 1];
        Arrays.fill(dist, Integer.MAX_VALUE);
        dist[1] = 0;

        //Comparator를 사용하여 우선순위를 정의
        PriorityQueue<int[]> pq = new PriorityQueue<>((o1, o2) -> o1[1] - o2[1]);
        
        //시작 노드
        pq.offer(new int[]{1, 0});

        while (!pq.isEmpty()) {
            
            //현재 노드 꺼내기
            int[] cur = pq.poll();
            int now = cur[0];
            int cost = cur[1];
            
            //이미 더 좋은 경로가 있다면 건너뜀
            if (dist[now] < cost) continue;
            
            //인접 노드 탐색
            for (int[] next : graph[now]) {
                //다음 비용 계산
                int nextNode = next[0];
                int nextCost = cost + next[1];
                
               //더 좋은 경로를 발견하면 갱신
                if (dist[nextNode] > nextCost) {
                    dist[nextNode] = nextCost;
                    pq.offer(new int[]{nextNode, nextCost});
                }
            }
        }
        System.out.println(dist[N]);
    }
}
```

### 배운 점 && 보완할 점
- 처음 풀어보는 다익스트라 알고리즘 문제였다.
- 언뜻 보면 BFS, BFS나 DFS로 접근할 수 있을 것 같았지만, 이 문제는 각 간선마다 서로 다른 가중치(비용)가 존재하기 때문에 단순한 탐색이 아닌 최소 비용을 구해야 하는 문제였다.
- 따라서 가중치를 고려한 최단 경로 알고리즘인 다익스트라로 접근해야 했다.

- PriorityQueue<int[]> pq = new PriorityQueue<>((o1, o2) -> o1[1] - o2[1]); Comparator를 사용하여 우선 순위를 정의한다.
- 이때 반환값이 음수면 o1이 우선, 양수면 o2가 우선이다. 반환값이 0이면 두 값의 우선순위가 같다고 판단하며, 출력 순서가 보장되지 않는다.
- 만약 순서까지 보장하고 싶다면 추가 조건을 설정하여 정의할 수 있다.
- (o1, o2) -> {
  if (o1[1] == o2[1]) return o1[0] - o2[0]; // 비용이 같으면 노드 번호 기준
  return o1[1] - o2[1]; // 비용 기준 정렬
  }
