`#그래프_이론` `#다익스트라` `#최단_경로`

### 제출 일자

### 문제 설명 (요약)
농부 현서가 1번 헛간에서 N번 헛간에 있는 찬호에게 택배를 배달한다.
가는 길에는 여러 헛간과 양방향 소들이 있는 길이 있으며, 각 길을 지날 때마다 마주치는 소에게 여물을 줘야 한다.
최소한의 소(비용)를 만나면서 1번 헛간에서 N번 헛간까지 가는 최단 거리를 구하는 문제

- **N**: 헛간(노드)의 수 (1 <= N <= 50,000)
- **M**: 소가 있는 길(간선)의 수 (1 <= M <= 50,000)
- 길의 비용(소의 수)은 0부터 1,000 사이의 정수. (즉, 비용이 0인 길도 존재함)

### 입력
- N, M
- M개의 줄에 걸쳐 각 길의 정보 (출발 노드, 도착 노드, 소의 수)

### 출력
- 1번 헛간에서 N번 헛간까지 가는 최소 소의 수(`int`)

---

### 핵심 풀이 전략
1. N이 최대 50,000이므로 메모리 초과를 막기 위해 2차원 배열 대신 `ArrayList<int[]>[]` 인접 리스트를 사용하여 그래프를 구현.
2. 최단 경로 탐색을 위해 우선순위 큐(`PriorityQueue`)를 활용한 다익스트라(Dijkstra) 알고리즘 사용.
3. 큐에는 `int[]{현재 노드, 누적 비용}` 형태로 넣고, 누적 비용(`a[1]`)을 기준으로 오름차순 정렬되도록 `Comparator.comparingInt` 설정.
4. 최단 거리 배열 `dist`를 `Integer.MAX_VALUE`로 모두 초기화하고, **시작 노드(`dist[1]`)는 0으로 확정** 짓고 시작.
5. 큐에서 꺼낸 누적 비용이 이미 기록된 `dist[현재 노드]` 값보다 크면 무시(방문 최적화).
6. 인접 리스트를 돌면서 연결된 노드들의 누적 비용이 기존 `dist` 값보다 작으면 갱신하고 큐에 삽입.

---

### Java 구현 코드

```java
import java.io.*;
import java.util.*;

public class Main {

    static int N, M;
    public static void main(String[] args) throws IOException {

        BufferedReader br = new BufferedReader(new InputStreamReader(System.in));
        BufferedWriter bw = new BufferedWriter(new OutputStreamWriter(System.out));

        StringTokenizer str = new StringTokenizer(br.readLine());
        N = Integer.parseInt(str.nextToken());
        M = Integer.parseInt(str.nextToken());

        // 0. 배열 초기화
        ArrayList<int[]>[] nodes = new ArrayList[N+1];
        for(int i = 0; i <= N; i++){
            nodes[i] = new ArrayList<>();
        }
        for(int i = 0; i < M; i++){
            str = new StringTokenizer(br.readLine());
            int from = Integer.parseInt(str.nextToken());
            int to = Integer.parseInt(str.nextToken());
            int cost = Integer.parseInt(str.nextToken());

            nodes[from].add(new int[]{to, cost});
            nodes[to].add(new int[]{from, cost});
        }

        // 2. 최솟값
        int result =bfs(1, N, nodes);

        bw.write(result + "");
        br.close();
        bw.flush();
        bw.close();
    }

    public static int bfs(int start, int N, ArrayList<int[]>[] nodes){

        // 1-1. 우선순위 큐 사용
        // 1-2. int[]{curNode, curSum}
        PriorityQueue<int[]> que = new PriorityQueue<>(Comparator.comparingInt((int[] a) -> a[1]));
        int[] dist = new int[N+1];
        Arrays.fill(dist, Integer.MAX_VALUE);
        dist[start] = 0;

        que.offer(new int[]{start, 0});

        // 큐 시작
        while(!que.isEmpty()){

            int[] poll = que.poll();
            int curNode = poll[0];
            int curSum = poll[1];

            if(curSum > dist[curNode]) continue;

            for(int[] n : nodes[curNode]){
                if(dist[n[0]] > curSum + n[1]){
                    que.offer(new int[]{n[0], curSum+n[1]});
                    dist[n[0]] = curSum + n[1];
                }
            }
        }

        return dist[N];
    }
}
```

### 삽질 기록 🧠
- PriorityQueue 정렬 기준을 `간선 비용` 기준으로 정렬하려고 함 다익스트라는 `현재까지 누적 비용` 기준이어야 한다. -> 그래야 `내가 큐에서 꺼내 방문한 곳이 최소 비용임`을 **보장**할 수 있다.

- dist[] 배열 초기화 누락 -> Arrays.fill() 활용하자
  
- 처음에는 간선 정보를 int[][] nodes = new int[N+1][N+1];로 2차원 배열에 담았었다. N이 50,000이라 메모리 초과(MLE)의 위험이 크다는 것을 간과했다.

- 2차원 배열에서 인접 리스트로 자료구조를 바꾼 뒤에도, 배열 시절에 쓰던 예외 처리 로직(if(n[1] != 0) 및 -1 치환)을 그대로 두는 치명적인 실수를 했다. 이 때문에 비용이 0인 '공짜 지름길'을 큐에 넣지 않고 무시해버려서, 엉뚱하게 비싼 길로 돌아가는 바람에 오답이 발생했다.

- 다익스트라 시작 시 dist[start] = 0;으로 시작점의 거리를 0으로 초기화하는 것을 빼먹었다. 이렇게 되면 시작 노드로 다시 돌아오는 불필요한 탐색을 하게 된다.

### 배운 점 && 보완할 점
- 자료구조를 변경할 때는 그에 맞춰 로직도 완전히 새롭게 다듬어야 한다. 2차원 배열에서는 값이 0인 것을 '연결되지 않음'으로 판단했지만, 인접 리스트(ArrayList)는 데이터가 들어있다는 것 자체가 이미 '길이 있다'는 뜻이다. 따라서 간선 비용이 0이든 아니든 거를 필요가 전혀 없다. 레거시 코드를 맹목적으로 남겨두지 말자.

- 다익스트라 알고리즘의 기본 원칙을 확실히 기억하자. 시작 지점의 거리는 무조건 0으로 세팅하고 시작할 것!

- 비용이 1이상으로 다양하다면 `다익스트라(dist[])`를 써야함을 잊지말자.
