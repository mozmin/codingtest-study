### 문제 설명
세로 길이 m, 가로 길이 n인 격자에 비가 순서대로 내린다. <br>
drops[i] = [x, y]는 i+1번째 비가 (x, y) 위치에 떨어진다는 뜻이다. <br>

선인장을 크기 h × w의 직사각형 영역 안에 숨기려고 할 때, 해당 영역이 최대한 늦게 처음 비를 맞도록 하는 시작 좌표를 구하는 문제이다.<br>

가능한 모든 h × w 직사각형 중에서 그 직사각형 내부 칸들이 처음 비를 맞는 시점들 중 최솟값이 가장 큰 위치를 찾아야 한다. 

### 입력
- m: 격자의 세로 길이
- n: 격자의 가로 길이
- h: 선인장을 숨길 직사각형의 세로 길이
- w: 선인장을 숨길 직사각형의 가로 길이
- drops: 비가 떨어지는 위치를 순서대로 담은 2차원 배열

### 출력
- 선인장을 숨길 수 있는 최적의 시작 좌표 [x, y]


### 핵심 풀이 전략
- 먼저 각 칸이 몇 번째 비에 처음 젖는지를 저장하는 time 배열을 만들었다.
- 비가 한 번도 오지 않는 칸은 가장 늦게 젖는 칸으로 보기 위해 큰 값(INF)으로 초기화했다.
- 어떤 h × w 직사각형이 처음 비를 맞는 시점은, 그 직사각형 안에 있는 칸들 중 최솟값으로 결정된다.
- 따라서 모든 직사각형에 대해 “구간 최솟값”을 빠르게 구하는 것이 핵심이었다.
- 이를 위해 2단계 슬라이딩 윈도우를 사용했다.
- 먼저 각 행에서 가로 길이 w 구간의 최솟값을 deque로 구해 rowMin 배열에 저장했다.
- 이후 rowMin의 각 열에 대해 세로 길이 h 구간의 최솟값을 다시 deque로 구했다.
- 이렇게 하면 각 h × w 직사각형의 최솟값을 O(1)처럼 재사용할 수 있어서 전체를 O(m × n)에 처리할 수 있다.
- 최종적으로 직사각형 최소값이 가장 큰 위치를 정답으로 선택했다.


### 제출 코드
```java
import java.util.*;

class Solution {
    public int[] solution(int m, int n, int h, int w, int[][] drops) {
        int INF = drops.length + 1;

        // 1) 각 칸이 처음 비를 맞는 시점
        int[][] time = new int[m][n];
        for (int i = 0; i < m; i++) {
            Arrays.fill(time[i], INF);
        }

        for (int i = 0; i < drops.length; i++) {
            int r = drops[i][0];
            int c = drops[i][1];
            time[r][c] = i + 1; // drops는 서로 다른 칸
        }

        // 2) 가로 길이 w에 대한 행별 슬라이딩 윈도우 최소값
        int colCount = n - w + 1;
        int[][] rowMin = new int[m][colCount];

        for (int r = 0; r < m; r++) {
            Deque<Integer> dq = new ArrayDeque<>();

            for (int c = 0; c < n; c++) {
                // 현재 값보다 큰 뒤쪽 원소 제거
                while (!dq.isEmpty() && time[r][dq.peekLast()] >= time[r][c]) {
                    dq.pollLast();
                }
                dq.offerLast(c); // dq에는 인덱스 값이 들어감 

                // 윈도우 범위 밖 제거
                if (dq.peekFirst() <= c - w) {
                    dq.pollFirst();
                }

                // 길이 w가 되는 시점부터 최소값 기록
                if (c >= w - 1) {
                    rowMin[r][c - w + 1] = time[r][dq.peekFirst()];
                }
            }
        }

        // 3) 세로 길이 h에 대한 열별 슬라이딩 윈도우 최소값
        int rowCount = m - h + 1;
        int bestValue = -1;
        int bestR = 0;
        int bestC = 0;

        for (int c = 0; c < colCount; c++) {
            Deque<Integer> dq = new ArrayDeque<>();

            for (int r = 0; r < m; r++) {
                while (!dq.isEmpty() && rowMin[dq.peekLast()][c] >= rowMin[r][c]) {
                    dq.pollLast();
                }
                dq.offerLast(r);

                if (dq.peekFirst() <= r - h) {
                    dq.pollFirst();
                }

                if (r >= h - 1) {
                    int startR = r - h + 1;
                    int value = rowMin[dq.peekFirst()][c]; // (startR, c) 직사각형의 최소값

                    // 더 늦게 젖는 위치가 우선
                    // 같으면 더 위쪽, 더 왼쪽이 우선 -> 순회 순서상 첫 발견 유지
                    if (value > bestValue) {
                        bestValue = value;
                        bestR = startR;
                        bestC = c;
                    }
                }
            }
        }

        return new int[]{bestR, bestC};
    }
}

```

### 삽길 기록 🧠
처음에는 모든 시작 좌표에 대해 h × w 영역을 직접 돌면서 값을 확인하려고 했다.<br>
하지만 이렇게 하면 직사각형 하나를 확인할 때마다 다시 모든 칸을 봐야 해서 시간초과가 발생했다.<br>
또 처음에는 직사각형 안의 값을 합으로 계산하려고 했는데, 실제로 중요한 것은 선인장이 언제 처음 비를 맞는지이므로 합이 아니라 최솟값을 봐야 한다는 점을 놓쳤다.<br>
비가 오지 않는 칸을 0으로 두면 오히려 가장 빨리 젖는 것처럼 처리될 수 있어서, 큰 값으로 초기화해야 한다는 점도 다시 정리했다.<br>
결국 이 문제는 단순 구현이 아니라, 2차원 구간 최소값을 빠르게 구해야 하는 문제라는 걸 이해한 뒤에 deque 기반 슬라이딩 윈도우로 방향을 바꿨다.<br>
