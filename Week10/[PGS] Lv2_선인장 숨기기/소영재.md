### 문제 설명 (요약)
- 세로, 가로 크기의 격자에 빗방울이 순서대로 떨어진다.
- 크기가 h x w인 선인장 구역을 하나 놓을 때, 빗방울을 한 번도 맞지 않는 구역이 있으면 그 위치를 반환한다.
- 그런 구역이 없다면 해당 구역 안에 포함된 칸들 중 가장 먼저 비를 맞는 시점이 가장 늦은 구역을 찾고, 그 중 가장 위쪽, 그다음 가장 왼쪽 구역의 위치를 반환한다.

### 입력
- int m : 격자의 세로 길이
- int n : 격자의 가로 길이
- int h : 선인장 구역의 세로 길이
- int w : 선인장 구역의 가로 길이
- int[][] drops : 빗방울이 떨어지는 순서대로 칸의 좌표를 담은 2차원 정수 배열

### 출력
- 선인장이 비를 맞지 않는 구역이 있다면 해당 구역의 위치를 반환, 없다면 가장 늦게 비를 맞는 선인장 구역들 중 가장 위쪽, 그다음 가장 왼쪽 위치를 반환

<hr>

### 핵심 풀이 전략
1. m x n 크기의 전체 격자를 2차원 배열로 만들고, 비가 오지 않은 칸은 -1로 초기화한다.
2. drops 배열을 순서대로 순회하면서 각 칸에 몇 번째로 비가 왔는지를 등수로 저장한다.
3. 선인장을 놓을 수 있는 모든 시작 위치를 순회하며 해당 위치에서 시작하는 h x w 구역을 검사한다.
4. 현재 구역 안에 비가 온 칸이 하나도 없다면 그 구역은 선인장이 비를 맞지 않는 위치이므로 바로 정답으로 반환한다.
5. 비가 온 칸이 있다면, 그 구역에서 가장 먼저 비를 맞은 칸의 등수를 구한다.
6. 모든 구역을 비교한 뒤, 비를 전혀 맞지 않는 구역이 없었다면 구역의 최소 등수가 가장 큰 위치를 반환한다.

### 삽질 기록 🧠
```java
import java.util.*;
class Solution {
    public int[] solution(int m, int n, int h, int w, int[][] drops) {
        //전체 구역을 세팅하는 2차원 배열
        //비가 오지 않은 칸은 -1
        //비가 온 칸은 몇 번재로 비가 왔는지 등수를 저장
        int [][] map = new int[m][n];

        for(int i = 0; i < m; i++){
            Arrays.fill(map[i], -1);
        }

        int [] answer = new int[]{0,0};

        //전체 구역에 비가 오는 곳을 등수로 저장
        for(int i = 0; i < drops.length; i++){
            int a = drops[i][0];
            int b = drops[i][1];
            map[a][b] = i+1;
        }

        //정답을 반환할 x,y값을 저장하는 변수
        int answerX = 0;
        int answerY = 0;

        //현재까지 찾은 가장 좋은 구역의 값
        //구역 안에 비가 온 칸들 중 가장 빠른 등수(최소값)을 비교할 값
        int areaValue = -1;
        
        
        //전체 격자를 순회하면서 검사, -h, -w를 한 이유는 구역을 발견하면 선인장 크기만큼 계산해야 하기 때문
        for(int i = 0; i < map.length -h +1; i++){
            for(int j = 0; j < map[i].length -w +1; j++){
                //비가 오는 구역일 때
                if(map[i][j] != -1){
                    //해당 구역의 등수를 저장
                    int ex = map[i][j];
                    //선인장 크기만큼 배열을 탐색하며 만약 비가 오는 곳이라면 등수 비교
                    for(int k = 0; k < h; k++){
                        for(int l = 0; l < w; l++){
                            //비가 오면 등수와 비가 온 곳의 등수 중 작은 곳을 갱신(구역의 최소값)
                            if(map[i+k][j+l] != -1){
                                ex = Math.min(ex, map[i + k][j + l]);
                            }
                        }
                    }
                    // 현재 구역의 최소 등수(ex)가 지금까지의 등수(areaValue)보다 크다면 갱신
                    if (ex > areaValue) {
                        areaValue = ex;
                        answerX = i;
                        answerY = j;
                    }
                }
                //비가 오지 않는 구역일 때
                if (map[i][j] == -1) {
                    boolean isAnswer = true;
                    int ex = Integer.MAX_VALUE;

                    //만약 구역 중 비가 온 곳이 있다면 정답은 아님
                    for (int k = 0; k < h; k++) {
                        for (int l = 0; l < w; l++) {
                            if (map[i + k][j + l] != -1) {
                                isAnswer = false;
                                ex = Math.min(ex, map[i + k][j + l]);
                            }
                        }
                    }
                    //비가 온 곳이 없다면 해당 구역이 정답
                    if(isAnswer == true){
                        answer[0] = i;
                        answer[1] = j;
                        return answer;
                    }
                    // 현재 구역의 최소 등수(ex)가 지금까지의 등수(areaValue)보다 크다면 갱신
                    if (ex > areaValue) {
                        areaValue = ex;
                        answerX = i;
                        answerY = j;
                    }
                }
            }
        }

        answer[0] = answerX;
        answer[1] = answerY;
        return answer;
    }
}
```
- 풀면서 자잘한 오류들을 계속 수정하면서 풀어나갔다.
- 어려웠지만 AI의 도움을 받지 않고 푼 문제라 사실 좀 뿌듯했지만 전체 실행 결과에서 결국 시간 초과가 났다.
- 전체 구역을 검사하면서 선인장 구역만큼 또 반복해서 계산하는 부분에서 시간 복잡도가 크게 증가한 것 같다.
- 최적화를 하자면 첫 if문에서 단순히 들어가는게 아니라 등수를 비교하면서 들어간다면 시간 복잡도를 조금 줄일 수 있을 것 같지만 근본적으로 시간 복잡도가 O(m × n × h × w) 이기 때문에 시간 초과가 동일하게 나왔을 것 같다.
- 문제를 풀고나서 시간 복잡도를 줄일 수 있는 방법에 대해 물어보니 ..어려운 개념이 나와서 일단 스터디를 하면서 얘기해야 겠다.
