### 제출 일자
x

### 문제 설명 (요약)
$m \times n$ 크기의 사막 지도에 순서대로 빗방울이 떨어집니다. 이 사막에 $h \times w$ 크기의 직사각형 공간(선인장이 자랄 수 있는 공간)을 찾으려고 한다. 비를 아예 맞지 않거나, 비를 맞는 시점이 가장 늦은 $h \times w$ 크기의 공간을 찾아 그 공간의 좌측 상단 좌표를 반환하는 문제.

### 입력
- `m`, `n`: 사막 지도의 세로, 가로 길이
- `h`, `w`: 찾아야 하는 공간의 세로, 가로 길이
- `drops`: 빗방울이 떨어지는 좌표와 순서 정보가 담긴 2차원 배열

### 출력
- 조건을 만족하는 공간의 좌측 상단 좌표 `[y, x]`

---

### 핵심 풀이 전략
1. **발상의 전환 (Parametric Search):** "각 칸이 언제 비를 맞을까?"를 구하는 대신, **"T번째 빗방울까지 떨어졌을 때 비를 안 맞는 $h \times w$ 공간이 존재하는가?"**로 질문을 바꾸어 이분 탐색을 진행한다.
2. **지도 세팅:** T(limit)번째 이하의 빗방울 위치만 $m \times n$ 배열에 `1`로 표시한다.
3. **2차원 누적 합 계산:** 매번 $h \times w$ 구역을 순회하지 않기 위해, `(0,0)`부터 `(i,j)`까지의 빗방울 총합을 저장하는 2차원 누적 합 배열을 만든다.
4. **구간 탐색:** $O(1)$의 시간 복잡도를 가지는 누적 합 공식을 이용해, 지도 전체를 슬라이딩하며 빗방울 개수가 `0`인 구역을 빠르게 찾는다.

---

### 내가 풀이한 코드 (시간 초과)
```java
import java.util.*;
class Solution {
    public int[] solution(int m, int n, int h, int w, int[][] drops) {
        
        // m x n 크기의 사막 지도 초기화
        int[][] desert = new int[m][n];
        for(int[] d : desert) Arrays.fill(d, 0);
        
        // 빗방울 표시
        int rank = 1;
        for(int[] drop : drops){
            int y = drop[0];
            int x = drop[1];
            
            desert[y][x] = rank++;
        }
        
        // {dropRank, y, x}
        int[] result = {0, 0, 0};
        
        // 사막을 m x n 만큼 하나씩 반복
        for(int i = 0; i <= m-h; i++){
            for(int j = 0; j <= n-w; j++){
                
                // h x w 검사
                int curCheck = check(i, j, h, w, desert, result);
                
                // 만약 현재 위치가 빗방울을 더 늦게 맞는다면
                if(result[0] < curCheck){
                    result[0] = curCheck;
                    result[1] = i;
                    result[2] = j;
                    
                }else if(curCheck == 0){
                    return new int[]{i, j};
                }
            }
        }
        
        return new int[]{result[1], result[2]};
    }
    
    // return 0 일 경우 빗방울을 하나도 맞지 않았다는 의미
    // return dropRank
    public int check(int y, int x, int h, int w, int[][] desert, int[] result){
        
        // 가장 먼저 맞는 빗방울 순위 초기화
        int firstDrop = Integer.MAX_VALUE;
        
        // h x m 크기 만큼 반복
        for(int i = y; i < y+h; i++){
            for(int j = x; j < x+w; j++){
                
                // 만약 빗방울을 있다면, 가장 먼저 맞는 빗방울인지 검사
                if(desert[i][j] != 0 && firstDrop > desert[i][j]){
                    firstDrop = desert[i][j];
                }
            }
        }
        
        if(firstDrop == Integer.MAX_VALUE) return 0; // 만약 빗방울을 하나도 맞지 않았다면
        else return firstDrop; // 맞은 빗방울이 있다면
        
    }
    
}
```

### 정답 코드 (이분탐색, 누적합 사용)
```java
import java.util.*;

class Solution {
    public int[] solution(int m, int n, int h, int w, int[][] drops) {
        int left = 1; 
        int right = drops.length; 
        int[] bestLocation = {0, 0}; 
        
        // 이분 탐색: 버틸 수 있는 최대 시간(순서)을 찾음
        while (left <= right) {
            int mid = (left + right) / 2;
            
            // mid번째 빗방울까지 떨어졌을 때 안전한 공간이 있는지 확인
            int[] loc = getSafeLocation(mid, m, n, h, w, drops);
            
            if (loc != null) {
                bestLocation = loc;
                left = mid + 1; // 더 늦은 시간까지 버틸 수 있는지 탐색 (Up)
            } else {
                right = mid - 1; // 공간이 없으므로 시간을 줄임 (Down)
            }
        }
        
        return bestLocation;
    }
    
    private int[] getSafeLocation(int limit, int m, int n, int h, int w, int[][] drops) {
        // 1-based index 사용
        int[][] grid = new int[m + 1][n + 1];
        
        // 탐색 기준(limit) 이하의 빗방울만 1로 표시
        for (int i = 0; i < limit; i++) {
            int y = drops[i][0] + 1;
            int x = drops[i][1] + 1;
            grid[y][x] = 1;
        }
        
        // 2차원 누적 합 배열 생성
        int[][] psum = new int[m + 1][n + 1];
        for (int i = 1; i <= m; i++) {
            for (int j = 1; j <= n; j++) {
                psum[i][j] = psum[i - 1][j] + psum[i][j - 1] - psum[i - 1][j - 1] + grid[i][j];
            }
        }
        
        // h x w 크기의 창문을 슬라이딩
        for (int i = h; i <= m; i++) {
            for (int j = w; j <= n; j++) {
                // 2차원 누적합 공식을 이용해 구간 내 빗방울 합 계산
                int dropsInWindow = psum[i][j] - psum[i - h][j] - psum[i][j - w] + psum[i - h][j - w];
                
                if (dropsInWindow == 0) {
                    return new int[]{i - h, j - w}; // 0-based 원래 좌표로 복구하여 반환
                }
            }
        }
        
        return null;
    }
}
```
### 삽질 기록 🧠
1. 사막 전체를 순회하는 2중 for문 안에, 해당 영역에 빗방울이 있는지 검사하는 2중 for문을 또 넣어서 **4중 for문**을 돌렸다. 결과는 당연히 시간 초과(TLE). 시간 복잡도가 $O(m \cdot n \cdot h \cdot w)$로 지도가 조금만 커져도 터지는 구조였다.
2. 시간 초과를 잡으려고 흔히 아는 점화식 기반의 정통 DP로 접근하려 했으나, 규칙을 찾지 못해 헤맸다.
3. 2차원 누적 합 공식을 처음 봤을 때, 특정 구간을 구할 때 왜 굳이 위쪽과 왼쪽을 빼는지 직관적으로 이해하지 못했다.

### 배운 점 && 보완할 점
1. **이분 탐색의 활용성:** "최댓값/최솟값 찾기" 문제에서 조건을 만족하는지 묻는 "결정 문제(업다운 게임)"로 시각을 바꾸는 **매개변수 탐색(Parametric Search)** 기법을 익혔다.
2. **2차원 누적합 초기화 공식:** 누적 합 배열은 항상 `(0,0)`부터의 합만 가지고 있다. 따라서 누적합을 구하기 위한 공식은 아래와 같다.
$$P[i][j] - P[i-1][j] - P[i][j-1] + P[i-1][j-1] + grid[i][j]$$
4. **특정 구역 누적합 공식:** 이때 왼쪽 위 대각선 구역은 두 번 빼지게 되므로 한 번 다시 더해줘야 한다. 이 원리가 적용된 공식 
$$P[i][j] - P[i-h][j] - P[i][j-w] + P[i-h][j-w]$$
은 2차원 배열의 구간 합을 $O(1)$로 구할 때 필수적이니 꼭 외워두자.
