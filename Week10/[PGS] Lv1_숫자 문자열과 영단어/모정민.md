### 제출 일자

### 문제 설명 (요약)
네오가 프로도에게 숫자를 건넬 때 일부 자릿수를 영단어로 바꾸어 건넵니다. (예: 1478 -> "one4seveneight")
이렇게 숫자의 일부가 영단어로 바뀌어 있는 문자열 `s`가 매개변수로 주어질 때, 원래의 숫자를 찾아 반환하는 문제.

- **s**: 영단어와 숫자가 섞여 있는 문자열 (1 <= s의 길이 <= 50)
- "zero"부터 "nine"까지의 영단어가 각각 0부터 9에 대응됨.
- `s`가 "zero" 또는 "0"으로 시작하는 경우는 주어지지 않음.

### 입력
- `s`: 변환해야 할 영단어가 포함된 문자열

### 출력
- 원래 의미하는 정수(`int`)

---

### 핵심 풀이 전략
1. Map<>으로 숫자와 영단어 매핑
2. for문을 돌면서 숫자면은 그대로 ArrayList<String>에 add()
3. 만약 문자열이면 계속 String 변수에 플러스
4. 반복마다 Map<> 키 검사를 하며 키에 속할경우 매핑된 숫자를 ArrayList<String>에 add()
5. 최종적으로 ArrayList<String> -> String[] -> int로 변환

---

### Java 구현 코드

```java
import java.util.*;
class Solution {
    public int solution(String s) {
        ArrayList<String> list = new ArrayList<>();
            
        Map<String, String> map = new HashMap<>();
        map.put("zero", "0");
        map.put("one", "1");
        map.put("two", "2");
        map.put("three", "3");
        map.put("four", "4");
        map.put("five", "5");
        map.put("six", "6");
        map.put("seven", "7");
        map.put("eight", "8");
        map.put("nine", "9");
        
        String ch = "";
        for(int i = 0; i < s.length(); i++){
            char c = s.charAt(i);
            
            if (c - '0' >= 0 && c - '0' <= 9) {
                list.add(String.valueOf(c));
            } else {
                ch += c;
                
                if (map.containsKey(ch)) {
                    list.add(map.get(ch));
                    ch = "";
                }
            }
        }
            
        String[] stringList = list.toArray(new String[0]);
        String result = String.join("", stringList);
        int answer = Integer.parseInt(result); 
        return answer;
    }
}
```

### 삽질 기록 🧠
1. `if (c - '0' >= 0 && c - '0' <= 9)`으로 숫자인지 문자인지 판별할 때, 숫자 0 또는 9 대신 '0', '9'를 사용해서 틀렸다.
2. String ch 대신 Character ch 로 선언하여 풀이했었다.
3. 문자가 나오다가 숫자가 나올 때, Map<> 파싱을 시도했다. 하지만 문자열 다음 바로 문자열이 오는 경우도 있기 때문에 최종적으로는 containsKey()를 사용하였다.

### 배운 점 && 보완할 점
1. 개선할게 무척 많다.
- map.put()으로 수동으로 넣고 있는데 이런 방식보다는 replace()를 사용하는게 훨씬 효율적임을 알게 되었다. 굳이 매핑을 위한 자료구조를 쓰지말고 문자열을 바로 치환하면 조금 더 깔끔한 코드가 된다.
- `if (c - '0' >= 0 && c - '0' <= 9)` -> `if (Character.isDigit(c))` 로 숫자 판별을 대신할 수 있다.
- `String ch` 대신 `StringBuilder`를 사용하는 것이 메모리 관리 차원에서 훨씬 좋다.
